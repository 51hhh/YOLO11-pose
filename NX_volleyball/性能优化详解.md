# 排球追踪系统 - 性能优化详解

## 🎯 优化目标
- **当前性能**: ~20 FPS (采集14ms + 检测35ms)
- **目标性能**: ~100 FPS (<10ms总延迟)

---

## 📊 性能瓶颈分析

### 原始性能分布
```
采集:   13.94ms (28.5%)  ← 图像拷贝开销
检测:   34.89ms (71.3%)  ← CPU预处理 + 同步推理
立体:    0.05ms (0.1%)
追踪:    0.03ms (0.1%)
─────────────────────────
总计:   48.92ms → 20.4 FPS
```

### 主要瓶颈
1. **YOLO检测 (35ms)**: 
   - CPU预处理: resize + BGR2RGB + normalize + HWC2CHW (~10ms)
   - 同步数据传输: CPU ↔ GPU 拷贝 (~5ms)
   - 同步推理等待: executeV2阻塞 (~20ms)

2. **相机采集 (14ms)**:
   - 像素格式转换: SDK Buffer → BGR8 (~8ms)
   - 内存分配: new/delete每帧 (~3ms)
   - 数据拷贝: Mat.clone() (~3ms)

## ⚡ 优化方案

### 1. CUDA异步流 (Stream Pipeline)

#### 原理
使用CUDA流实现**采集→预处理→推理→后处理**的流水线并行。

#### 实现
```cpp
// 创建异步流
cudaStream_t cuda_stream_;
cudaStreamCreate(&cuda_stream_);

// 异步拷贝 (使用pinned memory自动加速)
cudaMemcpyAsync(dst, src, size, cudaMemcpyHostToDevice, cuda_stream_);

// 异步推理 (不阻塞CPU)
context_->enqueueV2(bindings, cuda_stream_, nullptr);

// 异步回传
cudaMemcpyAsync(dst, src, size, cudaMemcpyDeviceToHost, cuda_stream_);

// 仅在需要结果时同步
cudaStreamSynchronize(cuda_stream_);
```

#### 预期收益
- **同步拷贝**: Host→Device (2ms) + Inference (20ms) + Device→Host (3ms) = **25ms串行**
- **异步流**: Host→Device ‖ Inference ‖ Device→Host = **~20ms并行** (重叠5ms)
- **减少等待**: CPU不阻塞，可并行处理下一帧
- **加速**: ~5-8ms

---

### 2. 零拷贝Pinned Memory

#### 原理
使用页锁定内存(pinned memory)绕过OS分页机制，实现DMA直接传输。

#### 实现
```cpp
// 原始: 分页内存 (可能被swap)
input_buffer_host_ = new float[size];

// 优化: 页锁定内存 (不会被swap，DMA加速)
cudaMallocHost(&input_buffer_host_, size);
```

#### DMA传输路径对比
```
分页内存:   [Host Pageable] → [Host Pinned临时] → [GPU]  (2次拷贝)
Pinned内存: [Host Pinned] ──────────────────────→ [GPU]  (1次拷贝)
```

#### 预期收益
- **传输速度**: 提升 **2-3倍** (PCIe带宽充分利用)
- **延迟减少**: ~2-3ms

---

### 3. GPU加速预处理

#### 原理
将Resize + BGR2RGB + Normalize + HWC2CHW全部在GPU上完成。

#### CPU预处理 (原始)
```cpp
cv::resize(image, resized, Size(640, 640));        // ~4ms
cv::cvtColor(resized, rgb, COLOR_BGR2RGB);         // ~2ms
rgb.convertTo(rgb, CV_32F, 1.0/255.0);             // ~2ms
// HWC→CHW循环                                      // ~2ms
```

#### GPU预处理 (优化)
```cpp
__global__ void preprocessKernel(uchar* src, float* dst, ...) {
  // 双线性插值resize
  float src_x = x * scale_x;
  float src_y = y * scale_y;
  
  // BGR→RGB + 归一化 + CHW布局
  for (int c = 0; c < 3; ++c) {
    int src_c = 2 - c;  // BGR→RGB
    float v = interpolate(src, src_x, src_y, src_c) / 255.0f;
    dst[c * H * W + y * W + x] = v;  // CHW
  }
}
```

#### 预期收益
- **并行度**: CPU串行 → GPU 16x16 block = **256并行线程**
- **内存带宽**: CPU DDR4 ~30GB/s → GPU LPDDR5 ~100GB/s
- **加速**: ~10ms → **~2ms** (减少8ms)

---

### 4. 相机采集零拷贝

#### 原理
- **零拷贝路径**: 直接使用SDK缓冲区 (如果已是BGR8)
- **复用缓冲区**: 使用thread_local避免每帧new/delete

#### 实现
```cpp
// 原始: 每帧new/delete
unsigned char* convert_data = new unsigned char[size];
// ... 转换 ...
image = Mat(..., convert_data).clone();
delete[] convert_data;

// 优化: 复用thread_local缓冲区
static thread_local std::vector<unsigned char> convert_buffer;
if (convert_buffer.size() < size) {
  convert_buffer.resize(size);  // 仅首次分配
}
// ... 转换到复用缓冲区 ...
image = Mat(..., convert_buffer.data()).clone();
```

#### 预期收益
- **内存分配**: 每帧new/delete (~3ms) → 0ms
- **拷贝次数**: 3次 → 2次
- **加速**: ~3-5ms

## ⚠️ 注意事项

### CUDA版本要求
- **Jetson Orin NX**: CUDA 11.4+, TensorRT 8.5+
- 确认版本: `nvcc --version`

### 内存限制
- **Pinned Memory**: 最多物理内存的50% (~8GB可用4GB)

## 🎓 技术原理深度解析

### CUDA Stream并行度
```
Frame N:   [采集] → [GPU预处理] → [推理] → [后处理]
Frame N+1:         [采集] → [GPU预处理] → [推理] → ...
                   ↑───────重叠时间─────↑
```

### Pinned Memory工作原理
```
OS内存管理:
┌─────────────┐
│ Virtual Mem │ ← 可能被swap到磁盘
└─────────────┘
      ↓ Page Fault可能
┌─────────────┐
│Physical Mem │
└─────────────┘

Pinned Memory:
┌─────────────┐
│ Physical    │ ← 锁定在物理内存，DMA可直接访问
│ Memory      │
└─────────────┘
      ↓ 零拷贝DMA
┌─────────────┐
│ GPU Memory  │
└─────────────┘
```

### GPU Kernel并行度
```
单个Block: 16x16 = 256线程 (并行处理256像素)
Grid配置: (640/16, 640/16) = 40x40 = 1600个Block
总并行度: 256 x 1600 = 409,600线程同时运行!
```
