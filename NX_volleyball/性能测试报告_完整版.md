## 7. DLA 部署指南

### 7.1 DLA 可用性确认

```
硬件: 2× NVDLA v2.0 核心 ✅
内核模块: nvhost_nvdla 已加载 ✅
控制设备: /dev/nvhost-ctrl-nvdla0, nvdla1 ✅
TensorRT DLA 构建: 成功 ✅  (trtexec --useDLACore=0 --int8 --allowGPUFallback)
```

### 7.2 DLA 层兼容性 (YOLOv26n)

| 类型 | 运行位置 | 层数 | 占比 |
|------|---------|------|------|
| Conv, Sigmoid, Mul, Add, MaxPool, Split, Concat | **DLA** | 367 | 96.3% |
| MatMul, Softmax (attention), Reshape (后处理) | GPU 回退 | 14 | 3.7% |

### 7.3 构建 DLA 引擎命令

```bash
# 单 DLA 核心
trtexec --onnx=yolo26n.onnx \
  --saveEngine=yolo26n_dla0_int8.engine \
  --int8 --useDLACore=0 --allowGPUFallback

# 另一个 DLA 核心
trtexec --onnx=yolo26n.onnx \
  --saveEngine=yolo26n_dla1_int8.engine \
  --int8 --useDLACore=1 --allowGPUFallback
```

### 7.4 DLA 注意事项（来自 NVIDIA 教程）

1. **DLA 仅支持 INT8/FP16 精度**, 必须指定 `--int8`
2. **不支持的算子自动回退 GPU**, 需要 `--allowGPUFallback`
3. DLA 不支持: GroupNorm, GlobalAveragePool (可用 AveragePool 替代), dynamic shapes
4. 模型必须是**静态 shape** (不能用 `--shapes` 参数)
5. 使用 ONNX Graph Surgeon 可替换不兼容层以提高 DLA 覆盖率
6. 推荐用 `nsys profile --trace=cuda,nvtx,cublas,cudla,cusparse,cudnn,nvmedia` 分析 DLA timeline
