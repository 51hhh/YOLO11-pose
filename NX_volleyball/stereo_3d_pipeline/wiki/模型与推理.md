# 模型与推理

## 生产推理路径

实时检测使用 TensorRT engine。配置入口:

```yaml
detector:
  engine_path: "/home/nvidia/NX_volleyball/model/yolo26_gpu_fp16.engine"
  input_size: 640
  input_format: "bgr"
  use_dla: false
  dual_yolo:
    enabled: true
```

`detector` 和 `neural_feature_matching` 的完整字段说明见 [配置参考](配置参考.md)。构建目标和条件依赖见 [构建目标与依赖](构建目标与依赖.md)。

模型和引擎结论必须随配置一起记录:

- 模型文件名和训练数据版本。
- engine 构建命令。
- 精度模式: FP16、INT8、DLA/GPU。
- NX 上 `trtexec` 延迟和全管线 fps。
- 是否启用左右双 YOLO。

## 神经特征匹配

DISK、SuperPoint、LightGlue、XFeat 等神经特征匹配模型先作为离线验证和专项实时优化候选，不应直接进入默认 100fps 路径。当前本地 probe 已覆盖 XFeat、ALIKED、SuperPoint+LightGlue；DISK 尚未接入本地 probe 或 C++ 实时后端。

进入实时路径前必须满足:

- ROI 小图推理和匹配总耗时在目标预算内。
- 输出 10 个左右稳定关键点时 validation pass rate 明显优于传统方法。
- TensorRT 或 CUDA 路径明确，避免 Python/Torch 运行时进入生产程序。
- 输入、输出、归一化、坐标反变换和显存生命周期写入文档。

当前 C++ TensorRT 神经特征路径、fused engine 输入输出限制和离线 Python probe 见 [特征匹配实现](特征匹配实现.md)。

XFeat extractor-only engine 可通过 `scripts/export_xfeat_extractor_onnx.py` 和 `scripts/nx_build_xfeat_extractor_engine.sh` 构建；该路径当前仍会在 C++ 后处理阶段回到 CPU 做关键点筛选和匹配。ALIKED/SuperPoint 等若导出固定 shape `keypoints/descriptors/scores`，C++ 可走 direct extractor fallback 做真实 descriptor mutual-NN；`use_lightglue=true` 时 `matcher_engine_path` 支持固定 schema split matcher，但复杂动态 LightGlue schema 仍建议优先导出 fused 点对 engine。

## 输入和后端差异

实时路径不能只看模型名，必须同时看输入通道、执行后端和 CPU round-trip。

| 路径 | 输入 | GPU 工作 | CPU 工作 | 实时内存要求 |
|---|---|---|---|---|
| YOLO 检测 | BGR 或 gray，取决于 engine | TensorRT detector | collect/NMS 结果整理 | 使用 `FrameSlot` 中校正图，检测前等待 rect event |
| 几何/patch ROI 候选 | 校正 gray GPU | 自研 CUDA batch kernel | 候选 gate 和记录 | async worker 固定复制 gray GPU snapshot |
| 彩色 IoU/edge patch | 校正 BGR GPU | 自研 CUDA batch kernel | 候选 gate 和记录 | 只有 BGR 管线且彩色候选启用时复制 BGR GPU snapshot |
| OpenCV CUDA ORB | 校正 gray GPU | `cv::cuda::ORB` 和 CUDA matcher | keypoints/matches 元数据、几何 gate、MAD 聚合 | 不需要 host gray；不会为了 ORB 单独 D2H |
| OpenCV BRISK/AKAZE/SIFT | 校正 gray host | 无真实 CUDA 后端 | OpenCV CPU detect/describe/match 和 gate | 配置启用时预分配 pinned host gray，按帧需要时 D2H |
| XFeat extractor-only | 1ch gray 或 3ch BGR ROI | TensorRT extractor | NMS、descriptor 采样、mutual-NN、gate | 输入在 GPU；输出 D2H；3ch engine 需要 BGR snapshot |
| ALIKED/SuperPoint direct extractor | 1ch gray 或 3ch BGR ROI | TensorRT extractor | descriptor mutual-NN、gate | 要求 engine 输出固定 `keypoints/descriptors/scores` schema |
| Split LightGlue matcher | extractor 输出点/描述子 | TensorRT matcher | schema 整理和最终 gate | 当前存在中间 D2H/H2D；只支持固定 schema |
| Fused neural matcher | 2ch gray-pair、6ch BGR-pair，或两个 1/3ch 输入 | TensorRT fused matcher | 输出点对 gate | 优先实时验证路径；6ch/3ch 需要 BGR snapshot |

3/6 通道神经 engine 在 gray-only 管线中会被标记为 unsupported input，不会把 gray 指针当彩色图读。Python probe 只用于验证算法语义和门限，不等同于 NX TensorRT 实时性能。

## 推荐管理方式

- `scripts/` 放 engine 构建和部署脚本。
- `tools/` 放本机 CPU/GPU 验证工具。
- `config/` 放运行时是否启用对应模型的开关。
- Wiki 只写当前推荐路径和验证过的替代路径。
- 历史模型报告迁移到 [文档迁移索引](文档迁移索引.md)。

SOT tracker 的 NanoTrack/MixFormer engine、YOLO 帧间补框和 ROI 深度复用路径见 [追踪与补帧](追踪与补帧.md)。
