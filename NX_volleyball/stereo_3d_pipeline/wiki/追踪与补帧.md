# 追踪与补帧

最后核对: 2026-07-02

本页说明 SOT tracker 在实时管线中的位置。当前主配置 `pipeline_dual_yolo_roi.yaml` 中 tracker 默认关闭；启用前必须单独做 NX 性能和深度稳定性测试。

## 配置入口

解析位置: `src/main.cpp`

```yaml
tracker:
  enabled: false
  type: "nanotrack"
  engine_path: ""
  search_engine_path: ""
  head_engine_path: ""
  detect_interval: 3
  lost_threshold: 5
  min_confidence: 0.3
```

字段:

| 字段 | 说明 |
|---|---|
| `enabled` | 是否启用 YOLO 帧间 SOT 补帧 |
| `type` | `nanotrack` 或 `mixformer` |
| `engine_path` | TensorRT engine；NanoTrack 可作为 template backbone |
| `search_engine_path` | NanoTrack search backbone engine |
| `head_engine_path` | NanoTrack head engine |
| `detect_interval` | 每 N 帧跑一次 YOLO，非检测帧由 tracker 填充 |
| `lost_threshold` | 连续无有效输出达到阈值后重置 tracker |
| `min_confidence` | tracker 输出最低置信度 |

`PipelineConfig::TrackerConfig` 里还有 `gpu_sync_mode` 字段，但当前 YAML 解析没有读取它。若后续启用，需要同步 `src/main.cpp`、配置和本页。

## 与 ROI 管线的关系

ROI 调度中:

- 检测帧: `stage1_detect()` 提交 YOLO，Stage2 collect YOLO 后调用 `tracker_handle_detect_result()` 刷新 tracker template。
- 非检测帧: Stage1 等待校正完成后调用 `tracker_infill()`，生成 `sot_bbox_result`。
- tracker 填充帧: Stage2 调用 `stage2_roi_fuse_tracker()`，把 tracker bbox 转成 `Detection`，复用 ROI match 和深度融合路径。

当前检测节拍是固定的:

```cpp
slot.frame_id % effective_detect_interval_ == 0
```

代码注释明确当前不再因为 tracker 是否处于 TRACKING 状态而抢跑检测。

## 性能注意事项

当前 ROI 管线为了避免 NX 上 YOLO 和 tracker 并行导致功耗或资源冲突，在 tracker infill 前会显式等待已提交 YOLO 完成:

- `cudaStreamSynchronize(streams_.cudaStreamDLA)`
- dual YOLO 时同步右目 stream

这意味着 tracker 不是“免费补帧”。启用后必须重新测试:

- `Stage1_TrackerInfill`
- `Stage1_WaitYOLOComplete`
- `Stage2_ROIFuseTracker`
- 总 FPS 和 `stale_drop`
- GPU/DLA/CPU 占用和温度

## 失效处理

检测帧没有 YOLO detection:

- `tracker_lost_count_` 增加。
- 达到 `lost_threshold` 后 tracker reset，状态回到 IDLE。

非检测帧 tracker 输出无效或低于 `min_confidence`:

- 当前帧不使用 tracker bbox。
- 连续失败达到 `lost_threshold` 后 reset。

## 启用前检查

1. 确认 TensorRT engine 路径存在且与当前输入尺寸匹配。
2. 先在短时实机测试中只启用 tracker，不同时打开重 ROI 特征。
3. 对比启用前后的 bbox y 残差、深度 MAD、帧间深度抖动。
4. 记录 tracker 填充帧和 YOLO 检测帧的深度差异。
5. 若 tracker bbox 偏移导致深度变差，应保持默认关闭。
