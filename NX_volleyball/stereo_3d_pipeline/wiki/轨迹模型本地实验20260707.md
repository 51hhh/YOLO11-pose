## ReliabilityNet 加权物理平滑

新增 `trajectory_fusion/evaluate_reliability_smoother.py`，用于验证推荐架构:

```text
ReliabilityNet 输出 per-method bias/sigma/outlier_prob
  -> z_corrected = z_raw - bias
  -> variance = sigma^2 / inlier_prob
  -> 按方法族分组，保留族内分歧为额外方差
  -> robust_smoother 物理状态更新
```

这条链路和 `evaluate_reliability_checkpoint.py` 的区别是: 模型不再直接输出最终 `smooth_z`，只影响每路观测的偏置修正和方差，最终轨迹仍由物理滤波器生成。

| 方案 | smooth mean | smooth std | peak-to-peak | raw std ratio | 判断 |
|---|---:|---:|---:|---:|---|
| robust smoother, 手工 sigma | 3.5239m | 0.0021m | 0.0154m | 0.049 | 当前未知真值 baseline |
| ReliabilityNet direct h32/e120/bias1.0 | 3.4738m | 0.0053m | 0.0742m | 0.122 | 直接 consensus，仍有漂移 |
| reliability smoother h32/e80 | 3.4975m | 0.0046m | 0.0333m | 0.107 | learned sigma 接物理层，可跑 |

## 关键结论

1. 当前最可靠的本地方案是“候选方法分组 + 鲁棒物理 smoother”，不是单独 GRU consensus。
2. ReliabilityNet 的职责仍应是输出每路候选的 `bias/sigma/outlier_prob`，最终轨迹必须交给 EKF/UKF/RTS 或当前 `robust_smoother` 类物理滤波器。
3. 没有真实 `known_z`、平面、落点或重投影弱标签时，纯自监督 ReliabilityNet 会出现绝对深度漂移。`static_jitter` 只能让曲线更平滑，不能保证深度正确。
4. 已在训练脚本中加入 `--bias-reg-weight` 和 `bias_regularizer()`，用于抑制无标签阶段的全方法 bias 漂移。它只能降低风险，不能替代真实距离标签。
5. 明天采集数据时，必须至少覆盖:
   - 静止已知距离，多个距离点。
   - 静止不同画面位置，验证 bbox/circle anchor 的系统偏差。
   - 轻微上下/前后移动，验证速度/加速度特征。
   - 抛球/击球，验证物理轨迹残差和落点预测。

## 推荐下一阶段模型结构

```text
原始候选 z/disparity/support/std/confidence/top5/pair gate/dy
  -> MeasurementReliabilityNet
     输出每路 bias/sigma/outlier_prob
  -> 分组鲁棒观测构造
  -> EKF/UKF/RTS 物理轨迹滤波
  -> x/y/z/v/a 和落点预测
```

不要把 legacy 在线 `x/y/z/v/a` 作为 ReliabilityNet 的训练输入；本轮 `--use-online-position` 已显示它会把深度稳定性拉差。后续若要加入 `xy/v/a`，应由每个原始候选自身反投影和短窗差分生成，而不是复用旧在线融合状态。

## 复现实验命令

```bash
CSV=NX_volleyball/stereo_3d_pipeline/test_logs/fullres_zoom_xfeat_20260705_132756/fullres_artifacts_xfeat_20260705_132756.csv
META=NX_volleyball/stereo_3d_pipeline/test_logs/trajectory_model_experiments_20260707/fullres_artifacts_xfeat_static.metadata.yaml
OUT=NX_volleyball/stereo_3d_pipeline/test_logs/trajectory_model_experiments_20260707

python3 NX_volleyball/stereo_3d_pipeline/trajectory_fusion/check_dataset.py "$CSV"

python3 NX_volleyball/stereo_3d_pipeline/trajectory_fusion/robust_smoother.py \
  "$CSV" --metadata "$META" --gravity-y 0 \
  -o "$OUT/fullres_artifacts_xfeat_robust_smooth.csv"

.venv-stereo-neural/bin/python NX_volleyball/stereo_3d_pipeline/trajectory_fusion/train_reliability.py \
  "$CSV" --metadata "$META" --epochs 120 --hidden 32 \
  --static-jitter-weight 1.0 --bias-reg-weight 1.0 \
  --known-z-weight 0 --known-z-range-weight 0 \
  -o "$OUT/reliability_static_h32_e120_j1_bias10.pt"

.venv-stereo-neural/bin/python NX_volleyball/stereo_3d_pipeline/trajectory_fusion/evaluate_reliability_checkpoint.py \
  "$OUT/reliability_static_h32_e120_j1_bias10.pt" "$CSV" \
  --metadata "$META" \
  -o "$OUT/fullres_artifacts_xfeat_reliability_h32_e120_j1_bias10.csv"

.venv-stereo-neural/bin/python NX_volleyball/stereo_3d_pipeline/trajectory_fusion/evaluate_reliability_smoother.py \
  "$OUT/reliability_static_h32_e120_j1_bias10.pt" "$CSV" \
  --metadata "$META" --gravity-y 0 \
  -o "$OUT/fullres_artifacts_xfeat_reliability_smoother_h32_e120_j1_bias10.csv" \
  --json-out "$OUT/reliability_smoother_h32_e120_j1_bias10_apply.json"
```
