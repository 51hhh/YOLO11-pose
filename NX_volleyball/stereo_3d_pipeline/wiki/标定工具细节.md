最后核对: 2026-07-02

本页记录当前 C++ 标定采集和求解工具参数。高层流程见 [相机与标定](相机与标定.md)。

## capture_chessboard

文件: `src/calibration/capture_chessboard.cpp`

正式命令:

```bash
./capture_chessboard \
  -o calibration_images \
  -g 17.0 \
  --serial-left 00D39342665 \
  --serial-right 00219471413
```

参数:

| 参数 | 说明 |
|---|---|
| `-o DIR` | 输出目录，默认 `calibration_images` |
| `-e US` | 曝光时间，单位微秒 |
| `-g DB` | 增益 dB，默认 11.9906 |
| `--free-run` | 自由运行，不使用硬触发 |
| `--no-pwm` | 不启动 PWM 输出 |
| `--headless` | SSH/无 GUI 模式 |
| `-n COUNT` | 自动保存 COUNT 对后退出，会启用 headless |
| `--width W`, `--height H` | 相机分辨率，默认 1440x1080 |
| `--left-index N`, `--right-index N` | 按索引绑定相机 |
| `--serial-left S`, `--serial-right S` | 按序列号绑定相机，推荐 |
| `--image-node-num N` | SDK FIFO 深度，默认 3 |

交互键:

| 按键 | 动作 |
|---|---|
| 空格 | 保存当前同步帧 |
| `c` | 清空已采集图像 |
| `q` 或 `ESC` | 退出 |

采集工具不做实时棋盘检测和质量评分，目的是保持 100fps 触发和流畅显示。质量检查放到求解阶段和人工预览阶段。

## stereo_calibrate

文件: `src/calibration/stereo_calibrate.cpp`

正式命令:

```bash
./stereo_calibrate -s 26.0 --board-w 5 --board-h 8
```

参数:

| 参数 | 说明 |
|---|---|
| `-s`, `--square-size MM` | 棋盘格边长，必填，单位 mm |
| `-d`, `--images-dir DIR` | 图像目录，默认 `calibration_images` |
| `-o`, `--output FILE` | 输出 YAML，默认 `stereo_calib.yaml` |
| `--board-w N` | 内角点宽度，默认 5 |
| `--board-h N` | 内角点高度，默认 8 |
| `--no-vis` | 跳过可视化预览 |
| `--gpu-preprocess` | CUDA 预处理 Bayer/BGR 到灰度 |
| `--optimize-intrinsics` | stereoCalibrate 同时优化内参，默认固定单目标定内参 |
| `--sb` | classic 棋盘检测失败后启用 SB fallback |
| `--exhaustive` | 启用更慢的 SB exhaustive fallback |
| `--jobs N` | 并行角点检测 worker 数；0 表示自动 |

当前求解步骤:

1. 读取左右图像对。
2. 检测棋盘角点，必要时使用 fallback。
3. 左右单目标定。
4. 生成每张图的重投影诊断。
5. 双目标定。
6. stereo rectification。
7. 保存 YAML 和深度精度报告。
8. 可视化校正结果。

## 长基线采集要求

长基线下不要依赖“删掉坏图”来掩盖采集问题。优先重新采集:

- 图像覆盖全视场，尤其四角和边缘。
- 棋盘有不同距离、倾角和朝向。
- 避免所有棋盘都近似平行于成像平面。
- 棋盘平整，角点清晰，无运动模糊。
- 左右曝光/增益固定，避免自动曝光造成亮度漂移。
- `frame_counter_delta` 稳定为 0。

## 输出检查

标定结束后至少记录:

- 左右单目 RMS。
- stereo RMS。
- baseline 是否接近实测。
- `T` 向量方向和尺度。
- 校正预览中对应角点是否水平对齐。
- 实时排球左右 bbox/circle 的 y 残差是否稳定。

如果 stereo RMS 很低但实时 y 残差系统性偏大，优先检查棋盘尺寸、内角点数量、左右图是否错配、图像是否被错误 resize 或裁剪。
