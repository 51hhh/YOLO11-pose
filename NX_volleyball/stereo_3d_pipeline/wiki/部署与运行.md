# 部署与运行

## NX 环境

目标平台:

- Jetson Orin NX Super 16GB。
- JetPack 6.x。
- CUDA、TensorRT、VPI、OpenCV、yaml-cpp。
- 海康 MVS SDK。
- libgpiod。

具体版本以当前 NX 实机为准。性能测试前必须记录 JetPack、CUDA、TensorRT、VPI、模型 engine、配置文件和 git commit。

## 构建

联合项目必须构建ROS2版本：

```bash
cd ~/NX_volleyball/stereo_3d_pipeline
source /opt/ros/humble/setup.bash
source ~/volleyball_ros2_ws/install/setup.bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DCUDA_ARCH=87 \
  -DREQUIRE_ROS2=ON \
  -DCMAKE_CUDA_COMPILER=/usr/local/cuda-12.6/bin/nvcc \
  -DCUDAToolkit_ROOT=/usr/local/cuda-12.6
cmake --build build -j$(nproc)
```

`REQUIRE_ROS2=ON`会在找不到`ament_cmake`或`volleyball_interfaces`时直接失败，禁止静默生成无ROS bridge版本。独立标定工具需要无ROS构建时，才使用`build_standalone`。

构建产物、条件依赖和 CMake target 见 [构建目标与依赖](构建目标与依赖.md)。

## 运行

首次联合部署先将NX系统时钟锁定到RDK。正式直连地址为NX `10.43.0.10/24`、
RDK `10.43.0.20/24`：

```bash
sudo scripts/install_rdk_chrony_client.sh
```

RDK端需要先运行当前 `USB2CAN_motor` 仓库中的
`deploy/transport/install_chrony_rdk_server.sh`。

正式联合运行使用统一脚本。它默认使用NX `10.43.0.10`、RDK `10.43.0.20` 和
`ROS_DOMAIN_ID=42`，会先确认Chrony偏差小于2 ms，再启动双目管线；管线生成唯一
`source_epoch` 后，脚本在NX启动 `nx_time_sync_publisher`：

```bash
cd ~/NX_volleyball/stereo_3d_pipeline
VOLLEYBALL_ROS_SETUP=~/volleyball_ros2_ws/install/setup.bash \
scripts/start_rdk_joint_runtime.sh
```

RDK不得启动 `nx_time_sync_publisher`。`NxBallObservation.header.stamp` 由左相机设备时钟
映射到NX `CLOCK_REALTIME`，映射预热期间或不确定度超过2 ms时消息会标记为无效并被RDK
拒绝。`timestamp_offset_us` 用于以后按实测校正到曝光中点；目前为0，因为尚未确认海康
设备时间戳对应曝光开始还是结束。

正式配置关闭 XFeat、P2 实时特征、ROS 图像诊断和 CSV 录制；专项调试继续使用
`pipeline_dual_yolo_roi.yaml`。配置文件用途见 [配置矩阵](配置矩阵.md)。

显示窗口:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml --visualize
```

## 录制基准片段

用于离线特征点、双目测距和彩色 patch 算法对比时，直接录左右校正图和检测框 CSV:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 2.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

只需要连续左右目图像、不等待检测框触发时，加 `--baseline-start-immediately`。录制默认先缓存到内存，clip 完成后再写盘；详细字段、输出结构和检查命令见 [录制与离线评估](录制与离线评估.md)。

## 性能准备

测试前 **必须** 锁频（否则 GPU 动态调频会导致 YOLO 推理抖动、stale drop 暴增）:

```bash
sudo nvpmodel -m 0
sudo jetson_clocks
```

同时运行 `tegrastats` 或 `jtop` 查看 CPU/GPU/DLA/内存/温度。对比不同分支或算法时，必须固定输入、配置和性能模式。

完整性能记录格式见 [性能测试](性能测试.md)。
