# 部署与运行

## NX 环境

目标平台:

- Jetson Orin NX Super 16GB。
- JetPack 6.x。
- CUDA、TensorRT、VPI、OpenCV、yaml-cpp。
- 海康 MVS SDK。
- libgpiod。

具体版本以当前 NX 实机为准。性能测试前必须记录 JetPack、CUDA、TensorRT、VPI、模型 engine、配置文件和 git commit。

## 构建

```bash
cd ~/NX_volleyball/stereo_3d_pipeline
cmake -S . -B build_standalone -DCMAKE_BUILD_TYPE=Release -DCUDA_ARCH=87 \
  -DCMAKE_DISABLE_FIND_PACKAGE_ament_cmake=ON \
  -DCMAKE_CUDA_COMPILER=/usr/local/cuda-12.6/bin/nvcc \
  -DCUDAToolkit_ROOT=/usr/local/cuda-12.6
cmake --build build_standalone -j$(nproc)
```

构建产物、条件依赖和 CMake target 见 [构建目标与依赖](构建目标与依赖.md)。

## 运行

```bash
cd ~/NX_volleyball/stereo_3d_pipeline/build_standalone
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

主配置字段见 [配置参考](配置参考.md)，配置文件用途见 [配置矩阵](配置矩阵.md)。

显示窗口:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml --visualize
```

## 录制基准片段

用于离线特征点、双目测距和彩色 patch 算法对比时，直接录左右校正图和检测框 CSV:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 2.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

只需要连续左右目图像、不等待检测框触发时，加 `--baseline-start-immediately`。录制默认先缓存到内存，clip 完成后再写盘；详细字段、输出结构和检查命令见 [录制与离线评估](录制与离线评估.md)。

## 性能准备

测试前建议:

```bash
sudo nvpmodel -m 0
sudo jetson_clocks
```

同时运行 `tegrastats` 或 `jtop` 查看 CPU/GPU/DLA/内存/温度。对比不同分支或算法时，必须固定输入、配置和性能模式。

完整性能记录格式见 [性能测试](性能测试.md)。
