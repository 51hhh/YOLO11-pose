最后核对: 2026-07-04

本页是深度相关 Wiki 的总入口。先看这里判断应该读哪一类页面，避免把离线探针、实时矩阵、当前生产选用和训练数据口径混在一起。

## 当前结论

当前冻结的训练采集字段是 P0 + P1-promoted，`config/pipeline_dual_yolo_roi.yaml` 和 `config/pipeline_record_p0p1.yaml` 已对齐:

| 分层 | 字段 | 用途 |
|---|---|---|
| P0 稳定候选 | `z_bbox_center`, `z_circle_center`, `z_roi_edge_centroid`, `z_roi_radial_center`, `z_roi_edge_pair_center` | 实时稳定候选和训练主字段 |
| P1 主 CSV 训练候选 | `z_roi_multi_point`, `z_roi_center_patch`, `z_roi_patch_iou_color_edge`, `z_roi_iou_region_color_patch`, `z_roi_cuda_stereo_sgm` | 只记录原始字段，用于学习偏置和可靠性 |
| P1 diagnostic sidecar 候选 | `z_roi_vpi_template_match`, `z_roi_vpi_orb` | 由 `*.p2_diagnostic.csv` 按 `frame_id + mode` 合并，不回写主 `Object3D` |

`z_stereo` / `z` 是 legacy 兼容输出，不是训练候选。当前 C++ 只允许 P0 几何/bbox 和退化 fallback 写入 legacy `z_stereo/obj.z`，P1/实验候选不会抢占在线主深度。

P2 已完成 2026-07-04 08:30 NX isolated 全量矩阵、10:49 targeted 复核、10:53 artifact debug 和 11:08 final artifact，并按推荐顺序合并到 [稳定100fps深度方法与训练入口](稳定100fps深度方法与训练入口.md) 的“P2 推荐排序总表”。用户选定的排序 `1/2/5/7/8` 已提升到主管线 P1 候选；其中 1/2/5 写主 CSV，7/8 写 diagnostic sidecar。其余 OpenCV CUDA、VPI、Fixstars libSGM、TensorRT 神经特征均已跑真实后端或真实 engine case，但未达到默认实时准入。状态抽样图已复制到 `wiki/assets/p2_20260704/` 并在总表内联显示；这些图是状态/检测图，不是算法级左右特征点连线图。真实算法级样张已复制到 `wiki/assets/p2_20260704_artifacts/` 和 `wiki/assets/p2_20260704_final/`，当前覆盖 OpenCV CUDA ORB/Template/SGM/GFTT-LK、CUDA Hough circle、VPI Template 和 VPI ORB，详见 [P2算法效果与可视化审查](P2算法效果与可视化审查.md)。CUDA-SIFT 本轮不做，只保留 unsupported 入口。

## 判断规则

- 实时是否可用，只看 NX 实时矩阵和连续片段，不用离线单帧结果替代。
- 是否进入默认配置，只看 [稳定100fps深度方法与训练入口](稳定100fps深度方法与训练入口.md)。
- 训练输入只读原始候选字段，不读 `z_stereo` / `z` / `depth_method` / `stereo_depth_source` 作为方法输入。
- 新增候选必须先进入 P2 isolated A/B，不直接叠加到当前 P0+P1-promoted 采集集合。
