## 当前结论

当前冻结的 100fps 训练采集字段以 P0 + P1 基线为准，GFTT/LK 低频写 diagnostic sidecar，XFeat TensorRT 已接入但默认关闭。若要做 XFeat A/B，必须同时打开 `neural_feature_matching.enabled` 和 `p2_realtime_lane_decision_enabled`:

| 分层 | 字段 | 用途 |
|---|---|---|
| P0 稳定候选 | `z_bbox_center`, `z_circle_center`, `z_roi_edge_centroid`, `z_roi_radial_center`, `z_roi_edge_pair_center` | 实时稳定候选和训练主字段 |
| P1 主 CSV 训练候选 | `z_roi_multi_point`, `z_roi_center_patch` | 只记录原始字段，用于学习偏置和可靠性 |
| P1 diagnostic sidecar 候选 | `z_roi_opencv_cuda_gftt_lk` | 由 `*.p2_diagnostic.csv` 按 `frame_id + mode=opencv_cuda_gftt_lk` 合并，不回写主 `Object3D` |
| 测试态神经候选 | `z_roi_neural_feature` | XFeat TensorRT 已接入，但当前组合只有 `93-95fps`；正式 100fps 默认关闭 neural 和 P2 realtime lane |

`z_stereo` / `z` 是 legacy 兼容输出，不是训练候选。当前 C++ 只允许 P0 几何/bbox 和退化 fallback 写入 legacy `z_stereo/obj.z`，P1/实验候选不会抢占在线主深度。

## 判断规则

- 实时是否可用，只看 NX 实时矩阵和连续片段，不用离线单帧结果替代。
- 是否进入默认配置，只看 [稳定100fps深度方法与训练入口](稳定100fps深度方法与训练入口.md)。
- 训练输入只读原始候选字段，不读 `z_stereo` / `z` / `depth_method` / `stereo_depth_source` 作为方法输入。
- 新增候选必须先进入 P2 isolated A/B，不直接叠加到当前 P0+P1 基线采集集合。
