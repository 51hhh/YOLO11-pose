## `stereo_pipeline` 常用模式

实时运行:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

显示窗口:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml --visualize
```

录制基准片段:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 3.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

连续采集但不等待检测框:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 2.0 \
  --baseline-image-mode bgr \
  --baseline-format png \
  --baseline-start-immediately
```

单帧特征匹配可视化:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --debug-feature-matches \
  --debug-feature-matches-dir debug_feature_matches
```

## 离线验证脚本

| 脚本 | 用途 | 关键参数 |
|---|---|---|
| `tools/offline_volleyball_keypoint_probe.py` | 单对左右图传统关键点/IoU/patch 深度诊断 | `--left`, `--right`, `--calib`, `--out`, validation thresholds |
| `tools/neural_feature_probe.py` | XFeat/ALIKED/SuperPoint+LightGlue 离线试验 | `--backends`, `--device`, `--top-k`, `--roi-size`, gate 参数 |
| `tools/compare_offline_online_depth.py` | 对比离线和在线 CSV 深度结果 | `--offline-summary`, `--online-csv`, `--fail-on-mismatch` |
| `trajectory_fusion/evaluate_fusion.py` | 轨迹融合模型评估 | 以该脚本 help 为准 |
| `trajectory_fusion/train_reliability.py` | 可靠性/融合训练 | 以该脚本 help 为准 |
| `scripts/export_xfeat_extractor_onnx.py` | XFeat extractor-only ONNX 导出 | `--xfeat-repo`, `--weights`, `--out`, `--roi-size`, `--opset` |
| `scripts/nx_build_xfeat_extractor_engine.sh` | NX 上构建 XFeat extractor TensorRT engine | `ROOT_DIR`, `XFEAT_REPO`, `OUT_DIR`, `ROI_SIZE`, `OPSET`, `TRT_FP16` 环境变量 |

```bash

  --out test_logs/offline_keypoint_probe_latest
```

```bash
python3 tools/neural_feature_probe.py \
  --backends xfeat,aliked,superpoint_lightglue \
  --device cpu \
  --out test_logs/neural_feature_probe_latest
```


## NX 矩阵测试

脚本:

```text
scripts/nx_algorithm_matrix_test.py
```

它会从 `pipeline_dual_yolo_roi.yaml` 生成临时配置，并轮流测试:

- 默认几何候选。
- ROI center patch。
- ROI subpixel。
- corner/texture/binary sparse。
- ORB/BRISK/AKAZE/SIFT。
- 彩色 IoU/edge patch。
- neural XFeat/ALIKED/SuperPoint+LightGlue。

输出:

```text
summary.csv
report.md
static_findings.txt
*.log
```

注意: 矩阵测试只能说明“能运行”和粗略耗时，最终启用还需要连续帧质量指标。

## 维护规则

- 新工具必须有 `--help` 或文件头用法。
- 产生测试输出的脚本默认写入 `test_logs/` 或显式 `--out`。
- 实机脚本要写明是否会占用相机、启动 PWM、写盘或发布 ROS2 话题。
