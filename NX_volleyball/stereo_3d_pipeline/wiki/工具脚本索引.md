## `stereo_pipeline` 常用模式

实时运行:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

显示窗口:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml --visualize
```

录制基准片段:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 3.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

连续采集但不等待检测框:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 2.0 \
  --baseline-image-mode bgr \
  --baseline-format png \
  --baseline-start-immediately
```

单帧 legacy CPU 特征匹配可视化:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --debug-feature-matches \
  --debug-feature-matches-dir debug_feature_matches
```

## NX 矩阵测试

脚本:

```text
scripts/nx_algorithm_matrix_test.py
```

它会从 `pipeline_dual_yolo_roi.yaml` 生成临时配置，并逐个 case 单独测试。每个 case 先关闭全部 depth mode、subpixel、fallback 和 neural，再只打开目标候选:

- 几何候选: `bbox_pair`、`circle_center`、`roi_edge_centroid`、`roi_radial_center`、`roi_edge_pair_center`。
- ROI patch: `roi_center_patch`、`roi_subpixel`。
- 真实 GPU/OpenCV 路径: OpenCV CUDA ORB、自研 CUDA Template/NCC、OpenCV CUDA StereoBM/StereoSGM、OpenCV CPU BRISK/AKAZE/SIFT debug 对照。OpenCV CUDA TemplateMatching 仅作为 `STEREO_CUDA_TEMPLATE_BACKEND=opencv` baseline。
- 彩色 IoU/edge patch 及其 P2 参数 sweep。
- neural XFeat/ALIKED/SuperPoint+LightGlue 及其 ROI/top-k engine sweep。
- VPI/libSGM diagnostic-only: VPI Template/Stereo/Harris-LK/ORB、OpenCV CUDA GFTT-LK/Hough、Fixstars libSGM availability/实测 case。

`scripts/nx_build_libsgm.sh` 用于在 NX 上下载并构建 Fixstars libSGM 到 `third_party/libSGM/install`；主工程 CMake 会自动搜索该 install prefix，也可用 `-DLIBSGM_ROOT=...` 指定其他安装路径。

`corner/texture/binary` sparse-lite 是诊断近似实现，默认不跑；需要时追加 `--include-approx`，不能把这些结果写成真实 ORB/BRISK/AKAZE/SIFT。

常用过滤:

```bash
python3 scripts/nx_algorithm_matrix_test.py \
  --out test_logs/nx_algorithm_matrix_targeted \
  --duration-sec 8 \
  --cases opencv_cuda_orb,neural_xfeat
```

失败项 relaxed 诊断:

```bash
python3 scripts/nx_algorithm_matrix_test.py \
  --out test_logs/nx_algorithm_matrix_relaxed \
  --duration-sec 8 \
  --include-experimental \
  --cases opencv_cuda_orb_relaxed,neural_xfeat_relaxed
```

输出:

```text
summary.csv
report.md
static_findings.txt
*.log
```

`report.md` 会显示 `algo_stage`、算法 avg/max、async worker avg/max、deadline/drop 诊断和候选有效率。失败后需要抓图时追加 `--debug-on-failure`，脚本会额外生成 `debug/<case>/feature_matches/` 和 `debug/<case>/realtime_zoom/`；该模式只用于诊断，不计入准入测速。

注意: `debug/<case>/feature_matches/` 来自 `--debug-feature-matches` 的 legacy CPU debug 路径，不是 P2 GPU/VPI/TRT/libSGM 后端的算法级匹配图。`debug/<case>/realtime_zoom/` 是实时状态图，只画 bbox、circle 和 `z_*` 字段。

注意: 矩阵测试只能说明“能运行”和粗略耗时，最终启用还需要连续帧质量指标。正式矩阵不要使用旧的 `all_sparse_gpu` 混跑方式；`--include-experimental` 只用于定位 gate/匹配失败原因，不是生产配置。P2 失败和超时排查见 [P2算法NX测试排查指南](P2算法NX测试排查指南.md)。

神经特征 case 需要先准备对应 TensorRT engine。当前 `neural_xfeat` case 期望 `xfeat_extractor_128.engine`，`scripts/nx_build_xfeat_extractor_engine.sh` 默认就生成该尺寸。SuperPoint case 期望 `superpoint_extractor_224_top128.engine`，可由 `BACKENDS=superpoint scripts/nx_build_lightglue_extractor_engines.sh` 生成。官方 ALIKED-t16 DCN 使用 `scripts/export_aliked_dcn_extractor_onnx.py`、`scripts/nx_build_dcnv2_plugin.sh` 和 `scripts/nx_build_aliked_dcn_engine.sh` 生成 `aliked_t16_dcn_extractor_128_top64_b2.engine`；矩阵脚本的当前 ALIKED case 会同时配置 `plugins/libdcnv2.so`。

## 维护规则

- 新工具必须有 `--help` 或文件头用法。
- 产生测试输出的脚本默认写入 `test_logs/` 或显式 `--out`。
- 实机脚本要写明是否会占用相机、启动 PWM、写盘或发布 ROS2 话题。
