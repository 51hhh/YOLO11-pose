# 性能测试

## 记录格式

任何性能结论必须包含:

- 日期。
- git commit。
- NX 型号和 JetPack 版本。
- 配置文件路径。
- 模型 engine 路径和精度。
- 运行命令。
- 输入数据来源。
- 是否启用显示、记录、baseline clip、debug 输出。
- 平均值、P95/P99 或最大值，而不是只写单个最好值。

## 必测项

| 项目 | 目标 |
|---|---|
| PWM/水印 | 触发频率稳定，左右 frame counter delta 为 0 |
| 取流 | USB 到达抖动不导致长期积压 |
| 检测 | YOLO 推理不阻塞采集 |
| ROI 后处理 | 落后时可丢旧帧 |
| 写盘 | CSV/图片队列限流，不无限增长 |
| GPU/CPU 拷贝 | 不在主路径出现高频无意义往返 |
| 整体 fps | 接近 100Hz 触发频率 |

水印字段解释和独立探针见 [相机同步与水印](相机同步与水印.md)。实时 profile 名称和 Stage 级耗时解释见 [实时调度细节](实时调度细节.md)。

## 推荐命令

系统状态:

```bash
tegrastats --interval 1000
```

TensorRT engine:

```bash
trtexec --loadEngine=/home/nvidia/NX_volleyball/model/yolo26_gpu_fp16.engine
```

全管线:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

基准录制:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-duration 3.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

基准录制默认先缓存到内存再写盘。评估实时影响时重点看 `Stage2_FrameCB`，写盘时间发生在 clip 捕获完成后，不应计入捕获阶段的帧间耗时。

更多工具和脚本见 [工具脚本索引](工具脚本索引.md)。算法矩阵测试配置来源见 [配置矩阵](配置矩阵.md)。

分支合入前的实机复测步骤见 [实机测试清单](实机测试清单.md)。

## 回归标准

合入实时路径改动前至少确认:

- 无编译错误。
- 默认配置能启动。
- 100Hz 触发下水印稳定。
- 启用记录不会明显拖慢主循环。
- 打开新增算法时有独立耗时统计。
- 关闭新增算法时性能回到基线。
