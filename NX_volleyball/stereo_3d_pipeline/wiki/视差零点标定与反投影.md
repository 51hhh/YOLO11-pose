# 视差零点标定与反投影

最后核对: 2026-07-10

轨迹模型阶段1预处理。用米尺实测的 known_z 静态段拟合视差零点偏移 `d0`，再把每路深度候选反投影成米制 3D 点，供逐帧因果 3D 轨迹状态估计器使用。

## 当前有效结论

海康双目 `stereo_3d_pipeline` 的深度模型 `z = fB / disparity` 缺一个视差零点修正项。真实模型是:

```text
z = fB / (disparity - d0)
```

当前运行配置使用 2026-07-09 `test_logs/p1_dy_regression_20260709_*` 的静态米尺段、配合当前 `calibration/stereo_calib.yaml` 拟合出的 **runtime d0**。这批 CSV 是 raw 录制，未启用 d0；因此可以用于重新拟合。

| 参数 | 拟合值 | 说明 |
|---|---|---|
| `calib fB` | 1497.079 | 取当前 OpenCV 双目标定 `projection_left.fx × baseline_m` |
| `d0` | -5.324 px | 运行时使用的视差零点偏移，写入 `config/disparity_offset_fit_20260709.json` |
| known_z RMS | 0.049 m | raw `z=fB/disparity` 为 0.292 m |
| mean \|err\| | 0.043 m | raw 为 0.224 m |
| max \|err\| | 0.099 m | 出现在 9m 桶 |

拟合口径:

- 前 30 段静态按目录时间顺序每 3 段一桶。虽然当前数据目录临时记录为 3–13m，但最后三段 raw Z 中位是 12.28–12.87m；若按 13m 拟合，13m 桶残差会到 -0.67m，整体 RMS 约 0.27m，不自洽。因此 d0 拟合暂按 3–12m 十个距离桶计算，最后一桶标签需要回查现场记录/metadata。
- C++ 实时管线只使用 `d0`，`fB` 仍来自 `calibration/stereo_calib.yaml`，所以本轮 d0 是在固定当前标定 fB 的前提下拟合，不使用二参数 `(fB,d0)` 共同拟合作为运行尺度。

关键判断:

- **偏差不是数据/摆位/算法错**。`implied_fB = z_bbox × disparity` 与当前标定 fB 同量级，说明 `z = fB/disparity` 深度链路自洽、无 bug；距离相关误差主要来自缺失的 `d0` 项和远距小视差放大。
- **各路候选共享 d0**。d0 是整流属性，bbox/circle/几何/NCC/XFeat 各方法一致偏移(动态 11m 抛球段验证:加 d0 后各路 Z 一致下移约 1.0m)。
- **近距 d0 影响小、远距被 z² 放大**。本轮 raw bias 从 3m 约 -0.7cm 增长到 12m 约 +52cm；加 d0 后 3–12m 聚合残差压到约 ±10cm。因此动态远距数据同样必须做 d0 修正。

## d0 必须是单个常数,不做非线性(重要)

`z = fB/(disparity - d0)` 是校正后双目的物理正确模型，`d0` 是整流后视差零点这**一个物理常数**。它在 `1/z` 上线性，一次最小二乘拟合就是解析解，不需要迭代，也**不应该**加分段或二次项。

- 给 d0 加分段/二次项 = 过拟合。那是在拟合特定批次在 9m/12m 处的测量噪声，不是拟合物理。换一批数据、换个距离桶就失效。
- d0 不是直接来自标定的左右主点差；它吸收的是当前整流、检测锚点和 subpixel 视差估计在整个量程上的系统性偏移。换标定文件或换录制口径后必须重算，不能沿用旧值。

历史记录: 2026-07-07 旧标定/旧静态集曾拟合出二参数 `fB=1500.845, d0=-13.325px`，RMS 0.125m。该值不再作为当前运行配置使用。

当前 d0 已经把静态尺度误差压到厘米级。剩余 9m 附近的 +10cm 和个别坏段/标签问题，交给模型质量特征、manifest 剔除和后续外部真值复核处理，不继续用分段 d0 或非线性项硬拟合。

分工原则:

```text
确定性物理偏差(全局尺度)   -> d0 单常数,已正确
距离相关残余 bias(9m/12m)  -> 模型学,不碰 d0
随机噪声(帧间抖动)         -> 模型时序滤波去抖
```

**当前配置结论**: `pipeline_dual_yolo_roi.yaml` 启用 d0，用于实际运行/ROS/可视化；`pipeline_record_p0p1.yaml` 关闭 d0，用于多距离采集和复标。训练侧如果读取 raw 录制，仍从 `disparity_*` 经 `reproject.py` 或同一 d0 文件转换，不直接把 raw `z_*` 当最终米制真值。

## 适用范围与一致性约束

- 训练/离线：轨迹模型阶段1，把 CSV 候选深度反投影成米制 XYZ 前必须先修 d0。**用于 d0 拟合/多距离训练采集的 CSV 必须保持 `z=fB/disparity` 原始口径**，训练加载路径再经 `reproject.py` 转换。
- 实时 C++ 主路径：已支持 `disparity_offset` 配置块。`config/pipeline_dual_yolo_roi.yaml` 实际运行启用 d0；`config/pipeline_record_p0p1.yaml` 多距离录制显式关闭 d0，保留原始 `z_*` 供复标。
- **训练与录制端必须口径一致**。如果录制配置关闭 d0，训练侧必须用同一个 `disparity_offset_fit.json` 反投影；如果某次运行配置开启 d0，其 CSV 的 `z_*` 已是修正值，不得再用同一个 d0 二次修正。
- **换相机/重新标定后 d0 会变**,必须用新数据重跑 `calibrate_disparity_offset.py` 重新拟合,旧 `disparity_offset_fit.json` 作废。

## 反投影公式

每路深度候选共享左图球心像素 `(u, v)`，用各自的 disparity 反投影:

```text
Z = fB / (disparity - d0)
X = (u - cx) * Z / fx
Y = (v - cy) * Z / fy
```

因为像素锚点共享，某路 disparity 抖动会产生成比例的 Z 抖动，进而产生成比例的 X/Y 抖动。这让"XY 抖动源于 Z 抖动"的耦合对模型显式可见，是喂给状态估计器的关键结构。

内参(取自 `calibration/stereo_calib.yaml` 的 `projection_left`):

```text
fx = fy = 1608.21    cx = 653.43    cy = 617.35    baseline = 925.73 mm
```

像素锚点约定:circle 类方法(`circle_center`, `roi_radial_center`, `roi_edge_pair_center`)用 `left_circle_cx/cy`，其余用 `left_bbox_cx/cy`；disparity 用各方法自己的 `disparity_<method>` 列。

## 命令

拟合视差零点偏移(数据驱动，不要硬编码 d0):

```bash
cd NX_volleyball/stereo_3d_pipeline
../../.venv-stereo-neural/bin/python trajectory_fusion/calibrate_disparity_offset.py \
  test_logs \
  --calib ../calibration/stereo_calib.yaml \
  -o config/disparity_offset_fit_YYYYMMDD.json
```

注意: 当前 `test_logs/p1_dy_regression_20260709_*` 的 metadata 仍是 `known_z: null`，不能直接喂给上面的通用脚本；本轮 d0 是按数据目录/现场顺序临时分桶后计算。正式复标前应先把 metadata 写入真实 `known_z/static`。

反投影单段抽查:

```bash
../../.venv-stereo-neural/bin/python trajectory_fusion/reproject.py \
  test_logs/p1_dy_regression_20260709_135244/traj.csv \
  --calib ../calibration/stereo_calib.yaml \
  --offset-fit config/disparity_offset_fit_20260709.json
```

实时运行启用:

```yaml
disparity_offset:
  enabled: true
  file: "config/disparity_offset_fit_20260709.json"
```

多距离 d0/训练采集关闭:

```yaml
disparity_offset:
  enabled: false
  file: "config/disparity_offset_fit_20260709.json"
```

远距噪声段可用 `--exclude-known-z 12` 从拟合中剔除，避免最远点主导。

## 相关源码入口

- `trajectory_fusion/calibrate_disparity_offset.py`：拟合 `(fB, d0)`，线性最小二乘(`disp = fB·(1/z) + d0`)。
- `trajectory_fusion/reproject.py`：`load_reprojection_model()` 从标定 YAML + 拟合 JSON 构建 `ReprojectionModel`；`reproject_row()` 把一帧各候选反投影为 `ReprojectedPoint`。
- `trajectory_fusion/backfill_known_z.py`：从本页数据集目录回填 known_z 静态标签(拟合的输入)。
- `src/main_config_loaders.cpp` / `src/pipeline/pipeline_config.h`：读取 `disparity_offset.enabled/file/d0`。
- `src/pipeline/*`、`src/stereo/*`、`src/fusion/coordinate_3d.*`：实时候选深度统一使用 active d0；关闭时退回 `d0=0`。

## 验证方法

- `calibrate_disparity_offset.py` 或临时分桶脚本报告每个距离的 `z_pred - known_z` 残差，当前 3–12m 聚合应在约 ±0.10m 内。
- `reproject.py` 抽查:8m 段 raw `z_bbox≈8.18` 修正后 `Z≈7.95`(米尺真值 8.0，误差从 0.18m 降到 0.05m)。
- 用在线 `x` 反推像素 `u=(x·fx/z)+cx` 应与 CSV `left_bbox_cx` 吻合(验证内参用对，且反投影是在线逻辑的去 bias 版本)。
