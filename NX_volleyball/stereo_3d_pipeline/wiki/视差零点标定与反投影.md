# 视差零点标定与反投影

最后核对: 2026-07-08

轨迹模型阶段1预处理。用米尺实测的 known_z 静态段拟合视差零点偏移 `d0`，再把每路深度候选反投影成米制 3D 点，供逐帧因果 3D 轨迹状态估计器使用。

## 当前有效结论

海康双目 `stereo_3d_pipeline` 的深度模型 `z = fB / disparity` 缺一个视差零点修正项。真实模型是:

```text
z = fB / (disparity - d0)
```

从 2026-07-07 采集的 3–12m 米尺实测静态段拟合(26 段、10 个距离):

| 参数 | 拟合值 | 说明 |
|---|---|---|
| `fB` | 1500.845 | 与标定理论值 fx×baseline=1488.77 相差 0.8% |
| `d0` | -13.325 px | 视差零点偏移(整流后左右主点残差) |
| known_z RMS | 0.125 m | 修正前为 0.643 m |
| max \|err\| | 0.370 m | 出现在 12m(最远、视差最小、量化误差最大) |

关键判断:

- **偏差不是数据/摆位/算法错**。`implied_fB = z_bbox × disparity` 在所有距离段恒为 ~1488.8，说明 `z = fB/disparity` 深度链路自洽、无 bug；fB 拟合值与标定理论值几乎一致，说明焦距/基线标定准确。偏差全部来自缺失的 `d0` 项。
- **各路候选共享 d0**。d0 是整流属性，bbox/circle/几何/NCC/XFeat 各方法一致偏移(动态 11m 抛球段验证:加 d0 后各路 Z 一致下移约 1.0m)。
- **近距 d0 影响小、远距被 z² 放大**。3m 处修正量仅 ~3cm，8m 处 ~0.55m，11m 处 ~1.0m。因此动态远距数据同样必须做 d0 修正。

## d0 必须是单个常数,不做非线性(重要)

`z = fB/(disparity - d0)` 是校正后双目的物理正确模型，`d0` 是整流后视差零点这**一个物理常数**。它在 `1/z` 上线性，一次最小二乘拟合就是解析解，不需要迭代，也**不应该**加分段或二次项。

- 给 d0 加分段/二次项 = 过拟合。那是在拟合特定批次在 9m/12m 处的测量噪声，不是拟合物理。换一批数据、换个距离桶就失效。
- d0=−13.3px 并非来自标定的左右主点差(projection_left/right 的 cx 都是 653.43，几何上 d0 应接近 0)。它实际吸收的是 **subpixel 视差估计在整个量程上的系统性偏移**，是一个经验常数——单值是最诚实的建模。

**逐距离深度校验(2026-07-08，9 字段 × 10 距离，反投影后 vs known_z):**

| 区间 | 表现 | 归属 |
|---|---|---|
| 3–8m | 最佳字段 ±8cm 以内，d0 几乎完美 | 确定性尺度，d0 已解决 |
| 9m | 全字段一致偏低 0.16–0.24m，MAD 小 | 距离相关残差 → **模型学** |
| 12m | 全字段一致偏低 0.40–0.50m，MAD 极小(0.008m) | 视差量化 + d0 外推极限 → **模型学** |

9m/12m 的残差特征是"同一距离下所有字段一致偏移、跨方法一致、帧间稳定"——这**正是模型能从 disparity/median_z/support 特征里学到的距离相关 bias**，是 KalmanNet 学习增益/bias 分支的职责，不该硬塞进 d0。

分工原则:

```text
确定性物理偏差(全局尺度)   -> d0 单常数,已正确
距离相关残余 bias(9m/12m)  -> 模型学,不碰 d0
随机噪声(帧间抖动)         -> 模型时序滤波去抖
```

**单路候选平均 |bias| 排名**(3–12m 全程，越小越准): `bbox_center` 0.109m 最优且 MAD 最小，`roi_multi_point` 0.155m 最差(且静态段覆盖率极低，应靠 valid/trust 自动降权)。注意 bbox_center 的 0.109m 被 9m/12m 拉高，在 3–8m 主力区间它是厘米级。

## 适用范围与一致性约束

- 训练/离线：轨迹模型阶段1，把 CSV 候选深度反投影成米制 XYZ 前必须先修 d0。**用于 d0 拟合/多距离训练采集的 CSV 必须保持 `z=fB/disparity` 原始口径**，训练加载路径再经 `reproject.py` 转换。
- 实时 C++ 主路径：已支持 `disparity_offset` 配置块。`config/pipeline_dual_yolo_roi.yaml` 实际运行启用 d0；`config/pipeline_record_p0p1.yaml` 多距离录制显式关闭 d0，保留原始 `z_*` 供复标。
- **训练与录制端必须口径一致**。如果录制配置关闭 d0，训练侧必须用同一个 `disparity_offset_fit.json` 反投影；如果某次运行配置开启 d0，其 CSV 的 `z_*` 已是修正值，不得再用同一个 d0 二次修正。
- **换相机/重新标定后 d0 会变**,必须用新数据重跑 `calibrate_disparity_offset.py` 重新拟合,旧 `disparity_offset_fit.json` 作废。

## 反投影公式

每路深度候选共享左图球心像素 `(u, v)`，用各自的 disparity 反投影:

```text
Z = fB / (disparity - d0)
X = (u - cx) * Z / fx
Y = (v - cy) * Z / fy
```

因为像素锚点共享，某路 disparity 抖动会产生成比例的 Z 抖动，进而产生成比例的 X/Y 抖动。这让"XY 抖动源于 Z 抖动"的耦合对模型显式可见，是喂给状态估计器的关键结构。

内参(取自 `calibration/stereo_calib.yaml` 的 `projection_left`):

```text
fx = fy = 1608.21    cx = 653.43    cy = 617.35    baseline = 925.73 mm
```

像素锚点约定:circle 类方法(`circle_center`, `roi_radial_center`, `roi_edge_pair_center`)用 `left_circle_cx/cy`，其余用 `left_bbox_cx/cy`；disparity 用各方法自己的 `disparity_<method>` 列。

## 命令

拟合视差零点偏移(数据驱动，不要硬编码 d0):

```bash
cd NX_volleyball/stereo_3d_pipeline
../../.venv-stereo-neural/bin/python trajectory_fusion/calibrate_disparity_offset.py \
  test_logs/recording_runs_20260707 \
  --calib ../calibration/stereo_calib.yaml \
  -o test_logs/recording_runs_20260707/disparity_offset_fit.json
```

反投影单段抽查:

```bash
../../.venv-stereo-neural/bin/python trajectory_fusion/reproject.py \
  test_logs/recording_runs_20260707/p1_dy_regression_20260707_180406/traj.csv \
  --calib ../calibration/stereo_calib.yaml \
  --offset-fit test_logs/recording_runs_20260707/disparity_offset_fit.json
```

实时运行启用:

```yaml
disparity_offset:
  enabled: true
  file: "test_logs/recording_runs_20260707/disparity_offset_fit.json"
```

多距离 d0/训练采集关闭:

```yaml
disparity_offset:
  enabled: false
  file: "test_logs/recording_runs_20260707/disparity_offset_fit.json"
```

远距噪声段可用 `--exclude-known-z 12` 从拟合中剔除，避免最远点主导。

## 相关源码入口

- `trajectory_fusion/calibrate_disparity_offset.py`：拟合 `(fB, d0)`，线性最小二乘(`disp = fB·(1/z) + d0`)。
- `trajectory_fusion/reproject.py`：`load_reprojection_model()` 从标定 YAML + 拟合 JSON 构建 `ReprojectionModel`；`reproject_row()` 把一帧各候选反投影为 `ReprojectedPoint`。
- `trajectory_fusion/backfill_known_z.py`：从本页数据集目录回填 known_z 静态标签(拟合的输入)。
- `src/main_config_loaders.cpp` / `src/pipeline/pipeline_config.h`：读取 `disparity_offset.enabled/file/d0`。
- `src/pipeline/*`、`src/stereo/*`、`src/fusion/coordinate_3d.*`：实时候选深度统一使用 active d0；关闭时退回 `d0=0`。

## 验证方法

- `calibrate_disparity_offset.py` 报告每个距离的 `z_pred - known_z` 残差，3–11m 应在 ±0.1m 内。
- `reproject.py` 抽查:8m 段 raw `z_*≈8.63` 修正后 `Z≈8.08`(米尺真值 8.0，误差从 0.63m 降到 0.08m)。
- 用在线 `x` 反推像素 `u=(x·fx/z)+cx` 应与 CSV `left_bbox_cx` 吻合(验证内参用对，且反投影是在线逻辑的去 bias 版本)。
