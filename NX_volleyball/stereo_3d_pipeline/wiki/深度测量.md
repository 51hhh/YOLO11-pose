## 数据流

```text
左右 YOLO bbox pair
  -> ROI 几何 / patch / 特征 / fallback 多候选深度
  -> buildDepthCandidateObservations() 写完整候选列表
  -> selectLegacyDepthOutputCandidate() 写 legacy z_stereo / obj.z
  -> Object3D / TrajectoryRecorder 写出所有原始 z_* 字段
  -> 轨迹可靠性模型直接读取原始候选字段
```

关键边界:

- `z_stereo` / `z` 是兼容输出，不是候选方法。
- Legacy `z_stereo/obj.z` 只允许 P0 几何/bbox 和退化 fallback；P1/实验候选只写原始字段。
- 训练可靠性模型读取原始 `z_*`、质量字段、pair gate 和水印字段，不把 legacy 输出当标签。

| 圆/球心几何 | `z_circle_center`, `z_circle_left_edge`, `z_circle_right_edge` | P0 主观测和边缘诊断 |

| sparse/描述子 | `z_roi_orb_points`, `z_roi_brisk_points`, `z_roi_akaze_points`, `z_roi_sift_points` | A/B 或 debug，不进入当前稳定集合 |

## 质量门

特征或 patch 深度进入候选前必须通过几何和鲁棒性检查:

1. 左右点必须落在 projected overlap mask 内。
2. 校正后 y 残差必须小；如存在系统倾斜，先显式建模 `feature_y_slope` / `feature_y_offset_px`。
3. 视差必须为正，且在 `max_disparity` 内。
4. 左右反查或互检一致。
5. 视差聚合使用 weighted median / MAD / 1D RANSAC，不能简单平均。
6. 三角化点应接近排球物理半径，明显偏离球体表面的点丢弃。

## 实时与离线边界

- 实时可用性只看 NX 连续矩阵和连续片段，不用离线单帧结果替代。
- 离线 probe 用于定位 ROI、匹配点、gate 失败原因和数值偏差。
- 新算法先进入 P2 isolated A/B，单项记录 fps、有效率、中位深度、MAD、worker 耗时和长尾。
- 当前训练采集字段冻结为 P0+P1；新候选仍先进入 P2 isolated A/B，通过联合测试和字段合并后再提升为 P1。
