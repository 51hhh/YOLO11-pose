# 深度测量

## 多候选深度方法

系统包含多类深度候选，目标是让排球在不同距离、亮度、纹理和遮挡条件下仍有可用观测。

| 类别 | 方法 |
|---|---|
| 单目 | `z_mono` |
| Box 双目 | `z_bbox_center`, `z_bbox_left_edge`, `z_bbox_right_edge` |
| 圆拟合双目 | `z_circle_center`, `z_circle_left_edge`, `z_circle_right_edge` |
| ROI 亚像素/特征 | `z_roi_edge_centroid`, `z_roi_radial_center`, `z_roi_edge_pair_center`, `z_roi_corner_points`, `z_roi_texture_points`, `z_roi_binary_points`, `z_roi_center_patch`, `z_roi_multi_point` |
| OpenCV 描述子 | `z_roi_orb_points`, `z_roi_brisk_points`, `z_roi_akaze_points`, `z_roi_sift_points` |
| 彩色区域 | `z_roi_iou_region_color_patch`, `z_roi_patch_iou_color_edge` |
| 神经特征 | `z_roi_neural_feature` |
| Fallback | `z_fallback`, `z_fallback_template`, `z_fallback_feature_points` |

具体 schema 以 `trajectory_fusion/schema.md` 为准。若新增字段，必须同步更新 schema、记录器、离线 dataset 和本页。

字段分组、单位和 `stereo_depth_source` 映射见 [深度候选字段参考](深度候选字段参考.md)。特征点提取、匹配和误匹配剔除实现见 [特征匹配实现](特征匹配实现.md)。

CSV 中的 `z_yolo_bbox_pair`、`z_circle`、`z_subpixel` 等为旧兼容别名，不计入新的深度方法数量。

实时能力边界:

- ORB 当前是 true OpenCV CUDA ORB，但 keypoint/match 元数据和几何 gate 仍回 CPU，且 NX 同步实测不守 100fps。
- BRISK/AKAZE/SIFT 当前没有 true CUDA 后端；启用时走 OpenCV CPU，需要 pinned host gray，只适合 A/B 或 debug。
- 彩色 IoU/edge patch 需要校正 BGR GPU；灰度录制或 gray-only 管线不能验证真实颜色语义。
- 神经特征必须看 TensorRT engine schema；1/2 通道可走 gray，3/6 通道必须有 BGR 管线，extractor-only/direct/split 路径仍有 CPU 后处理。

## 生产策略

实时默认优先轻量候选:

- bbox center/edge。
- circle center/edge。
- ROI edge centroid。
- ROI radial center。
- ROI edge pair center。
- epipolar fallback。

较重方法用于离线 A/B 和专项 profile:

- ORB/BRISK/AKAZE/SIFT。
- ZNCC 多点 subpixel。
- 彩色 IoU/edge patch。
- 模板 fallback。
- 神经特征匹配。

## 错误点剔除

特征点深度进入融合前，应按以下顺序验证:

1. 点必须落在左右 projected overlap mask 内。
2. 校正后 y 残差必须小，一般 1-2 px；若系统性倾斜存在，应先从多点拟合 epipolar slope/offset，再做残差门限。
3. 左右反查一致。
4. 视差使用 weighted median，不能简单平均。
5. 使用 MAD 或 1D RANSAC 剔除离群视差。
6. 三角化点必须接近球体物理半径，明显偏离球面的点丢弃。

## ROI 预处理

对大 ROI:

- 降采样到固定球直径再匹配，控制特征提取时间。
- ROI 内预计算灰度、Lab、label、edge 等中间图。
- 坐标输出必须反变换回校正图全局坐标。

对小/远球:

- 不应过度降采样。
- 可降低最大特征点数，优先保留边缘和高置信候选。
- 若点太少，记录 invalid reason，不要伪造深度。

## 验证指标

连续图像评估必须输出:

- 每帧有效点数。
- validation pass rate。
- 深度中位数和 MAD。
- 帧间深度抖动。
- 每帧耗时。
- 球运动模型残差。
