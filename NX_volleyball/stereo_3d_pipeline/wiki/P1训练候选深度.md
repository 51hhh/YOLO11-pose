# P1 训练候选深度

最后核对: 2026-07-04

本页记录当前正式 100fps 采集里随 P0 一起写入主 CSV 的 P1 候选。P1 的定位是训练输入和偏置学习，不是在线主深度，也不会抢占 legacy `z_stereo/obj.z`。

## 当前结论

当前正式 P1 主 CSV 候选只有两个:

| 字段 | 方法 | 当前角色 |
|---|---|---|
| `z_roi_multi_point` | CUDA multi-point subpixel ZNCC | 覆盖率高，速度可接受，但相对 P0 有系统偏置 |
| `z_roi_center_patch` | ROI center patch ZNCC | 可作为补充维度，覆盖率和偏置需用真实数据校正 |

XFeat 的 `z_roi_neural_feature` 已接入但默认关闭，不属于正式 P1 采集字段；见 [XFeatTensorRT深度候选](XFeatTensorRT深度候选.md)。

## 左右视差怎么来

`z_roi_multi_point` 在左球 ROI 内选取多个采样点，用小 patch 沿右图极线附近搜索 ZNCC 最佳位置。多个点的视差经过一致性、MAD/stddev 和初始视差 gate 后聚合。

`z_roi_center_patch` 只围绕中心 patch 做局部匹配，输出单个中心视差。它比多点更简单，但对中心纹理和初始几何偏置更敏感。

两者都以 P0/direct pair 的初始视差为约束，不是全图搜索:

```text
initial_disp = left_center_x - right_center_x
search range = initial_disp +/- small_range
z = focal * baseline / matched_disparity
```

## 配置入口

```yaml
detector:
  dual_yolo:
    depth_modes:
      roi_center_patch: true
      roi_subpixel: true
    subpixel_enabled: true
    subpixel_patch_radius: 5
    subpixel_search_radius_px: 8
    subpixel_max_points: 12
    subpixel_min_points: 4
    subpixel_min_confidence: 0.25
    subpixel_max_disp_delta_px: 2.0
    subpixel_max_stddev_px: 1.0
```

主要实现入口:

- `src/pipeline/pipeline_dual_yolo_match.cpp`
- `src/stereo/dual_yolo_depth_gpu.cu`
- `src/stereo/dual_yolo_depth_gpu_roi_match.cuh`
- `src/stereo/depth_candidate_builder.cpp`

## 已知表现

历史单项测试:

| 字段 | FPS | 有效率 | 中位深度 | MAD | 结论 |
|---|---:|---:|---:|---:|---|
| `z_roi_multi_point` | `100.1` | `628/628` | `3.5384m` | `0.0000m` | 覆盖率和速度好，但相对 P0 有约 `7cm` 系统差 |
| `z_roi_center_patch` | `100.0` | `45/45` | `3.5384m` | `0.0000m` | 单项覆盖有限，只能作补充候选 |

P0+P1 联合录制复核中:

- `z_roi_multi_point`: `2229/2304`
- `z_roi_center_patch`: `2239/2304`
- multi-point 相对 P0 几何中值约 `-5.17cm`
- center patch 相对 P0 几何中值约 `+5.30cm`

因此 P1 当前价值是让模型学习“什么时候偏、偏多少、是否可用”，不是直接替换 P0。

## 质量风险

- P1 patch 类方法在排球弱纹理或重复纹理上可能得到稳定但有系统偏差的视差。
- 当前静态测试证明了速度和覆盖率，不等于动态片段可靠。
- 对动态旋转排球，patch 可能锁到相似条纹区域，需要用帧间跳变和 known_z bias 分组评估。
- P1 不应写入 `z_stereo/obj.z`，否则会污染在线主观测。

## 采集和训练口径

正式采集时保留两个字段:

```text
z_roi_multi_point
z_roi_center_patch
```

训练时建议把它们作为 P1 组单独统计:

- valid ratio
- median / MAD
- 相对 P0 的 bias
- 相对 known_z 的 bias
- 动态片段帧间跳变
- direct pair / fallback 分组表现

如果 P1 在动态片段里出现稳定系统偏置，可以由可靠性模型学习；如果出现离散误匹配跳变，应降低权重或从该场景输入中剔除。
