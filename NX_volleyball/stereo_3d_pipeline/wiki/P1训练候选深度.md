# P1 训练候选深度

最后核对: 2026-07-19

本页记录 P1 候选及其历史测试。当前生产配置已将 `z_roi_multi_point` 提升为 direct pair 的在线首选；`z_roi_center_patch` 和 sidecar 候选仍只用于辅助记录、训练和专项测试。

P1 现在分两类承载:

- 主 CSV 每帧候选: `z_roi_multi_point`, `z_roi_center_patch`。
- P1 sidecar 候选: `z_roi_cuda_template_match`, `z_roi_neural_xfeat`, `z_roi_neural_superpoint`，由 `*.p2_diagnostic.csv` 按 `frame_id + mode` 合并进入训练表。

这里的 “P1 sidecar” 是训练分层，不是在线融合分层。运行实现仍使用 diagnostic worker，目的是不阻塞 P0/P1 主线。

## 当前结论

当前 P1 候选为五个:

| 字段 | 方法 | 承载 | 当前角色 |
|---|---|---|---|
| `z_roi_multi_point` | CUDA multi-point subpixel ZNCC | 主 CSV/在线 | 当前 direct pair 首选；历史测试显示相对 P0 有系统偏置，仍需持续实距验证 |
| `z_roi_center_patch` | ROI center patch ZNCC | 主 CSV | 可作为补充维度，覆盖率和偏置需用真实数据校正 |
| `z_roi_cuda_template_match` | 自研 CUDA Template/NCC | sidecar 合并 | 单点模板候选，速度快，`support=1`，用于给模型提供局部 patch 观测 |
| `z_roi_neural_xfeat` | XFeat TensorRT | sidecar 合并 | 有真实神经特征点对，但有效率仍低，作为辅助训练候选 |
| `z_roi_neural_superpoint` | SuperPoint TensorRT | sidecar 合并 | 当前神经候选里覆盖率最好，但相对 P0/P1 有系统差，需要模型学习偏置 |

## 左右视差怎么来

`z_roi_multi_point` 在左球 ROI 内选取多个采样点，用小 patch 沿右图极线附近搜索 ZNCC 最佳位置。多个点的视差经过一致性、MAD/stddev 和初始视差 gate 后聚合。

`z_roi_center_patch` 只围绕中心 patch 做局部匹配，输出单个中心视差。它比多点更简单，但对中心纹理和初始几何偏置更敏感。

`z_roi_cuda_template_match` 使用左 bbox/ROI 中心附近灰度 patch，在右图初始视差附近的小极线窗口内做自研 CUDA NCC/peak reduce，输出单点 peak 视差。

`z_roi_neural_xfeat` 和 `z_roi_neural_superpoint` 使用 TensorRT extractor 提取左右 ROI keypoint/descriptor，再做 mutual match、几何 gate 和鲁棒视差聚合。

这些 P1 候选都以 P0/direct pair 的初始视差为约束，不是全图搜索:

```text
initial_disp = left_center_x - right_center_x
search range = initial_disp +/- small_range
z = focal * baseline / matched_disparity
```

## 质量风险

- P1 patch 类方法在排球弱纹理或重复纹理上可能得到稳定但有系统偏差的视差。
- NCC 是单点模板，`support=1`，不能和多点特征的 support 直接比较。
- XFeat 有效率仍低，invalid 主要来自匹配数不足、inlier 不足和空间覆盖不足。
- SuperPoint 覆盖率高，但深度和 P0 几何存在系统差。
- 当前静态测试证明了速度和覆盖率，不等于动态片段可靠。
- 对动态旋转排球，patch 可能锁到相似条纹区域，需要用帧间跳变和 known_z bias 分组评估。
- P1 不应写入 `z_stereo/obj.z`，否则会污染在线主观测。

## 采集和训练口径

正式采集时保留这些字段:

```text
z_roi_multi_point
z_roi_center_patch
z_roi_cuda_template_match
z_roi_neural_xfeat
z_roi_neural_superpoint
```

训练时建议把它们作为 P1 组单独统计:

- valid ratio
- median / MAD
- 相对 P0 的 bias
- 相对 known_z 的 bias
- 动态片段帧间跳变
- direct pair / fallback 分组表现

如果 P1 在动态片段里出现稳定系统偏置，可以由可靠性模型学习；如果出现离散误匹配跳变，应降低权重或从该场景输入中剔除。
