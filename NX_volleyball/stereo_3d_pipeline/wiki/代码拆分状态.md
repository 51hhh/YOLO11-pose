# 代码拆分状态

最后核对: 2026-07-04

本页只记录单文件膨胀治理的当前状态。原则是先拆职责边界清楚、可本地验证的 Python/工具层，再处理需要 NX/CUDA/TensorRT 编译验证的实时 C++/CUDA 路径。

当前扫描: Python 最大文件约 `299` 行，工具/融合层剩余大多是单一离线诊断编排、测试断言或 helper。C++/CUDA 最大文件仍在实时 pipeline、CUDA depth candidate、TensorRT detector 和 viewer/calibration 工具，涉及编译环境、GPU stream、deadline 或设备 SDK 的改动必须在 NX 上验证。
