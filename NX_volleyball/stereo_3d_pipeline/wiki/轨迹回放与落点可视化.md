# 轨迹回放与落点可视化

最后更新: 2026-07-10

用录制的 `traj.csv` 在 RViz 中查看：

- 连续实际轨迹
- Student-t EKF 滤波轨迹
- RK4 预测飞行曲线
- 预测落点

相关代码：

- 回放节点: `NX_volleyball/replay_node/replay_trajectory.py`
- Docker 一键启动: `NX_volleyball/replay_node/start_replay_in_humble.sh`
- 本机 ROS 启动: `NX_volleyball/replay_node/run_replay_landing.sh`
- 落点算法: `trajectory_fusion/landing_pipeline/`
- 实时落点说明: [实时落点预测EKF](实时落点预测EKF.md)
- ROS 话题: [ROS2接口](ROS2接口.md)

## 1. 当前默认行为

```text
读取 traj.csv
  -> 默认按时间连续拼接多 track 检测
  -> bbox_center + d0 反投影
  -> Student-t EKF (Cd=0.10)
  -> RK4 落点
  -> 发布 /ball/* 到 RViz
```

默认参数：

| 项 | 默认 | 说明 |
|---|---|---|
| 播放模式 | 连续时间线 | 不默认只锁一个 `track_id` |
| 落点方案 | `landing_ekf` | bbox + d0 + EKF + RK4 |
| `d0` | `-5.324` | runtime 标定 `disparity_offset_fit_20260709.json` |
| `camera-height` | `0.50` | 当前数据集相机高 50cm |
| `ground_h` | `-0.50` | 默认 `-camera-height` |
| 距离剔除 | **关闭** | 只有手动 `--max-range-m` 才剔除 |
| 循环播放 | 开启 | Docker 脚本默认 `--loop` |

## 2. track_id 与连续播放

### 在线录制

`traj.csv` 中的 `track_id` 来自 `HybridDepthEstimator`：

```text
YOLO 检测
  -> 已有 track 预测
  -> bbox IoU 贪心关联
  -> 匹配: 继承 track_id
  -> 未匹配: 分配新 track_id
```

因此：

- 在线是多目标跟踪，不是“全场唯一球 ID”
- 远处静球/误检也会有独立 `track_id`
- 丢检、遮挡、形变大时，同一物理球可能切 ID

### 回放层

默认 **continuous** 模式：

```text
按时间把多 track 检测串成一条可视化轨迹
切 track / 大时间间隔时：重置 EKF
实测路径 /ball/actual_path：保持连续
```

这是显示用拼接，不改写原始 csv 的 `track_id`。

调试时仍可：

```bash
--track-id 3      # 只看某个 ID
--auto-track      # 旧行为：自动只选一个飞行 track
```

### 环境变量

| 变量 | 默认 | 含义 |
|---|---|---|
| `SPEED` | `0.5` | 回放倍速 |
| `CAMERA_HEIGHT` | `0.50` | 相机高度 |
| `TRACK_ID` | 空 | 指定后只播该 track |
| `MAX_RANGE_M` | 空 | 指定后才按距离剔除 |

示例：

```bash

```

也可直接把参数传给脚本：

```bash
./NX_volleyball/replay_node/start_replay_in_humble.sh <csv> --max-range-m 10 --speed 0.5
```

## 5. RViz 看什么

Fixed Frame:

```text
vision_world
```

| 话题 | 含义 |
|---|---|
| `/ball/realtime` | 当前球位置 |
| `/ball/actual_path` | 实测轨迹（连续拼接后） |
| `/ball/filtered_path` | EKF 滤波轨迹 |
| `/ball/predicted_path` | 预测飞行曲线 |
| `/ball/landing` | 预测落点 |
| `/ball/ball_marker` | 当前球体 |
| `/ball/landing_marker` | 落点球体 |
| `/ball/ground_plane` | 地面参考平面 |

坐标变换：

```text
相机 (x右, y下, z前)
  -> RViz (x右, y前, z上)
  即 (x, y, z) -> (x, z, -y)
```

说明：

- 蓝线 `/ball/actual_path` 是观测轨迹；录制未到触地时会停在空中
- 橙线/红点才是预测落地
- 绿地面是水平参考面；物理 `g_hat` 可能略有倾角，红点可能轻微“悬空”

## 6. 键盘

| 键 | 作用 |
|---|---|
| `Space` | 暂停/继续 |
| `←/→` 或 `a/d` | ±1 帧 |
| `,/·` 或 `s/w` | ±10 帧 |
| `+/-` | 加减速 |
| `r` | 重头 |
| `q` | 退出 |

Seek 行为：

- **前进**：从当前状态续算，不整段重播
- **后退**：为保持 EKF 因果性，从开头重建到目标帧

## 7. 常用参数

```bash
python3 NX_volleyball/replay_node/replay_trajectory.py \
  --csv <traj.csv> \
  --mode landing_ekf \
  --camera-height 0.50 \
  --speed 0.5 \
  --loop
```

| 参数 | 默认 | 说明 |
|---|---|---|
| `--mode` | `landing_ekf` | `landing_ekf` / `legacy_filter` / `raw` |
| `--camera-height` | `0.50` | 相机高度 |
| `--ground-h` | `-camera-height` | 落点平面参数 |
| `--continuous` | 开 | 多 track 连续拼接 |
| `--no-continuous` | 关 | 关闭拼接 |
| `--track-id N` | 空 | 只播一个 track |
| `--auto-track` | 关 | 自动单飞行 track |
| `--max-range-m` | `0` | `>0` 时剔除更远检测；默认不剔除 |
| `--residual` | 关 | 可选 TinyGRU 残差 |
| `--loop` | 脚本默认开 | 循环播放 |

### 距离剔除

按**相机深度 z**过滤：

```bash
--max-range-m 10   # 丢掉 z>10m 的点
--max-range-m 0    # 不剔除（默认）
```

默认不剔除，避免误伤远距真实出手点。远处干扰多时再手动加。

## 8. 数据与观感说明

很多录制并不是“一条 track 完整落到地面”：

- 飞行中途可能切 `track_id`
- 部分片段在触地前结束
- 同文件常混有远距静球/误检 track

因此：

1. 看完整过程：用默认连续模式
2. 远处误检多：手动 `--max-range-m 10~12`
3. 判断落地预测：看橙线/红点，不要只看蓝线是否贴地
4. 录制规范建议：出手前开始，连续到第一次触地后 0.3~0.5s，尽量减少丢检切 ID

推荐查看片段：

| 片段 | 说明 |
|---|---|
| `.../20260710_042552/traj.csv` | 效果较好 |
| `.../20260710_042259/traj.csv` | 接近落地 |
| `.../20260710_042318/traj.csv` | 偏差较大，对照用 |

## 10. 故障排查

| 现象 | 原因/处理 |
|---|---|
| `192.168.31.72 does not match an available interface` | 旧 DDS 绑定网卡；用 `start_replay_in_humble.sh` 或 `cyclonedds_local.xml` |
| 蓝线空中截断 | 数据未录到触地，或 track 中断；用连续模式并看红点橙线 |
| 远处乱跳 | 额外误检/静球；手动 `--max-range-m` |
| 只有静球 | 文件本身以远距目标为主，或需换片段 |
| 前进像整段重播 | 旧 bug，已修复为前进续算 |
| 无 `d0=-5.324` 日志 | 确认跑的是当前 `replay_trajectory.py`，且 `mode=landing_ekf` |
