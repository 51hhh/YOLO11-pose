# 轨迹融合与数据集

## 目标

轨迹融合模块记录多种深度候选、检测几何、同步状态和诊断字段，用于训练或评估可靠性模型。它不应掩盖实时深度算法的问题，而是帮助判断不同候选在不同条件下的可信度。

## 数据入口

实时记录配置:

```yaml
recording:
  enabled: true
  output_path: "dual_yolo_observation_data.csv"
  detail_level: "extended"
  raw_mode: true
  max_queue_frames: 1000
```

字段定义以 `trajectory_fusion/schema.md` 为准。

实时 `Object3D`、TrajectoryRecorder 和 baseline clip CSV 的字段关系见 [数据结构与Schema](数据结构与Schema.md)。深度候选字段分组见 [深度候选字段参考](深度候选字段参考.md)，实机采集流程见 [轨迹模型数据采集流程](轨迹模型数据采集流程.md)。

训练候选以 `trajectory_fusion/dataset.py` 的 `METHOD_COLUMNS` 为准，只包含原始 `z_*` 候选字段，不包含 legacy `z_stereo` 或在线 `z`。实时 C++ 为兼容 ROS、日志和旧轨迹预测仍会用 `buildDepthCandidateObservations()` + `selectLegacyDepthOutputCandidate()` 写出 `z_stereo/z`，但这只允许 P0 几何/bbox 和退化 fallback，不是训练模型的深度决策。

当前 100fps 采集口径以 P0 几何、P1 主 CSV `z_roi_multi_point` / `z_roi_center_patch` 和 P1 sidecar 三算法为主。`dataset.py` 会自动查找同名前缀 `*.p2_diagnostic.csv`，并按 `frame_id + mode` 合并 `cuda_template` -> `z_roi_cuda_template_match`、`neural_xfeat` -> `z_roi_neural_xfeat`、`neural_superpoint` -> `z_roi_neural_superpoint`，同时合并对应 disparity/support/std/confidence。OpenCV CUDA GFTT/LK、color/color-edge、SGM 和 VPI 字段保留为历史/实验候选，默认不启用。没有 sidecar 时这些候选保持无效。

## 数据集代码

| 文件 | 职责 |
|---|---|
| `trajectory_fusion/schema.md` | CSV 字段定义 |
| `trajectory_fusion/dataset.py` | 读取和构建训练样本 |
| `trajectory_fusion/check_dataset.py` | 采集后离线质量检查 |
| `trajectory_fusion/manifest.py` | 多段 CSV 数据集 manifest 读取 |
| `trajectory_fusion/train_reliability.py` | 单 CSV、多 CSV 或 manifest 训练入口 |
| `trajectory_fusion/evaluate_fusion.py` | 评估融合结果，支持 JSON/CSV 导出 |
| `trajectory_fusion/README.md` | 实验说明和历史结论 |

多段训练数据使用 `dataset_manifest.yaml` 管理:

```yaml
clips:
  - csv: traj_p0p1_001.csv
    metadata: traj_p0p1_001.metadata.yaml
    split: train
  - csv: traj_p0p1_002.csv
    metadata: traj_p0p1_002.metadata.yaml
    split: val
```

路径相对 manifest 所在目录解析。`train_reliability.py` 默认只优化 `split: train` 的片段，其他 split 会作为 heldout 记录进 checkpoint metadata。

`check_dataset.py` 除基础行数、FPS、水印和 P0/P1 主 CSV 深度统计外，还会输出:

- `source_breakdown`: direct pair、fallback L2R/R2L、circle source 和 depth source 组合计数。
- `frame_summary`: 自动读取同名前缀 `.frames.csv`，汇总每帧 raw/stereo/direct/fallback 数量。
- `depth_jump`: 各候选相邻有效帧深度跳变的 median/MAD/p95/max。
- `p2_diagnostic`: 自动读取同名前缀 `*.p2_diagnostic.csv`，按 `mode` 汇总 rows、valid、median/MAD、algo p95、worker p95 和 deadline 次数。

VPI sidecar 不回写主 CSV，但 `check_dataset.py` 会输出 sidecar 摘要；`dataset.py` 训练入口会把 `vpi_template_match`、`vpi_orb` 自动合并成候选字段。

## 训练和评估原则

- 不把明显错误的标定或同步数据当成正常噪声学习。
- 多候选深度字段需要记录 invalid reason 或置信度，避免训练时把缺失值误作真实值。
- bbox、circle 和 ROI 几何候选存在强相关，评估时要按组看贡献。
- 在线 100fps 只保留验证过的少量候选；离线可以全量记录用于分析。
- 当前 `legacy_feature_names()` 不把在线 `x/y/z/vx/vy/vz` 状态字段作为训练输入；它读取候选深度、候选统计、质量字段、pair gate 和水印。严格候选-only 可靠性模型仍应保留“不读 `z_stereo/z`”这个约束，并在新增特征时避免把在线 legacy 选择结果重新引入。

## 必备评估指标

- 候选深度有效率。
- 候选间一致性。
- 深度中位数和 MAD。
- 帧间深度抖动。
- 运动模型残差。
- 最终轨迹对落点/实际轨迹的误差。
