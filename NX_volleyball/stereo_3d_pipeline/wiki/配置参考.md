# 配置参考

最后核对: 2026-07-03

主程序配置解析入口是 `src/main.cpp`。本页只记录当前代码实际读取的 YAML 节点；字段是否生效以代码为准。

## 主配置

默认配置路径是:

```bash
config/pipeline.yaml
```

当前实时测试和生产调参主配置是:

```bash
config/pipeline_dual_yolo_roi.yaml
```

运行时可通过 `--config` 或 `-c` 覆盖:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

## `baseline_recording`

用于录制连续左右目校正图像序列和检测 CSV。命令行 `--record-baseline-clip` 会强制启用该功能。

| 字段 | 说明 |
|---|---|
| `enabled` | 是否默认启用 baseline clip 录制 |
| `output_dir` | 输出根目录 |
| `duration_sec` | 单段 clip 时长；`frame_limit=0` 时按触发频率换算帧数 |
| `frame_limit` | 单段 clip 精确帧数，优先于 `duration_sec` |
| `clip_count` | 连续录制 clip 数 |
| `clip_gap_sec`, `clip_gap_frames` | clip 间隔；`clip_gap_frames>0` 优先 |
| `require_left_detection`, `require_right_detection` | 开始录制前是否要求对应目检测到目标 |
| `require_pair_gate` | 是否要求左右检测框通过极线/尺寸/视差 gate 才开始 |
| `min_confidence` | 参与开始条件和最佳框选择的最低检测置信度 |
| `pair_y_tolerance_px` | `require_pair_gate=true` 时左右框中心 y 差门限 |
| `pair_max_size_ratio` | `require_pair_gate=true` 时左右框宽高比例门限 |
| `pair_min_disparity_px` | `require_pair_gate=true` 时最小正视差门限 |
| `image_format` | `png` 或 `pgm`; BGR 模式会强制使用 `png` |
| `image_mode` | `gray` 或 `bgr`; `both` 仅用于调试兼容，会额外写 `left_bgr/`、`right_bgr/` |
| `png_compression` | PNG 压缩级别 0-9; 正式实时采集建议低压缩 |
| `write_after_capture` | `true` 时录制阶段只缓存到内存，clip 结束后再写盘 |
| `max_queue_frames` | 内存队列上限，`0` 为不限；`write_after_capture=true` 时建议不限或至少大于目标帧数 |
| `stop_after_clip` | 完成全部 clip 后是否退出程序 |

启用 baseline 录制时，`main.cpp` 会把实时程序切到轻量采集模式: `detection_only=true`、关闭 tracker、关闭神经特征、关闭双目深度输出，并禁用普通轨迹 CSV 写入。这个模式用于生成离线算法输入，不用于在线落点预测。

命令行覆盖关系:

| 命令行 | 覆盖字段 |
|---|---|
| `--record-baseline-clip` | `baseline_recording.enabled=true` |
| `--baseline-out DIR` | `output_dir` |
| `--baseline-duration SEC` | `duration_sec` |
| `--baseline-frames N` | `frame_limit` |
| `--baseline-clips N` | `clip_count` |
| `--baseline-gap SEC` | `clip_gap_sec` |
| `--baseline-format png|pgm` | `image_format` |
| `--baseline-image-mode gray|bgr|both` | `image_mode` |
| `--baseline-start-immediately` | `require_left_detection=false`, `require_right_detection=false`, `require_pair_gate=false` |

## `camera`

关键字段:

| 字段 | 说明 |
|---|---|
| `left_index`, `right_index` | 未指定序列号时按索引打开相机 |
| `serial_left`, `serial_right` | 推荐使用的相机绑定方式 |
| `exposure_us`, `gain_db` | 固定曝光和增益 |
| `auto_exposure`, `auto_gain` | 自动曝光/增益，标定和对比实验通常关闭 |
| `use_trigger` | 是否外触发 |
| `trigger_source`, `trigger_activation` | 触发线和边沿 |
| `trigger_frequency_hz` | 相机侧期望触发频率 |
| `image_node_num` | SDK FIFO 深度，用于吸收 USB 到达抖动 |
| `embedded_info_clear_rows` | 清除海康 FrameSpecInfo 水印行，避免污染 debayer/YOLO |
| `trigger_chip`, `trigger_line` | PWM GPIO 输出位置 |
| `width`, `height` | 原始采集分辨率 |

如果 `camera.trigger_frequency_hz` 未显式设置，代码会使用 `performance.pwm_frequency` 写入相机配置中的触发频率。

## `neural_feature_matching`

| 字段 | 含义 |
|---|---|
| `enabled` | 是否启用神经特征候选 |
| `backend` | `xfeat`, `aliked`, `superpoint_lightglue` |
| `extractor_engine_path` | extractor TensorRT engine |
| `matcher_engine_path` | split matcher TensorRT engine；仅 `use_lightglue=true` 且无 fused engine 时加载，当前只支持固定 keypoint/descriptor 输入和 `matches`/`matches0` 输出 schema |
| `fused_engine_path` | fused TensorRT engine |
| `roi_size` | 固定 ROI 输入尺寸 |
| `top_k` | 最大关键点数 |
| `descriptor_dim` | 描述子维度 |
| `min_matches` | 最少有效匹配数 |
| `max_y_error_px` | y 残差门限 |
| `max_disp_delta_px` | 与初始视差的最大差值 |
| `final_disp_gate_px` | 最终视差 inlier 门限 |
| `min_score` | 最低匹配分数 |
| `use_lightglue` | 是否启用 `matcher_engine_path` split matcher；关闭时即使配置 matcher path 也走 extractor-only descriptor matching |

实时 C++ 当前支持三类路径:

- `fused_engine_path` 非空: 使用 fused TensorRT matcher 输出点对。
- `backend: "xfeat"` 且 `extractor_engine_path` 可用、无 fused engine: 使用 XFeat extractor-only TensorRT + CPU 后处理/互反查匹配 fallback。
- 无 fused engine 且 `extractor_engine_path` 输出固定 shape `keypoints/descriptors/scores`: 使用 direct extractor TensorRT + CPU descriptor mutual-NN fallback, 适用于 ALIKED/SuperPoint extractor-only 类 engine 的接线验证。
- 无 fused engine、`use_lightglue=true` 且同时配置 `matcher_engine_path`: extractor 仍先在左右 ROI 上各跑一次；若 matcher engine 输入能按名称/shape 识别为左右 keypoints/descriptors/可选 scores/image_size，且输出为 `matches [M,2/3]` 或左到右 `matches0`/`scores0`，则执行 TensorRT split matcher 后再回 CPU 做几何 gate。

split matcher 不是无限通用的 LightGlue runtime：动态/未知 tensor schema、需要 mask/length/额外状态的 engine 会被标记为 unsupported，并回退到 direct descriptor matching 或返回失败。生产验证仍优先使用 fused 点对 engine，因为它少一次中间 D2H/H2D 往返，schema 也更稳定。

## `performance`

| 字段 | 含义 |
|---|---|
| `pwm_frequency` | 未显式配置 `camera.trigger_frequency_hz` 时的触发频率来源 |
| `stats_interval` | 运行日志和统计打印间隔 |
| `detection_only` | 只运行采集/校正/检测/回调，跳过 stereo/depth |
| `drop_stale_roi_frames` | 同步路径进入旧帧 ROI 后处理前，如果后一帧 YOLO 已 ready，则丢弃旧后处理 |
| `async_roi_stage2` | ROI_ONLY 检测帧把 IoU/ROI 特征/候选深度整段放入 async worker，下一帧 YOLO ready 后过期旧结果 |
| `async_roi_buffers` | async worker 自有快照 buffer 数，代码限制在 2-8；gray GPU 固定分配，BGR GPU 和 pinned host gray 按算法需求分配 |
| `async_roi_deadline_ms` | async worker 软预算；超过会记录 `Stage2_AsyncRoiOverDeadline`，结果是否写回仍由 frame deadline 判断 |

`async_roi_stage2` 只在 ROI_ONLY、非 detection-only、tracker 未启用或 `detect_interval<=1` 时生效。worker 不持有 `FrameSlot` 图像指针，只读取 snapshot buffer；slot 复用前只等待 snapshot copy event，不等待重特征完成。CPU OpenCV 特征或 CPU fallback 未启用时，不分配 pinned host gray buffer。

## 运行时覆盖

命令行只覆盖少量录制和调试字段:

```text
--record-baseline-clip
--baseline-out <dir>
--baseline-duration <sec>
--baseline-frames <n>
--baseline-clips <n>
--baseline-gap <sec>
--baseline-format <png|pgm>
--baseline-image-mode <gray|bgr|both>
--baseline-start-immediately
--debug-feature-matches
--debug-feature-matches-dir <dir>
```

如果要做算法矩阵测试，应生成临时配置文件，不要直接修改主配置。
