# 实机测试清单

最后核对: 2026-07-02

本页用于分支合入前的 NX 实机检查。测试结论必须写明日期、git commit、配置文件、模型 engine、标定文件和命令。

## 1. 相机和触发

先独立跑水印探针:

```bash
./hik_frame_metadata_probe \
  --left-sn 00D39342665 \
  --right-sn 00219471413 \
  --frames 500 \
  --pwm-hz 100
```

检查:

- `frame_counter_delta` 稳定为 0。
- 左右 `trigger_index` 连续递增。
- `timestamp_delta_us` 无明显漂移。
- 无周期性超时或丢帧。

如果相机由外部触发源控制，使用 `--no-pwm`，并记录触发源。

## 2. 标定和校正

检查:

- `stereo_calib.yaml` 是当前相机、分辨率和棋盘规格生成。
- baseline 与实测长基线一致。
- 校正预览中同一物点水平对齐。
- 实时左右球框 y 残差无系统性偏移。

异常时优先重新采集标定数据，不通过放宽 y 门限掩盖问题。

## 3. 默认实时管线

命令:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

检查:

- FPS 接近 100Hz 触发频率。
- `Stage0_WaitGrab` 无大量 outlier。
- `Stage1_DetectSubmit` 和 `Stage2_WaitYOLOComplete` 无异常增长。
- `Stage2_DualYoloGpuCandidates` 在预算内。
- `stale_drop` 不应持续快速增长；若增长，确认是后处理慢而不是取流问题。
- CSV 写盘队列不长期满。

## 4. 录制基准片段

命令:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 3.0 \
  --baseline-clips 3 \
  --baseline-gap 1.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

如果只需要连续左右目图像, 不想等检测框触发:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-frames 300 \
  --baseline-clips 3 \
  --baseline-gap 1.0 \
  --baseline-image-mode bgr \
  --baseline-start-immediately
```

检查:

- 输出 `left/`、`right/`、`frames.csv`、`metadata.yaml`。
- `--baseline-image-mode bgr` 时 `left/`、`right/` 内直接保存彩色图；`gray` 保存灰度图；`both` 仅用于调试兼容并额外输出 `left_bgr/`、`right_bgr/`。
- 帧数接近目标 duration 或 `--baseline-frames`。
- `frames.csv` 中同步 delta 稳定。
- 写盘队列满时丢录制帧，而不是拖慢主循环。

## 5. 特征和深度算法 A/B

默认配置只开轻量候选。打开重候选时:

- 每次只改一个候选组。
- 使用相同 baseline clip 做离线指标。
- 再在 NX 上跑短时矩阵测试。
- 记录有效点数、validation pass rate、深度中位数、MAD、帧间抖动、耗时和运动模型残差。

可用脚本:

```bash
python3 scripts/nx_algorithm_matrix_test.py --help
python3 tools/offline_volleyball_keypoint_probe.py --help
python3 tools/compare_offline_online_depth.py --help
```

P2 算法测试失败、超时、deadline 丢弃或需要 zoom 图时，按 [P2算法NX测试排查指南](P2算法NX测试排查指南.md) 处理。准入测试不要打开 `--debug-on-failure`；失败复盘时再单独开启。

## 6. ROS2 和显示

单独测试:

- `--visualize` 打开时的 FPS 和 CPU/GPU 占用。
- `ros2.enable=true/false` 的性能差异。
- `ros2.diagnostic.enable=true/false` 的性能差异。
- 诊断图像发布是否造成 GPU 到 CPU 拷贝瓶颈。
