## 转换脚本

| 脚本 | 运行位置 | 输出 | 当前默认 |
|---|---|---|---|
| `scripts/export_xfeat_extractor_onnx.py` | NX 或有 XFeat repo 的本机 | `xfeat_extractor_<roi>.onnx` | 只导出 ONNX |
| `scripts/nx_build_xfeat_extractor_engine.sh` | NX | XFeat ONNX + engine | `ROI_SIZE=128`, FP16 |
| `scripts/export_lightglue_extractor_onnx.py` | NX 或有 LightGlue repo 的本机 | ALIKED/SuperPoint fixed extractor ONNX | `--backend` 必填，支持 `--batch-size` |
| `scripts/nx_build_lightglue_extractor_engines.sh` | NX | SuperPoint/ALIKED ONNX + engine | `BACKENDS=superpoint`, `BATCH_SIZE=1`, FP16 |
| `scripts/export_aliked_dcn_extractor_onnx.py` | NX 或有 LightGlue repo 的本机 | 官方 ALIKED DCN fixed extractor ONNX | 需要 `onnxscript`，opset>=19，输出 `dcn::DCNv2` |
| `scripts/nx_build_dcnv2_plugin.sh` | NX | TensorRT DCNv2 plugin `.so` | 输出 `models/neural/plugins/libdcnv2.so` |
| `scripts/nx_build_aliked_dcn_engine.sh` | NX | 官方 ALIKED-DCN ONNX + engine | `aliked-t16`, `ROI_SIZE=128`, `TOP_K=64`, `BATCH_SIZE=2` |

`nx_build_lightglue_extractor_engines.sh` 默认只构建 SuperPoint。ALIKED 官方模型含 DCN，不再使用该 vanilla 脚本作为正式导出入口；需要复现 vanilla 失败时才显式运行:

```bash
BACKENDS=aliked ALIKED_MODEL=aliked-n16 \
  scripts/nx_build_lightglue_extractor_engines.sh
```

如果要一次尝试多个 backend 并继续保留成功项:

```bash
BACKENDS="superpoint aliked" ALLOW_FAILURES=1 \
  scripts/nx_build_lightglue_extractor_engines.sh
```

| `models/neural/superpoint_extractor_128_top64.engine` | `0.7907ms` | `0.7722ms` | engine 快；实时矩阵 `90.4fps`、`12/230` 有效，仍不准入 |

| `models/neural/superpoint_extractor_160_top64_b2.engine` | `1.7804ms` | `1.7427ms` | batch=2 engine；实时 sidecar stride=2 relaxed 为 `99.6fps`、`920/920` 有效，但每帧运行不准入 |
| `models/neural/superpoint_extractor_224_top64.engine` | `1.9199ms` | `1.8928ms` | engine 可跑；实时矩阵 `74.1fps`、`2/178` 有效 |
| `models/neural/superpoint_extractor_224_top128.engine` | `1.932ms` | `1.890ms` | engine 可跑, 但实时矩阵超预算 |

runtime schema:

```text
input:  images=BATCH_SIZEx1xROI_SIZExROI_SIZE
output: keypoints=BATCH_SIZExTOP_Kx2
output: descriptors=BATCH_SIZExTOP_Kx256
output: scores=BATCH_SIZExTOP_K
```

注意: LightGlue 原始 SuperPoint 导出会因动态 `MaxPool` shape 让 TensorRT parser 失败。当前成功 engine 来自 `export_lightglue_extractor_onnx.py` 中的 `FixedSuperPoint` wrapper，保留真实 SuperPoint 卷积权重，但把动态 Python 后处理替换为固定 top-k 输出。

## ALIKED

ALIKED 官方 `aliked-n16` / `aliked-t16` 均包含 DCN block。当前正式路线是: PyTorch dynamo/onnxscript 新 exporter 导出 ONNX `DeformConv`，再将节点改写为 TensorRT 插件识别的 `dcn::DCNv2`，最后用 DCNv2 plugin 构建 engine。历史 no-DCN engine 只作为工程回退和对照，不再代表官方 ALIKED。

已验证失败:

```text
aliked-n16: torch.onnx.errors.UnsupportedOperatorError: torchvision::deform_conv2d
aliked-t16: torch.onnx.errors.UnsupportedOperatorError: torchvision::deform_conv2d
```

原因: LightGlue ALIKED 配置的后两层仍使用 DCN block，`aliked-t16` 也没有绕开 deformable convolution。vanilla ONNX/trtexec 不能落地。

官方 DCN 构建步骤:

```bash
cd /home/nvidia/NX_volleyball/stereo_3d_pipeline

# 1. 构建 TensorRT DCNv2 plugin。
scripts/nx_build_dcnv2_plugin.sh

# 2. 导出官方 ALIKED-t16 DCN ONNX，并用 plugin 构建 TensorRT engine。
ROI_SIZE=128 TOP_K=64 BATCH_SIZE=2 ALIKED_MODEL=aliked-t16 \
  scripts/nx_build_aliked_dcn_engine.sh
```

输出:

```text
models/neural/plugins/libdcnv2.so
models/neural/aliked_t16_dcn_extractor_128_top64_b2.onnx
models/neural/aliked_t16_dcn_extractor_128_top64_b2.engine
```

2026-07-06 NX 构建结果:

| 项 | 结果 |
|---|---|
| DCN 模块 | patch `4` 个官方 ALIKED DCN layer |
| ONNX 节点 | `DeformConv -> dcn::DCNv2` 共 `4` 个 |
| TensorRT plugin | `models/neural/plugins/libdcnv2.so` |
| trtexec plugin 参数 | `--staticPlugins=.../libdcnv2.so` |
| engine | `models/neural/aliked_t16_dcn_extractor_128_top64_b2.engine` |
| 输出 schema | `keypoints=2x64x2`, `descriptors=2x64x64`, `scores=2x64` |
| trtexec latency | mean `2.5315ms`, p95 `2.5574ms` |
| trtexec GPU compute | mean `2.5074ms`, p95 `2.5305ms` |

注意:

- 该 plugin 库没有 TensorRT dynamic plugin `getCreators` 导出符号，`--dynamicPlugins` 会失败；当前构建必须使用 `--staticPlugins`。
- ONNXScript 会给部分 `ScatterND` 写入 `reduction="none"`，TensorRT 10.3 不支持这个显式属性；导出脚本会在 ONNX 后处理阶段移除它，语义等价于默认 no-reduction。
- NX 用户环境已安装 `onnxscript`。该安装会把用户级 `onnx` 升到 `1.22`，和 `onnx-graphsurgeon 0.5.7` 的声明约束冲突；后续若 ONNX GraphSurgeon 脚本受影响，应为 ALIKED-DCN 导出单独建 venv 或恢复用户级 `onnx==1.16.1` 后按需使用 venv 导出。

实时配置必须同时填写 engine 和 plugin:

```yaml
neural_feature_matching_aliked:
  backend: "aliked"
  extractor_engine_path: "/home/nvidia/NX_volleyball/stereo_3d_pipeline/models/neural/aliked_t16_dcn_extractor_128_top64_b2.engine"
  plugin_library_path: "/home/nvidia/NX_volleyball/stereo_3d_pipeline/models/neural/plugins/libdcnv2.so"
```

运行时会在反序列化 TensorRT engine 前 `dlopen(..., RTLD_GLOBAL)` 加载 plugin；插件库必须在 matcher 析构前保持加载。

重要结论:

- XFeat engine 可以进入实时管线推理，extractor trtexec 延迟远低于 1ms；当前瓶颈是 ROI 后处理/匹配和有效率。已测变体中，`neural_xfeat_128_top32` 有效率最高但只有 `24/375`，且 `94.7fps`，仍不准入。
- SuperPoint fixed extractor engine 可以进入实时管线推理；`128/top64` trtexec 平均 `0.79ms`，但实时管线只有 `90.4fps` 和 `12/230` 有效，仍不准入。
- ALIKED 官方 DCN 路线已补导出和插件构建入口；NX 上需要安装 `onnxscript`、构建 `libdcnv2.so`，再用 `nx_build_aliked_dcn_engine.sh` 生成 engine 后复测实时质量和 FPS。
