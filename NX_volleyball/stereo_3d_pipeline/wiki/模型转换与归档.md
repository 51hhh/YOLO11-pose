## 转换脚本

| 脚本 | 运行位置 | 输出 | 当前默认 |
|---|---|---|---|
| `scripts/export_xfeat_extractor_onnx.py` | NX 或有 XFeat repo 的本机 | `xfeat_extractor_<roi>.onnx` | 只导出 ONNX |
| `scripts/nx_build_xfeat_extractor_engine.sh` | NX | XFeat ONNX + engine | `ROI_SIZE=128`, FP16 |
| `scripts/export_lightglue_extractor_onnx.py` | NX 或有 LightGlue repo 的本机 | ALIKED/SuperPoint fixed extractor ONNX | `--backend` 必填 |
| `scripts/nx_build_lightglue_extractor_engines.sh` | NX | SuperPoint/ALIKED ONNX + engine | `BACKENDS=superpoint`, FP16 |

`nx_build_lightglue_extractor_engines.sh` 默认只构建 SuperPoint，因为 ALIKED 当前会在 vanilla ONNX export 卡在 `torchvision::deform_conv2d`。需要复现 ALIKED 失败时显式运行:

```bash
BACKENDS=aliked ALIKED_MODEL=aliked-n16 \
  scripts/nx_build_lightglue_extractor_engines.sh
```

如果要一次尝试多个 backend 并继续保留成功项:

```bash
BACKENDS="superpoint aliked" ALLOW_FAILURES=1 \
  scripts/nx_build_lightglue_extractor_engines.sh
```

## XFeat

NX 构建:

```bash
cd /home/nvidia/NX_volleyball/stereo_3d_pipeline
ROI_SIZE=96 scripts/nx_build_xfeat_extractor_engine.sh
ROI_SIZE=128 scripts/nx_build_xfeat_extractor_engine.sh
ROI_SIZE=160 scripts/nx_build_xfeat_extractor_engine.sh
ROI_SIZE=224 scripts/nx_build_xfeat_extractor_engine.sh
```

已归档:

| 文件 | `trtexec` latency mean | GPU compute mean | 运行结论 |
|---|---:|---:|---|
| `models/neural/xfeat_extractor_96.engine` | `0.3327ms` | `0.3185ms` | extractor 很快；实时 top32 仅 `6/229` 有效，top64 为 `0/248` |
| `models/neural/xfeat_extractor_128.engine` | `0.4977ms` | `0.4754ms` | 当前实时矩阵使用 |
| `models/neural/xfeat_extractor_160.engine` | `0.5855ms` | `0.5587ms` | 可用备选 |
| `models/neural/xfeat_extractor_224.engine` | `0.8020ms` | `0.7555ms` | 可用备选, 后处理成本更高 |

runtime schema:

```text
input:  images=1x1xROI_SIZExROI_SIZE
output: feats=1x64x(H/8)x(W/8)
output: keypoints=1x65x(H/8)x(W/8)
output: heatmap=1x1x(H/8)x(W/8)
```

## SuperPoint

NX 构建:

```bash
cd /home/nvidia/NX_volleyball/stereo_3d_pipeline
BACKENDS=superpoint scripts/nx_build_lightglue_extractor_engines.sh
ROI_SIZE=128 TOP_K=64 BACKENDS=superpoint scripts/nx_build_lightglue_extractor_engines.sh
ROI_SIZE=160 TOP_K=64 BACKENDS=superpoint scripts/nx_build_lightglue_extractor_engines.sh
ROI_SIZE=224 TOP_K=64 BACKENDS=superpoint scripts/nx_build_lightglue_extractor_engines.sh
```

已归档:

| 文件 | `trtexec` latency mean | GPU compute mean | 运行结论 |
|---|---:|---:|---|
| `models/neural/superpoint_extractor_128_top64.engine` | `0.7907ms` | `0.7722ms` | engine 快；实时矩阵 `90.4fps`、`12/230` 有效，仍不准入 |
| `models/neural/superpoint_extractor_160_top64.engine` | `1.1129ms` | `1.0932ms` | engine 可跑；实时矩阵 `57.9fps`、`0/147` 有效 |
| `models/neural/superpoint_extractor_224_top64.engine` | `1.9199ms` | `1.8928ms` | engine 可跑；实时矩阵 `74.1fps`、`2/178` 有效 |
| `models/neural/superpoint_extractor_224_top128.engine` | `1.932ms` | `1.890ms` | engine 可跑, 但实时矩阵超预算 |

runtime schema:

```text
input:  images=1x1xROI_SIZExROI_SIZE
output: keypoints=1xTOP_Kx2
output: descriptors=1xTOP_Kx256
output: scores=1xTOP_K
```

注意: LightGlue 原始 SuperPoint 导出会因动态 `MaxPool` shape 让 TensorRT parser 失败。当前成功 engine 来自 `export_lightglue_extractor_onnx.py` 中的 `FixedSuperPoint` wrapper，保留真实 SuperPoint 卷积权重，但把动态 Python 后处理替换为固定 top-k 输出。

## ALIKED

当前没有本地 engine，也不应把 Python probe 结果写成 realtime TensorRT 能力。ALIKED 算法概念保留在 P2 待测清单中，但必须先解决 TensorRT engine。

已验证失败:

```text
aliked-n16: torch.onnx.errors.UnsupportedOperatorError: torchvision::deform_conv2d
aliked-t16: torch.onnx.errors.UnsupportedOperatorError: torchvision::deform_conv2d
```

原因: LightGlue ALIKED 配置的后两层仍使用 DCN block，`aliked-t16` 也没有绕开 deformable convolution。vanilla ONNX/trtexec 不能落地。可选方向:

- 写 TensorRT deformable convolution plugin。
- 改模型结构为无 DCN 版本并重新训练/验证。
- 使用能导出固定 `keypoints/descriptors/scores` schema 的 ALIKED 变体。

若后续拿到可用 engine，矩阵脚本已预留:

```bash
ROI_SIZE=160 TOP_K=64 BACKENDS=aliked ALLOW_FAILURES=1 \
  scripts/nx_build_lightglue_extractor_engines.sh
ROI_SIZE=224 TOP_K=64 BACKENDS=aliked ALLOW_FAILURES=1 \
  scripts/nx_build_lightglue_extractor_engines.sh
```

重要结论:

- XFeat engine 可以进入实时管线推理，extractor trtexec 延迟远低于 1ms；当前瓶颈是 ROI 后处理/匹配和有效率。已测变体中，`neural_xfeat_128_top32` 有效率最高但只有 `24/375`，且 `94.7fps`，仍不准入。
- SuperPoint fixed extractor engine 可以进入实时管线推理；`128/top64` trtexec 平均 `0.79ms`，但实时管线只有 `90.4fps` 和 `12/230` 有效，仍不准入。
- ALIKED 没有 TensorRT engine，当前不能进入实时管线；`torchvision::deform_conv2d` 仍是导出阻塞点。
