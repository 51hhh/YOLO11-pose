# 排球轨迹模型训练方案

最后核对: 2026-07-03

本页是轨迹模型的讨论稿。结论先用于方案确认；在确认前不把模型推理接入 C++ 实时主路径。

## 需要确认的架构决策

| 问题 | 建议结论 | 原因 |
| --- | --- | --- |
| 模型主输出是什么 | 当前 3D 状态 `p,v,covariance`，不是直接落点 | 系统目标是 100fps 当前 3D 信息；落点是下游物理 rollout 的输出 |
| 是否直接训练落点 | 只做辅助 head 或评估项 | 没有完整真实轨迹和准确落点标签时，直接落点网络容易学习相机/检测偏差 |
| C++ 是否选择深度方法 | 训练口径不使用 C++ 选择；实时兼容输出只从 P0/fallback allow-list 选择 | 实时管线会为 ROS、日志和旧轨迹预测写 `z_stereo/z`，但 P1/实验候选只写原始字段，可靠性模型只读取原始候选字段 |
| 物理约束放哪里 | 训练 loss、离线 smoother、在线 rollout 都使用 | 物理约束提供可解释外推和无标签监督，但不能替代真实距离弱标签 |
| 没有完整真值如何训练 | 静态已知距离 + 固定平面运动 + 落地区域弱标签 + 无标签自监督 | 先解决绝对尺度和 bias，再学运动状态可靠性 |
| 100fps 实时如何部署 | tiny reliability model + hand-written EKF/UKF + 物理落点 | 离线 factor graph / smoother 可以重，实时必须轻 |

待确认后再实现的边界:

1. `z_stereo` / `z` 只作为 legacy baseline，不进入候选方法输入或标签。
2. 训练第一阶段只做候选深度统计和离线评估，不急着把 neural head 接进 C++。
3. 实时 C++ 后续最多接入小型 reliability head；离线 Python 负责重模型、画图、smoother 和训练。
4. 落点输出由当前状态经过物理积分得到；direct landing head 只作为辅助监督和误差校正。

## 核心结论

模型不应该直接替代整条物理轨迹链路，也不应该直接从候选深度一步输出落点。更合理的分层是:

1. 实时管线计算候选观测:
   - 每帧输出 P0/P1 深度候选、disparity、support、confidence、bbox/circle、pair gate、水印。
   - C++ 当前仍用 `selectLegacyDepthOutputCandidate()` 写 legacy `z_stereo/z`，但训练和评估候选不读取这个结果。
2. 测量可靠性模型:
   - 输入当前帧和历史窗口特征。
   - 输出每个候选方法的 `bias`, `sigma`, `outlier probability`。
   - 输出当前时刻更可信的 3D 观测和观测协方差。
3. 状态估计器:
   - 使用 EKF/UKF/KalmanNet 类结构融合历史观测，输出当前状态:
     `p=(x,y,z), v=(vx,vy,vz)`，可选 `drag/spin residual`。
4. 落点预测器:
   - 使用当前状态和物理模型积分到地面/网口/落点平面。
   - 直接落点网络只能作为辅助 head 或校正项，不能作为唯一输出。

这样做的原因:

- 当前无法获取完整真实轨迹。直接训练落点会缺标签，且很容易把相机/检测偏差学进模型。
- 100fps 实时系统需要每帧输出当前 3D 信息，落点只是下游任务。
- 物理积分可以提供可解释性和外推能力；模型主要负责“不确定度、偏置、异常点和残差”。

## 架构合理性 review

当前方向是合理的，但必须把“观测生成”和“观测解释”分开:

```text
C++ 实时观测生成:
  bbox/circle/ROI/P0/P1 深度候选 + pair gate + timing + support

模型观测解释:
  每个候选的 bias/sigma/outlier + fused measurement covariance

状态/落点:
  EKF/UKF/filter 得到当前 p,v
  physics rollout 得到 landing
```

这样做能避免两个常见错误:

- 错误 1: 在 C++ 里用规则切换“当前最可信深度”。这会把手写经验固化进训练标签，模型只能学习已有规则。
- 错误 2: 直接从历史候选输出落点。落点标签稀疏且不稳定，网络会绕过当前 3D 状态，后续难以诊断误差来源。

当前代码主要风险:

- `trajectory_fusion/dataset.py` 的 `METHOD_COLUMNS` 已不含 legacy `z_stereo/z`，`legacy_feature_names()` 当前也不读取在线 `x/y/z/vx/vy/vz` 状态字段。严格验证“只让模型从原始候选决定深度”时，需要继续防止后续特征把 legacy 在线选择结果重新引入。
- 训练脚本目前偏自监督，缺已知距离/固定平面/落地区域弱标签入口。
- recorder 若只保存 online filtered 结果，会让离线训练缺少原始多候选观测；训练采集必须保存完整候选字段。
- 当前模型偏 depth-only；最终要把 2D anchor、disparity、camera geometry 和 state covariance 纳入评估。

## 架构备选方案对比

| 方案 | 优点 | 主要问题 | 结论 |
| --- | --- | --- | --- |
| 直接从历史候选输出落点 | 实现表面最简单，指标直接面向最终任务 | 需要大量准确落点标签；中间 3D 状态不可解释；100fps 当前 3D 输出目标被弱化 | 不做主方案，只可作为辅助 head |
| 多候选深度 -> 当前 3D 状态 -> 物理落点 | 可解释；能利用弱标签；实时可控；便于定位误差来源 | 需要设计可靠性、滤波和评估流程 | 推荐主方案 |
| 离线 factor graph / smoother 直接产轨迹 | 能利用全序列和未来帧，伪真值质量高 | 不能实时；实现较重 | 用于离线标注增强和训练，不进主线程 |
| 只用 hand-tuned EKF | 轻量、可实时、容易部署 | 难学习不同深度候选在速度/距离/光照下的可靠性 | 作为 baseline 和 fallback |
| 端到端图像/视频模型 | 理论上信息最完整 | 训练数据和标注成本高；NX 100fps 风险大；难解释 | 当前不适合主线 |
| 只用物理抛物线拟合 | 数据需求低，外推稳定 | 会忽略观测偏置、拖拽、误匹配、遮挡和检测延迟 | 作为约束，不作为完整模型 |

什么时候可以调整结论:

- 如果后续能稳定获得毫米级/厘米级外部真值轨迹，例如 mocap、雷达、多相机高精度标注，可以考虑更强的监督轨迹网络。
- 如果能大量记录真实落点区域并且落点误差是唯一业务目标，可以把 direct landing head 权重提高，但仍建议保留当前 3D 状态输出。
- 如果 NX 上 tiny reliability head 实测超过实时预算，则先退回离线训练 + hand-tuned EKF，只把 learned bias/MAD 表固化为参数。
- 如果球面纹理、自旋或 event camera 观测可用，再考虑显式 Magnus/spin 状态；当前不要强行估自旋。

## 当前实现评审

已有代码:

```text
trajectory_fusion/
  dataset.py
  models.py
  losses.py
  train_reliability.py
  evaluate_fusion.py
```

当前模型:

- `MeasurementReliabilityNet`
  - `Linear + LayerNorm + SiLU + GRU`
  - 输出每个方法的 `log_sigma`, `bias`, `outlier_logit`
- `weighted_depth_consensus`
  - 对多个深度候选做 learned reliability 加权融合
- `TinyCausalFusionHead`
  - 可作为后续实时部署实验头

合理部分:

- 方向正确: 先学测量可靠性，不直接学最终轨迹。
- 输出 `sigma/bias/outlier` 比输出单个深度更适合无完整真值场景。
- Student-t / Huber 风格鲁棒 loss 适合排球检测中的离群点。

问题:

- 目前主要是 depth-only。最终 3D 轨迹需要同时处理 anchor、x/y/z 和状态协方差。
- 纯自监督只能学习候选之间的一致性，不能修正所有方法共有的系统偏差。
- 如果训练数据里没有已知距离/已知平面/落地事件，模型会缺少绝对尺度约束。
- `z_stereo`/`z` 不能作为训练方法或标签；它们是代码旧选择结果，会造成标签泄漏。
- 需要显式加入运动状态特征，让模型学会静止、匀速、快速运动、遮挡/fallback 时的可靠性变化。

## 输出定义

推荐模型输出分两层。

### Measurement head

每帧、每个候选方法输出:

```text
bias_i
log_sigma_i
outlier_logit_i
```

其中候选方法先限制为:

```text
P0:
z_bbox_center
z_circle_center
z_roi_edge_centroid
z_roi_radial_center
z_roi_edge_pair_center

P1:
z_roi_multi_point
z_roi_center_patch
```

### State head / filter

输出当前状态:

```text
x, y, z
vx, vy, vz
covariance or confidence
```

推荐先不要让神经网络直接输出完整状态。第一版应使用:

```text
候选深度可靠性模型 -> fused measurement -> EKF/UKF/RTS smoother -> state
```

后续如果 NX 实时预算允许，再考虑 KalmanNet 风格网络，让模型学习滤波增益或模型误差。

### Landing predictor

落点由物理积分得到:

```text
state_t + gravity + drag + optional Magnus/residual -> landing
```

直接输出落点的 neural head 可以作为辅助任务:

```text
landing_x_aux, landing_y_aux, landing_t_aux
```

但它不能替代物理积分。它的价值是检测物理模型漏掉的系统项，例如旋转、空气流动、检测延迟。

## 物理约束

排球飞行可以先使用低阶物理模型:

```text
dp/dt = v
dv/dt = g + drag(v) + residual
drag(v) = -k * |v| * v
```

暂不建议第一版显式估计 Magnus/spin。原因是当前没有球自旋观测，直接估 spin 会不可辨识。可以先把 spin/Magnus 作为小残差项或过程噪声；后续如果视频纹理/姿态/击球动作可用，再估计 spin prior。

物理 loss:

```text
L_physics_position =
  ||p[t+1] - p[t] - v[t] * dt||_Huber

L_physics_velocity =
  ||v[t+1] - v[t] - (g + drag(v[t])) * dt||_Huber

L_jerk =
  Huber(second_difference_or_jerk)
```

对于静态和固定距离数据:

```text
L_static_z = |z_model - z_known|
L_plane = distance_to_known_plane
L_no_motion = ||v|| + ||a||
```

对于多候选一致性:

```text
L_measurement =
  StudentT(z_i - bias_i - z_latent, sigma_i) * valid_i * inlier_prob_i
```

对于配对质量:

```text
pair_shifted_iou low -> expected sigma high
pair_epipolar_dy high -> expected sigma high
fallback source -> expected sigma high
```

这些不是硬规则，而是先验/正则。模型仍应从数据中学习在不同速度、位置、距离、光照下哪个候选可靠。

## 无完整真值时的数据集

不能只依赖“正常打球视频”。需要专门录几类弱标签数据。

### A. 静态已知距离

目的: 绝对尺度和每个方法 bias。

采集:

- 距离: `2m`, `3m`, `4m`, `5m`, `6m`
- 横向: 中心、左、右
- 高度: 低、中、高
- 每个点 `5-10s`

标签:

```text
known_z
known_region
static=true
```

可训练:

- 每个候选的绝对 bias
- 画面位置导致的系统偏差
- bbox/circle/edge 方法在不同球尺寸下的可靠性

### B. 固定距离平面内运动

目的: 让模型知道运动状态会改变可靠性。

采集:

- 球大致保持固定距离相机，例如 `3m`
- 左右移动、上下移动、来回上下拍球
- 尽量覆盖慢速、快速、模糊、遮挡

标签:

```text
known_z or known_z_range
motion=true
plane_constraint=true
```

可训练:

- 快速运动时 bbox/circle/edge/subpixel 的可靠性变化
- motion feature 与候选 sigma/outlier 的关系
- 运动模糊下哪种候选更稳定

### C. 已知起落区域动态球

目的: 训练物理外推和落点评估。

采集:

- 从已知位置抛球/发球
- 标记落地区域或落地帧
- 可用场地线、地面点、网前区域作为弱标签

标签:

```text
landing_region
landing_frame
optional start_region
```

可训练:

- 物理积分残差
- 落点辅助 head
- 异常轨迹检测

### D. 正常排球运动

目的: 泛化和稳定性验证。

标签可以很弱:

```text
valid_track
no_label
```

用于:

- 自监督候选一致性
- 物理 residual
- 长序列稳定性
- no-output / fallback 统计

必须记录的字段类别:

- 所有 P0/P1 深度候选: `z_bbox_center`, `z_circle_center`, `z_roi_*`。
- 每个候选的 disparity、support、std、confidence。
- 左右配对质量: `pair_initial_disparity`, `pair_epipolar_dy`, `pair_shifted_iou`, `pair_size_ratio`, `pair_score`。
- 水印和时间: frame counter、trigger index、timestamp delta、实际 `dt`。
- 检测质量: bbox/circle、confidence、ROI pixels、fallback source。

训练候选方法中不要使用:

```text
z_stereo
z
depth_method
stereo_depth_source as label
```

这些字段只能用于诊断/对比，因为它们已经包含旧实时规则选择结果。

## 评估门槛

第一阶段不以“模型 loss 下降”为通过标准，而以可解释指标为准。

### 候选方法统计

静态已知距离:

```text
per_method_bias_m
per_method_mad_m
per_method_valid_rate
per_region_bias_m
per_distance_bias_m
```

固定平面运动:

```text
motion_valid_rate
motion_bias_or_plane_error_m
depth_mad_m
frame_to_frame_jitter_m
pair_gate_fail_rate
```

### 离线 smoother 质量

```text
known_z_error_m
reprojection_error_px
physics_rollout_residual_m
outlier_rate_by_method
outlier_rate_by_pair_gate_bucket
```

### 实时部署质量

```text
100fps sustained
watermark interval stable
candidate field valid rate
reliability model latency
EKF update latency
landing uncertainty stable
```

### 模型必须超过的 baseline

至少比较:

```text
P0 median
best single P0 method by static MAD
legacy z_stereo/z
hand-tuned EKF
learned reliability + EKF
```

如果 learned reliability 只是在无标签片段上 loss 更低，但静态已知距离 bias/MAD 没有改善，不能认为模型有效。

## 训练阶段

### Stage 1: 候选方法标定

输入: 静态已知距离。

输出:

- 每个候选方法的 bias/MAD 表
- 不同距离、画面区域、球大小下的误差统计

不要训练复杂模型，先做统计报告。

### Stage 2: 可靠性模型

输入:

- 静态已知距离
- 固定距离平面内运动
- 正常动态片段

输出:

- 每个方法的 `sigma/bias/outlier`
- fused depth measurement

Loss:

```text
L = L_known_z
  + L_measurement_consistency
  + L_pair_quality_regularizer
  + L_motion_conditioned_uncertainty
  + L_uncertainty_regularizer
```

其中 `L_known_z` 只在有弱/强标签的片段上启用。

### Stage 3: 状态滤波

输入:

- fused 3D observation
- observation covariance

输出:

- current state: `p, v, covariance`

方案:

- 第一版: EKF/UKF 或当前 Kalman 的升级版。
- 第二版: KalmanNet 类模型，学习滤波增益/模型误差。

### Stage 4: 落点预测

输入:

- current state
- drag 参数
- ground/net plane

输出:

- `landing_x`
- `landing_y`
- `landing_t`
- confidence / valid flag

训练:

- 有落点区域标签时做 supervised landing loss。
- 无标签时只做物理积分，不训练落点 head。

## 更完整的轨迹生成路线

推荐把训练和实时部署拆成两条链路。

### 离线链路: 生成高质量伪真值

离线链路不受 100fps 限制，目标是把连续录制片段变成尽可能可信的状态序列:

```text
CSV 多候选观测 + 彩色/灰度帧 + 弱标签
  -> robust smoother / factor graph
  -> p(t), v(t), covariance, per-frame inlier mask
  -> 训练 measurement reliability model
```

离线优化变量:

```text
p_k = (x_k, y_k, z_k)
v_k = (vx_k, vy_k, vz_k)
optional: drag_k or global drag coefficient
optional: per-method bias_i, sigma_i
```

离线残差:

```text
measurement residual:
  z_method_i[k] - z_state[k]

reprojection residual:
  project(p_k, left/right camera) - observed center/edge/circle

physics residual:
  p[k+1] - p[k] - v[k] * dt
  v[k+1] - v[k] - (g + drag(v[k])) * dt

weak label residual:
  static known_z
  fixed-z plane
  landing region / landing frame
  court/net plane crossing
```

误匹配和坏帧不要硬删除，离线优化里用 Student-t、Huber、Cauchy 或 switchable weight 降权。输出的 inlier/outlier mask 反过来可以作为训练模型的伪标签。

### 在线链路: 只做轻量状态估计

在线链路必须保持 100fps:

```text
P0/P1 深度候选字段
  -> tiny measurement reliability model
  -> fused measurement + covariance
  -> EKF/UKF or hand-written constant-accel filter
  -> physical rollout landing prediction
```

C++ 实时管线仍然计算并写出全部候选字段；同时为了兼容 ROS、日志和旧轨迹预测，会用 `buildDepthCandidateObservations()` + `selectLegacyDepthOutputCandidate()` 写出 legacy `z_stereo/z`。这个兼容选择只允许 P0 几何/bbox 和退化 fallback，不让 P1/实验候选抢占在线主深度；它不作为训练标签，也不作为候选方法输入。模型可以输出各方法权重和不确定度；滤波器用这些不确定度融合观测。

## 物理模型细化

第一版建议使用可辨识的低阶模型:

```text
state = [x, y, z, vx, vy, vz]

dp/dt = v
dv/dt = g - c_d * |v| * v + a_res
```

其中 `c_d` 可以先按片段或全局标定，不要逐帧自由估计。原因是当前没有可靠自旋观测，逐帧估计 `drag/spin` 会不可辨识。

第二版再考虑 Magnus/spin:

```text
dv/dt = g - c_d * |v| * v + c_m * (omega x v)
```

但这需要额外观测:

- 球面纹理/Logo 旋转；
- event camera 或更高快门/更大球面分辨率；
- 或者只把 `c_m * (omega x v)` 当作小残差，不直接解释为真实自旋。

落点不需要神经网络直接猜。更稳的做法是:

```text
current state + covariance
  -> RK4 / semi-implicit Euler rollout
  -> 与地面/目标平面求交
  -> landing mean + confidence ellipse
```

如果有落地区域标签，直接落点 head 可以作为辅助训练项，用来修正系统偏差；不能替代当前 3D 状态输出。

## 有限标签下的 loss 设计

无完整 3D 真值时，训练必须混合强约束、弱标签和自监督。

### 观测可靠性

```text
L_measure =
  StudentT(z_i - bias_i - z_latent, sigma_i)
  * valid_i
  * inlier_weight_i
```

`sigma_i` 不只是方法常数，要依赖:

- 球大小、距离、画面区域；
- `pair_epipolar_dy`, `pair_shifted_iou`, `pair_size_ratio`;
- support 点数、MAD、RANSAC inlier rate;
- 速度、模糊、水印间隔、曝光增益。

### 物理一致性

```text
L_one_step =
  Huber(p[k+1] - p[k] - v[k] * dt)
  + Huber(v[k+1] - v[k] - (g + drag(v[k])) * dt)

L_rollout =
  Huber(p[k+h] - rollout(p[k], v[k], h * dt))
```

`L_rollout` 比单步差分更重要，因为落点预测关心长期外推。建议训练时随机取 `h=3,5,10,20` 帧。

### 几何重投影

在没有 3D 标签时，重投影是最便宜的几何监督:

```text
state p_k
  -> project to rectified left/right
  -> compare with bbox center, circle center, edge point, ROI matched point
```

注意: 重投影 loss 只能证明“和 2D 检测一致”，不能独立解决绝对深度尺度；仍然需要静态已知距离和固定平面数据。

### 弱标签

推荐标签 sidecar:

```yaml
clip_id: ...
known_z: 4.0
known_z_tolerance: 0.08
static: true
motion_type: vertical_bounce
landing_region: left_back_zone
landing_frame: 1830
start_region: center_3m
notes: ...
```

对应 loss:

```text
L_known_z    = Huber(z_state - known_z)
L_plane      = Huber(distance_to_known_plane)
L_static     = ||v|| + ||a||
L_landing    = distance_to_landing_region_or_plane_crossing
L_no_teleport = Huber(frame_to_frame_speed_limit_violation)
```

## 建议优先实现的训练实验

1. 先做离线统计，不训练网络:
   - 每个候选方法在 `2m-6m` 静态已知距离下的 bias/MAD。
   - 按画面九宫格、球直径、pair gate 分组。
2. 做 robust smoother 伪真值:
   - 输入所有候选深度，不输入 `z_stereo` / `z`。
   - 输出每帧 state 和每个候选的 residual/outlier。
3. 训练 reliability model:
   - 输入候选字段和历史窗口。
   - 输出 `bias/sigma/outlier`。
   - 先只学 depth，再扩展到完整 3D state covariance。
4. 对比三个基线:
   - P0 median。
   - hand-tuned EKF。
   - learned reliability + EKF。
5. 最后才加 landing loss:
   - 有落点区域标签才训练。
   - 否则只做物理 rollout 评估，不用伪落点当真值。

## 同意后的最小实现顺序

确认方案后，建议按这个顺序改代码:

1. `dataset.py`
   - 保持 `METHOD_COLUMNS` 只包含原始候选，不加入 `z_stereo` / `z`。
   - 加入 `pair_*`, support, confidence, timing, bbox/circle 几何字段。
   - 为严格候选-only 实验保持不读取 `z_stereo/z` 和在线 `x/y/z/v*` legacy 状态字段。
   - 支持 `metadata.yaml` 弱标签读取。
2. 新增离线统计脚本:
   - 输出每个候选方法的 known-distance bias/MAD。
   - 按距离、画面区域、运动/静止、pair gate bucket 分组。
3. 新增 robust smoother 原型:
   - 第一版可用 Python least-squares/RTS smoother。
   - 后续再切 GTSAM/Ceres。
4. 扩展训练 loss:
   - 有标签片段启用 `known_z`, `plane`, `landing_region`。

- 直接训练落点模型。

## 可参考实现

### learned filter / state estimator

- KalmanNet 官方代码: https://github.com/KalmanNet/KalmanNet_TSP
  - 可借鉴: RNN 学习 Kalman gain / correction 的结构。
  - 不建议直接照搬: 它的示例系统与当前多候选深度输入不同，需要改成候选观测 + 物理状态。
- torchfilter: https://github.com/stanford-iprl-lab/torchfilter
  - 可借鉴: PyTorch 中的 EKF/UKF/PF 接口、可微滤波结构、自动 Jacobian。
  - 适合离线训练/验证，不一定直接上 NX 实时。
- RTSNet: https://arxiv.org/abs/2110.04717
  - 可借鉴: learned RTS smoother，用于离线把整段轨迹平滑成伪真值。
  - 不建议第一版上实时，因为 smoothing 依赖未来帧。

### offline smoother / factor graph

- GTSAM: https://github.com/borglab/gtsam
  - 可借鉴: 用 factor graph 把每帧状态、物理模型、观测残差、弱标签约束统一优化。
  - 适合离线标注增强，不适合放进 100fps 主线程。
- Ceres Solver: https://github.com/ceres-solver/ceres-solver
  - 可借鉴: 直接写 nonlinear least squares，快速实现 batch trajectory smoothing。
- pykalman: https://github.com/pykalman/pykalman
  - 可借鉴: Kalman smoother 和 EM 估计 Q/R，用于离线初步估噪声。
- FilterPy: https://github.com/rlabbe/filterpy
  - 可借鉴: EKF/UKF/RTS/least-squares 的清晰 Python 参考实现。

### 可微几何 / reprojection loss

- Kornia: https://github.com/kornia/kornia
  - 可借鉴: PyTorch 可微几何、epipolar/camera geometry 工具。
- nvTorchCam: https://github.com/NVlabs/nvTorchCam
  - 可借鉴: camera model 独立的可微投影/反投影。
- torchdiffeq: https://github.com/rtqichen/torchdiffeq
  - 可借鉴: 可微 ODE rollout，可用于训练 drag 参数和落点辅助 loss。
- torchcde: https://github.com/patrick-kidger/torchcde
  - 可借鉴: 不规则采样序列建模。若水印间隔抖动明显，可以作为离线实验参考。

后续如果要加 reprojection loss，可以用:

```text
fused_3d_state -> project to left/right rectified image
              -> compare bbox/circle/center/edge observations
```

这能在没有完整 3D 真值时增加几何约束，但第一版不必强依赖这些库。

### 物理约束 tracking

- Physics-Informed Tracking (PIT): https://arxiv.org/abs/2604.16895
  - 参考点: 无标签视频中用 differentiable physics 约束 landmark trajectory。
  - 对本项目的启发: 可以用物理 rollout 将连续候选点约束成合理轨迹，而不是只做逐帧点估计。
- Multi-Camera Ball Localization with Factor Graphs: https://arxiv.org/abs/2401.17185
  - 参考点: 多相机异步球定位、factor graph、速度/自旋隐藏状态、早期观测不足时使用先验。
  - 对本项目的启发: 用 factor graph / smoother 做离线标注增强，再训练实时小模型。
- Ball Trajectory Inference from Multi-Agent Sports Contexts: https://arxiv.org/abs/2306.08206
  - 参考点: 缺失球轨迹时用上下文和 reality loss 约束输出物理合理。
  - 对本项目的启发: 可用 reality/physics loss 约束缺标签片段，但不能替代已知距离数据。
- Differentiability in Unrolled Training of Neural Physics Simulators: https://arxiv.org/abs/2402.12971
  - 参考点: rollout/unroll 训练会影响长期预测误差。
  - 对本项目的启发: 落点预测相关 loss 不能只做 one-step，应增加短窗口 rollout 验证。
- Robust Student-t smoothers: https://arxiv.org/abs/1001.3907
  - 参考点: Student-t 观测噪声对极端 outlier 更稳。
  - 对本项目的启发: 左右误匹配、YOLO 退行、ROI 错点都应该进入 robust loss，而不是简单删除。
- Robust Bayesian filtering with Student-t: https://arxiv.org/abs/1703.02428
  - 参考点: Student-t filter/smoother 形式接近 KF/RTS，工程迁移成本低。
- Generalised Bayes robust Kalman filtering: https://arxiv.org/abs/2405.05646
  - 参考点: 面向在线 outlier 的低成本鲁棒 Kalman 更新。

### ball trajectory / sports trajectory

- Physically grounded synthetic-to-real table tennis trajectory/spin: https://arxiv.org/abs/2504.19863
  - 参考点: 没有真实 3D/spin 标签时，用物理正确的合成数据训练后端，再用真实 2D/观测适配。
  - 对本项目的启发: 可以用仿真生成带 drag/不同速度/不同相机位置的轨迹，训练模型学会物理残差和缺测鲁棒性。
- Uplifting Table Tennis: https://arxiv.org/abs/2511.20250
  - 参考点: 前端感知和后端 2D-to-3D uplifting 分离，后端只用物理合成数据训练，并专门处理缺失检测和帧率变化。
  - 对本项目的启发: 你的 C++ 候选深度就是前端观测，后端训练应模拟缺失、错配、帧率抖动。
- Event camera ball spin estimation: https://arxiv.org/abs/2404.09870
  - 参考点: 自旋很难从普通低帧率/模糊图像直接观测，event camera 更适合。
  - 对本项目的启发: 现在不要把 spin 当成必须估计的显式变量；先用 residual/process noise 覆盖。
- Event-based real-time spin system: https://arxiv.org/abs/2606.26780
  - 参考点: 真实自旋估计需要高时间分辨率、主动视线控制和专门传感器。
  - 对本项目的启发: NX 双目 100fps 下更现实的是估 3D 状态和落点置信度，而不是解释真实自旋。

## 方案判断

建议采用:

```text
候选深度字段 -> 可靠性模型 -> 当前 3D 状态估计 -> 物理落点预测
```

不建议采用:

```text
候选深度字段 + 历史轨迹 -> 直接输出落点
```

直接落点可以作为辅助 head，但不能做主输出。主输出必须是当前 3D 状态和不确定度，因为系统目标是稳定 100fps 3D 信息获取，落点只是后级任务。
