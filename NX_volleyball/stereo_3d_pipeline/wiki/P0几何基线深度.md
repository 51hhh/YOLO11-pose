# P0 几何基线深度

最后核对: 2026-07-04

本页记录当前 100fps 正式采集的 P0 几何基线。P0 的定位是实时稳定候选和训练主字段；它们允许进入 legacy `z_stereo/obj.z` 兼容输出，也是明天实机录制优先验收的深度来源。

## 当前结论

P0 不是关键点匹配路线，而是左右 YOLO direct pair 之后的几何测距集合。它依赖 bbox、圆/球心拟合和 ROI 边缘几何，不依赖排球表面纹理，因此比当前 XFeat/GFTT-LK 更适合做 100fps 基线。

当前正式记录字段:

| 类别 | 字段 | 方法 | 当前角色 |
|---|---|---|---|
| bbox | `z_bbox_center` | 左右 YOLO bbox 中心视差 | 最轻量后备和训练 baseline |
| bbox edge | `z_bbox_left_edge`, `z_bbox_right_edge` | 左右 bbox 左/右边缘视差 | 诊断 bbox 偏移和宽度一致性 |
| circle | `z_circle_center` | ROI 圆心/球心拟合中心视差 | P0 主观测优先项 |
| circle edge | `z_circle_left_edge`, `z_circle_right_edge` | 拟合圆左右边缘视差 | 诊断圆拟合偏移 |
| ROI edge | `z_roi_edge_centroid` | ROI 边缘梯度质心视差 | 稳定几何候选 |
| ROI radial | `z_roi_radial_center` | 径向梯度中心视差 | 稳定几何候选 |
| ROI pair | `z_roi_edge_pair_center` | 左右 ROI 边缘成对中心视差 | 稳定几何候选 |

## 左右视差怎么来

P0 先由左右 YOLO 通过 epipolar、尺寸和 shifted IoU gate 形成 direct pair。每个字段都在 rectified 坐标里计算左点 `x_left` 和右点 `x_right`，视差统一为:

```text
disparity = x_left - x_right
z = focal * baseline / disparity
```

bbox 字段直接用 YOLO 框中心或边缘；circle 字段使用 ROI 内圆/球心拟合结果；ROI edge/radial/pair 字段使用球 ROI 内的边缘梯度或成对边缘中心。

## 配置入口

```yaml
detector:
  dual_yolo:
    depth_modes:
      bbox_pair: true
      bbox_edges: true
      circle_center: true
      circle_edges: true
      roi_edge_centroid: true
      roi_radial_center: true
      roi_edge_pair_center: true
```

## 已知表现

历史 isolated 和 P0+P1 联合录制显示 P0 可维持 100fps。代表结果见 [稳定100fps深度方法与训练入口](稳定100fps深度方法与训练入口.md):

| 字段 | FPS | 有效率 | 中位深度 | MAD |
|---|---:|---:|---:|---:|
| `z_bbox_center` | `100.0` | `611/611` | `3.6129m` | `0.0008m` |
| `z_circle_center` | `99.9` | `615/615` | `3.6139m` | `0.0010m` |
| `z_roi_edge_centroid` | `99.3` | `637/637` | `3.6091m` | `0.0013m` |
| `z_roi_radial_center` | `100.3` | `616/616` | `3.6138m` | `0.0009m` |
| `z_roi_edge_pair_center` | `100.1` | `641/641` | `3.6108m` | `0.0000m` |

## 质量风险

- bbox 和 circle 字段高度相关，训练时不能把多个近似重复字段当作完全独立证据。
- bbox 会受 YOLO 框偏心影响；circle 会受 ROI 边缘质量和遮挡影响。
- P0 几何适合做稳定 baseline，但不能解决动态遮挡、旋转纹理误匹配或真实球心偏置的全部问题。
- `z_stereo` / `obj.z` 是兼容输出，不是一个独立测距方法；训练必须读原始 `z_*` 字段。

## 验收方式

录制后用 `trajectory_fusion/check_dataset.py` 检查:

- P0 字段是否齐全。
- `frame_counter_delta` / `frame_number_delta` 是否异常。
- 每个 P0 字段的 valid、median、MAD 和 `known_z` bias。
- direct pair 与 fallback 分组后的稳定性。

P0 通过后，再看 [P1训练候选深度](P1训练候选深度.md) 和 [GFTT-LK诊断Sidecar](GFTT-LK诊断Sidecar.md) 是否能提供额外可学习信息。
