# 深度候选字段参考

最后核对: 2026-07-05

本页是字段参考，不判断算法是否进入默认配置。当前选用结论见 [稳定100fps深度方法与训练入口](稳定100fps深度方法与训练入口.md)，算法分类和左右视差语义见 [深度算法分类总览](深度算法分类总览.md)，深度相关页面总入口见 [深度算法导航](深度算法导航.md)。字段单位为米，视差单位为像素；无效值通常为 `-1`。

## 单目

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_mono` | bbox 宽度 + 已知球直径反推深度 | 远距离或单侧检测 fallback，可受检测框尺度影响 |

## bbox 双目

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_bbox_center` | 左右 bbox 中心视差三角化 | 最轻量，受 bbox 偏心影响 |
| `z_bbox_left_edge` | 左右 bbox 左边缘视差 | 与中心高度相关，冗余但可诊断 |
| `z_bbox_right_edge` | 左右 bbox 右边缘视差 | 与左边缘组成边缘一致性检查 |

## 圆拟合双目

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_circle_center` | ROI 圆拟合中心视差 | 依赖 `center_refine` |
| `z_circle_left_edge` | 圆左边缘视差 | 边缘候选 |
| `z_circle_right_edge` | 圆右边缘视差 | 边缘候选 |

## ROI 几何

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_edge_centroid` | ROI 边缘梯度质心视差 | 轻量，适合实时默认开启 |
| `z_roi_radial_center` | 径向梯度中心视差 | 对球形边缘较友好 |
| `z_roi_edge_pair_center` | 左右边缘成对中心视差 | 用边缘配对抑制单侧偏移 |

## ROI sparse/patch

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_corner_points` | ROI 角点采样 + 匹配 | 需连续帧验证稳定性 |
| `z_roi_texture_points` | 纹理/梯度采样 + 匹配 | 弱纹理球体上可能有效点少 |
| `z_roi_binary_points` | 二值特征采样 + 匹配 | 轻量实验模式 |
| `z_roi_center_patch` | 中心 patch ZNCC 视差 | P1 训练候选；记录原始字段，不参与 legacy `z_stereo` |
| `z_roi_multi_point` | 多点 ZNCC 亚像素视差 | P1 训练候选；记录原始字段，不参与 legacy `z_stereo` |

## 描述子和颜色区域

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_orb_points` | true OpenCV CUDA ORB + CUDA matcher | 使用校正 gray GPU；keypoint/match 元数据回 CPU gate；P2 默认关闭，已有 `fast48`/宽 y 参数复测 case |
| `z_roi_brisk_points` | OpenCV CPU BRISK 实验候选 | 当前项目实现是 CPU debug；算法概念需找到真实 GPU 后端后再进 P2 |
| `z_roi_akaze_points` | OpenCV CPU AKAZE 实验候选 | 当前项目实现是 CPU debug；算法概念需找到真实 GPU 后端后再进 P2 |
| `z_roi_sift_points` | OpenCV CPU SIFT 实验候选 | 当前项目实现是 CPU debug；CUDA-SIFT 本轮不做，不能用 CPU SIFT 代替实时 GPU 结论 |
| `z_roi_iou_region_color_patch` | 彩色区域 IoU + patch | 使用校正 BGR GPU；2026-07-04 artifact 显示重复纹理区域错配，已退出默认配置 |
| `z_roi_patch_iou_color_edge` | 彩色边缘 IoU + patch | 使用校正 BGR GPU；2026-07-04 artifact 显示 stripe/edge 系统性错配，已退出默认配置 |
| `z_roi_cuda_template_match` | 自研 CUDA Template/NCC 小 ROI 极线匹配 | P2 默认关闭；只记录原始字段，不参与 legacy `z_stereo`；OpenCV CUDA 版本仅作为 `STEREO_CUDA_TEMPLATE_BACKEND=opencv` baseline |
| `z_roi_cuda_stereo_bm` | OpenCV CUDA StereoBM 小 ROI dense disparity | P2 默认关闭；局部 `64` disparity，已做 isolated 复测但当前无有效深度 |
| `z_roi_cuda_stereo_sgm` | OpenCV CUDA StereoSGM 小 ROI dense disparity | 字段保留；联合运行长尾和覆盖率不满足 100fps，已退出默认配置 |
| `z_roi_ring_edge_profile` | 自研 CUDA ring/edge profile 小范围极线匹配 | P2 默认关闭；已接入实时字段和 diagnostic lane，待 NX 有球 isolated 实测 |

## Diagnostic sidecar 候选

这些字段不由 `Object3D` / `TrajectoryRecorder` 主 CSV 直接写出，而是由同名前缀 `*.p2_diagnostic.csv` 记录，再由 `trajectory_fusion/dataset.py` 按 `frame_id + mode` 合并为训练候选。没有 sidecar 时保持无效。

| 字段 | sidecar mode | 原理 | 备注 |
|---|---|---|---|
| `z_roi_cuda_template_match` | `cuda_template` | 自研 CUDA Template/NCC 小 ROI 极线匹配 | 当前 P1 sidecar 训练候选；也有主 CSV 同名字段，训练 loader 以 sidecar 合并补齐 |
| `z_roi_neural_xfeat` | `neural_xfeat` | XFeat TensorRT keypoint/descriptor mutual match | 当前 P1 sidecar 训练候选；有效率仍需优化 |
| `z_roi_neural_superpoint` | `neural_superpoint` | SuperPoint TensorRT keypoint/descriptor mutual match | 当前 P1 sidecar 训练候选；覆盖率高但有系统差 |
| `z_roi_opencv_cuda_gftt_lk` | `opencv_cuda_gftt_lk` | OpenCV CUDA GFTT/Harris + SparsePyrLK | 当前默认关闭；只作 A/B 观察项，不回写在线主深度 |
| `z_roi_vpi_template_match` | `vpi_template_match` | VPI CUDA TemplateMatching 小 ROI 极线 NCC | 字段保留；当前默认去 VPI，以避免联合运行尾延迟 |
| `z_roi_vpi_orb` | `vpi_orb` | VPI CUDA ORB/BRIEF + BruteForceMatcher | 字段保留；当前默认去 VPI，历史有效率低 |

## 神经特征

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_neural_xfeat` | XFeat TensorRT 神经特征匹配输出点对 | 当前 P1 sidecar 训练候选；sidecar mode 为 `neural_xfeat` |
| `z_roi_neural_superpoint` | SuperPoint TensorRT 神经特征匹配输出点对 | 当前 P1 sidecar 训练候选；sidecar mode 为 `neural_superpoint` |
| `z_roi_neural_feature` | legacy TensorRT 神经特征兼容字段 | split 模式下不再作为唯一神经字段，保留用于旧 CSV/旧代码兼容 |

## Fallback

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_fallback_epipolar` | 单侧漏检时极线搜索 | 可用于短时缺右框 |
| `z_fallback` | fallback 兼容汇总 | 可能来自 epipolar、template 或 feature fallback |
| `z_fallback_template` | 极线模板搜索 | CPU fallback，100fps 默认关闭 |
| `z_fallback_feature_points` | fallback 特征点匹配 | CPU fallback，100fps 默认关闭 |

`z_fallback` 是兼容聚合字段，优先看显式 `z_fallback_epipolar`、`z_fallback_template`、`z_fallback_feature_points`。旧 CSV 区分 fallback 类型时再看 [YOLO左右匹配分支与字段语义](YOLO左右匹配分支与字段语义.md) 中的 `stereo_match_source` 和 circle source 组合。

## `z_stereo` 兼容选择

`z_stereo` 不是独立取点方法。当前实时管线会先把所有候选放入 `buildDepthCandidateObservations()`，再用 `selectLegacyDepthOutputCandidate()` 从 P0 几何/bbox 和退化 fallback allow-list 中选择第一个可用候选，写入 `z_stereo`、`obj.z`、`raw_z` 和 `stereo_depth_source`。这个结果用于 ROS、日志和旧轨迹预测兼容，不作为训练候选或标签。

完整候选插入顺序:

```text
roi_multi_point
fallback_feature_points
fallback_template
fallback_epipolar 或 circle_center
roi_center_patch
roi_iou_region_color_patch
roi_patch_iou_color_edge
roi_cuda_template_match
roi_cuda_stereo_bm
roi_cuda_stereo_sgm
roi_ring_edge_profile
roi_neural_feature
roi_neural_xfeat
roi_neural_superpoint
roi_corner_points
roi_texture_points
roi_binary_points
roi_orb_points
roi_brisk_points
roi_akaze_points
roi_sift_points
roi_radial_center
roi_edge_pair_center
roi_edge_centroid
circle_left_edge / circle_right_edge
bbox_center
bbox_edges / bbox_left_edge / bbox_right_edge
```

`buildDepthCandidateObservations()` 仍会写出所有原始候选字段；legacy 输出选择会跳过 `roi_multi_point`、`roi_center_patch`、颜色、OpenCV CUDA P2、ring/edge profile、神经和传统特征候选。P1/P2 字段继续进入 CSV/训练数据，但不会抢占在线 `z_stereo` / `obj.z`。

当前测试配置已对齐到去 SGM、去 VPI、去 color/color-edge；NCC/XFeat/SuperPoint 通过 `*.p2_diagnostic.csv` 合并为 P1 训练候选。解释训练结果时应看各个 `z_*` 原始字段，不要用 `z_stereo` 反推“模型选择了哪个方法”。左右 YOLO direct pair 与单侧 fallback 的字段落点见 [YOLO左右匹配分支与字段语义](YOLO左右匹配分支与字段语义.md)。

## 兼容别名

`Object3D` 和 `TrajectoryRecorder` 仍写出以下旧字段，便于兼容旧 CSV 和旧训练脚本。它们不是新增取点方法。

| 字段 | 对应当前字段 |
|---|---|
| `z_yolo_bbox_pair` | `z_bbox_center` |
| `z_circle` | `z_circle_center` |
| `z_subpixel` | `z_roi_multi_point` |
| `disparity_yolo` | `disparity_bbox_center` |
| `disparity_circle` | `disparity_circle_center` |
| `disparity_subpixel` | `disparity_roi_multi_point` |

## 使用原则

- 训练可靠性网络时，应按类别分组看贡献，避免 bbox/circle 多个高度相关字段造成过拟合。
- `METHOD_COLUMNS` 应只包含原始候选深度字段；`z_stereo`、`z`、`depth_method` 和 `stereo_depth_source` 只用于诊断、分组或 baseline。
- 实时 100fps 默认只启用轻量稳定候选。
- 重候选必须通过 baseline clip 连续帧指标后再进入主配置。
- `stereo_depth_source` 只能表示最终选中的主观测来源，不能替代所有候选字段。
- 左右 YOLO/IoU 误匹配排查看 `pair_shifted_iou`, `pair_score`, `pair_bbox_prior_penalty`, `pair_epipolar_dy`, `pair_positive_disparity` 和 `stereo_match_source`。
