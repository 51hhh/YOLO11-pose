# 深度候选字段参考

最后核对: 2026-07-04

本页是字段参考，不判断算法是否进入默认配置。当前选用结论见 [稳定100fps深度方法与训练入口](稳定100fps深度方法与训练入口.md)，深度相关页面总入口见 [深度算法导航](深度算法导航.md)。字段单位为米，视差单位为像素；无效值通常为 `-1`。

## 单目

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_mono` | bbox 宽度 + 已知球直径反推深度 | 远距离或单侧检测 fallback，可受检测框尺度影响 |

## bbox 双目

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_bbox_center` | 左右 bbox 中心视差三角化 | 最轻量，受 bbox 偏心影响 |
| `z_bbox_left_edge` | 左右 bbox 左边缘视差 | 与中心高度相关，冗余但可诊断 |
| `z_bbox_right_edge` | 左右 bbox 右边缘视差 | 与左边缘组成边缘一致性检查 |

## 圆拟合双目

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_circle_center` | ROI 圆拟合中心视差 | 依赖 `center_refine` |
| `z_circle_left_edge` | 圆左边缘视差 | 边缘候选 |
| `z_circle_right_edge` | 圆右边缘视差 | 边缘候选 |

## ROI 几何

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_edge_centroid` | ROI 边缘梯度质心视差 | 轻量，适合实时默认开启 |
| `z_roi_radial_center` | 径向梯度中心视差 | 对球形边缘较友好 |
| `z_roi_edge_pair_center` | 左右边缘成对中心视差 | 用边缘配对抑制单侧偏移 |

## ROI sparse/patch

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_corner_points` | ROI 角点采样 + 匹配 | 需连续帧验证稳定性 |
| `z_roi_texture_points` | 纹理/梯度采样 + 匹配 | 弱纹理球体上可能有效点少 |
| `z_roi_binary_points` | 二值特征采样 + 匹配 | 轻量实验模式 |
| `z_roi_center_patch` | 中心 patch ZNCC 视差 | P1 训练候选；记录原始字段，不参与 legacy `z_stereo` |
| `z_roi_multi_point` | 多点 ZNCC 亚像素视差 | P1 训练候选；记录原始字段，不参与 legacy `z_stereo` |

## 描述子和颜色区域

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_orb_points` | true OpenCV CUDA ORB + CUDA matcher | 使用校正 gray GPU；keypoint/match 元数据回 CPU gate；P2 默认关闭，已有 `fast48`/宽 y 参数复测 case |
| `z_roi_brisk_points` | OpenCV CPU BRISK 实验候选 | 当前项目实现是 CPU debug；算法概念需找到真实 GPU 后端后再进 P2 |
| `z_roi_akaze_points` | OpenCV CPU AKAZE 实验候选 | 当前项目实现是 CPU debug；算法概念需找到真实 GPU 后端后再进 P2 |
| `z_roi_sift_points` | OpenCV CPU SIFT 实验候选 | 当前项目实现是 CPU debug；CUDA-SIFT 第三方依赖可作为 P2 调研方向 |
| `z_roi_iou_region_color_patch` | 彩色区域 IoU + patch | 使用校正 BGR GPU；P2 默认关闭，已准备离线推荐参数和宽搜索复测 case |
| `z_roi_patch_iou_color_edge` | 彩色边缘 IoU + patch | 使用校正 BGR GPU；P2 默认关闭，已准备离线推荐参数和宽搜索复测 case |
| `z_roi_cuda_template_match` | OpenCV CUDA TemplateMatching 小 ROI 极线匹配 | P2 默认关闭；只记录原始字段，不参与 legacy `z_stereo` |
| `z_roi_cuda_stereo_bm` | OpenCV CUDA StereoBM 小 ROI dense disparity | P2 默认关闭；局部 `64` disparity，待 NX isolated 实测 |
| `z_roi_cuda_stereo_sgm` | OpenCV CUDA StereoSGM 小 ROI dense disparity | P2 默认关闭；局部 `64` disparity，待 NX isolated 实测 |

## 神经特征

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_roi_neural_feature` | TensorRT 神经特征匹配输出点对 | 支持 gray 或 BGR engine schema；P2 默认关闭；XFeat/SuperPoint/ALIKED 都按 ROI/top-k/engine case 单项复测，fused `[N,4/5]` 入口优先 |

## Fallback

| 字段 | 原理 | 备注 |
|---|---|---|
| `z_fallback_epipolar` | 单侧漏检时极线搜索 | 可用于短时缺右框 |
| `z_fallback` | fallback 兼容汇总 | 可能来自 epipolar、template 或 feature fallback |
| `z_fallback_template` | 极线模板搜索 | CPU fallback，100fps 默认关闭 |
| `z_fallback_feature_points` | fallback 特征点匹配 | CPU fallback，100fps 默认关闭 |

`z_fallback` 是兼容聚合字段，优先看显式 `z_fallback_epipolar`、`z_fallback_template`、`z_fallback_feature_points`。旧 CSV 区分 fallback 类型时再看 [YOLO左右匹配分支与字段语义](YOLO左右匹配分支与字段语义.md) 中的 `stereo_match_source` 和 circle source 组合。

## `z_stereo` 兼容选择

`z_stereo` 不是独立取点方法。当前实时管线会先把所有候选放入 `buildDepthCandidateObservations()`，再用 `selectLegacyDepthOutputCandidate()` 从 P0 几何/bbox 和退化 fallback allow-list 中选择第一个可用候选，写入 `z_stereo`、`obj.z`、`raw_z` 和 `stereo_depth_source`。这个结果用于 ROS、日志和旧轨迹预测兼容，不作为训练候选或标签。

完整候选插入顺序:

```text
roi_multi_point
fallback_feature_points
fallback_template
fallback_epipolar 或 circle_center
roi_center_patch
roi_iou_region_color_patch
roi_patch_iou_color_edge
roi_cuda_template_match
roi_cuda_stereo_bm
roi_cuda_stereo_sgm
roi_neural_feature
roi_corner_points
roi_texture_points
roi_binary_points
roi_orb_points
roi_brisk_points
roi_akaze_points
roi_sift_points
roi_radial_center
roi_edge_pair_center
roi_edge_centroid
circle_left_edge / circle_right_edge
bbox_center
bbox_edges / bbox_left_edge / bbox_right_edge
```

`buildDepthCandidateObservations()` 仍会写出所有原始候选字段；legacy 输出选择会跳过 `roi_multi_point`、`roi_center_patch`、颜色、OpenCV CUDA P2、神经和传统特征候选。P1/P2 字段继续进入 CSV/训练数据，但不会抢占在线 `z_stereo` / `obj.z`。

默认 `config/pipeline_dual_yolo_roi.yaml` 下，P0 和 P1 候选都会计算并写入 CSV；颜色、神经、OpenCV 特征和 CPU fallback 默认关闭。解释训练结果时应看各个 `z_*` 原始字段，不要用 `z_stereo` 反推“模型选择了哪个方法”。

## 兼容别名

`Object3D` 和 `TrajectoryRecorder` 仍写出以下旧字段，便于兼容旧 CSV 和旧训练脚本。它们不是新增取点方法。

| 字段 | 对应当前字段 |
|---|---|
| `z_yolo_bbox_pair` | `z_bbox_center` |
| `z_circle` | `z_circle_center` |
| `z_subpixel` | `z_roi_multi_point` |
| `disparity_yolo` | `disparity_bbox_center` |
| `disparity_circle` | `disparity_circle_center` |
| `disparity_subpixel` | `disparity_roi_multi_point` |

## 使用原则

- 训练可靠性网络时，应按类别分组看贡献，避免 bbox/circle 多个高度相关字段造成过拟合。
- `METHOD_COLUMNS` 应只包含原始候选深度字段；`z_stereo`、`z`、`depth_method` 和 `stereo_depth_source` 只用于诊断、分组或 baseline。
- 实时 100fps 默认只启用轻量稳定候选。
- 重候选必须通过 baseline clip 连续帧指标后再进入主配置。
- `stereo_depth_source` 只能表示最终选中的主观测来源，不能替代所有候选字段。
- 左右 YOLO/IoU 误匹配排查看 `pair_shifted_iou`, `pair_score`, `pair_bbox_prior_penalty`, `pair_epipolar_dy`, `pair_positive_disparity` 和 `stereo_match_source`。
