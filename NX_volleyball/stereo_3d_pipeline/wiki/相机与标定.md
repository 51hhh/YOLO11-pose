# 相机与标定

## 标定采集

正式采集统一使用 C++ `capture_chessboard`:

```bash
cd ~/NX_volleyball/stereo_3d_pipeline/build_standalone
./capture_chessboard \
  -o calibration_images \
  -g 17.0 \
  --serial-left 00D39342665 \
  --serial-right 00219471413
```

操作:

- 空格: 保存当前同步帧。
- `q` 或 `ESC`: 退出。
- `c`: 清空已采集图像。

采集端保持 100fps 触发、约 60fps 显示，不做棋盘检测和质量评分，避免预览卡顿和触发抖动。

完整参数见 [标定工具细节](标定工具细节.md)。相机水印、触发和同步诊断见 [相机同步与水印](相机同步与水印.md)。

## 标定求解

NX 上 C++ 求解:

```bash
./stereo_calibrate -s 26.0 --board-w 5 --board-h 8
```

本机 Python 求解:

```bash
scp -r nvidia@10.42.0.149:~/NX_volleyball/stereo_3d_pipeline/build_standalone/calibration_images ./NX_volleyball/calibration
cd ./NX_volleyball/calibration
python3 stereo_calibration.py -s 26.0
```

## 采集质量要求

长基线双目标定要靠采集质量提升结果，而不是后处理选择性忽略差图。

建议:

- 采集 30-50 对以上有效图像。
- 棋盘覆盖画面中心、四角、左右边缘、上下边缘。
- 包含不同距离和姿态，避免所有图像都近似平行。
- 棋盘尽量平整、清晰、无明显运动模糊。
- 曝光和增益固定，避免左右亮度差过大。
- `capture_metadata.csv` 中 `frame_counter_delta` 应稳定为 0。

## 结果判断

标定输出至少检查:

- 左右单目 RMS。
- 立体 RMS。
- 基线长度是否接近实际测量。
- T 向量方向和尺度。
- 校正预览中同一水平线上的棋盘角点是否对齐。
- 运行管线时排球左右框的 y 残差是否稳定在合理范围内。

如果结果异常，优先重新采集更好的图像，而不是在标定求解后筛掉不喜欢的样本。

## 相机参数

运行配置中的相机序列号:

```yaml
serial_left: "00D39342665"
serial_right: "00219471413"
```

当前增益、曝光和触发参数以 `config/pipeline_dual_yolo_roi.yaml` 为准。修改增益或曝光后，应重新记录对应测试条件，避免把不同采集条件下的数据混在一起比较。
