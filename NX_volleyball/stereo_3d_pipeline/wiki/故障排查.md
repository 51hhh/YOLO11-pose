# 故障排查

## 标定慢

现象:

- 棋盘检测多核，但 RMS 和非线性求解阶段单线程。
- 140 张图求解耗时很长。

处理:

- 标定求解本质上是 OpenCV CPU 非线性优化，GPU 加速收益有限。
- 优先减少无效采集、提升棋盘质量和覆盖，而不是事后挑帧。
- 使用 `--no-vis` 跳过预览。
- 需要快速迭代时先用 30-50 对高质量图验证，再扩充数据。

工具参数见 [标定工具细节](标定工具细节.md)。

## 采集卡顿

处理:

- 标定采集端不要做棋盘检测、质量检测或重计算。
- 保持 100fps 硬触发和约 60fps 显示。
- 保存图像和日志不要阻塞取流。

## 左右不同步

检查:

- `frame_counter_delta`。
- `frame_number_delta`。
- `timestamp_delta_us`。
- 相机 trigger source、activation、serial 绑定。

长期不为 0 时，不要继续做深度评估，应先修相机同步。

水印字段解释和独立探针见 [相机同步与水印](相机同步与水印.md)。

## 关键点匹配不可信

检查:

- 关键点是否在左右 overlap mask 内。
- y 残差是否满足门限。
- 左右反查是否一致。
- 视差是否通过 MAD/RANSAC。
- 三角点是否接近球体物理半径。
- 是否把黑白灰度调试图误认为没有关键点；应查看带 drawMatches 的彩色或叠加输出。

实现位置和验证顺序见 [特征匹配实现](特征匹配实现.md)。

## 性能突然下降

先排查:

- 是否打开 `--visualize`、`--debug-feature-matches` 或 baseline 录制。
- 是否启用 ORB/BRISK/AKAZE/SIFT、subpixel 或神经匹配。
- 写盘队列是否满。
- GPU/CPU 间是否新增频繁拷贝。
- 是否没有锁性能模式。
