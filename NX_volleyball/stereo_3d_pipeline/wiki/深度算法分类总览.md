# 深度算法分类总览

最后核对: 2026-07-05

本页是深度方法的统一分类入口。它只记录每个方法属于哪一类、依赖什么输入、左右视差如何产生、当前是否用于默认录制或训练。算法细节、测试数据和问题跟踪放在子页面。

状态定义:

| 状态 | 含义 |
|---|---|
| 默认主线 | 当前 100fps 录制配置中写入主 CSV 或必须保留的字段 |
| 训练候选 | 训练读取器可读取，但不一定参与在线 `z_stereo/obj.z` |
| P1 sidecar | 已选入 P1 训练候选集合，但运行承载是 diagnostic sidecar，不回写主融合 |
| diagnostic sidecar | 写入 `*.p2_diagnostic.csv`，不回写主融合 |
| isolated 通过 | 单算法或单后端通过短测/isolated 测试，尚未证明联合默认配置稳定 |
| 默认关闭 | 已接入但不在默认录制配置启用 |
| 已退出 | 因速度、长尾、有效率或 artifact 错配退出当前默认结论 |
| 占位/待引入 | 配置或接口保留，真实后端未准入 |

## 当前默认录制口径

当前正式训练采集以 P0 几何 + P1 主 CSV `z_roi_multi_point` / `z_roi_center_patch` + P1 sidecar `z_roi_cuda_template_match` / `z_roi_neural_xfeat` / `z_roi_neural_superpoint` + fallback epipolar 为主。其它 P2 方法即使 isolated 通过，也需要单独联合长测后才能进入 P1。

当前重要边界:

- `z_stereo` / `obj.z` 是 legacy 在线选择结果，不应用作训练候选或标签。
- P1 sidecar 不回写主融合，训练读取器必须按 `frame_id + mode` 合并。
- XFeat/SuperPoint sidecar 和 CUDA Template/NCC 都不能替代 P0 默认主线，也不能抢占 legacy `z_stereo/obj.z`。
- GFTT/LK 每帧 diagnostic 已测不满足 100fps，应低频或单独 A/B。
- 已退出方法保留字段和文档，是为了复现实验和避免重复踩坑。
