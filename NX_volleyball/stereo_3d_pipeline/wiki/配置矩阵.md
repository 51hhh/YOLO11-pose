## 推荐使用方式

实时实机测试:

```bash
./stereo_pipeline --config ../config/pipeline_dual_yolo_roi.yaml
```

录制基准片段:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 3.0
```

算法矩阵测试:

```bash
python3 scripts/nx_algorithm_matrix_test.py \
  --project . \
  --out /tmp/codex_algo_tests \
  --duration-sec 10
```

注意: 当前脚本内部固定运行 `<project>/build/stereo_pipeline`。如果实机构建目录是 `build_standalone/`，需要先调整脚本或建立对应路径。

## 维护规则

- `pipeline_dual_yolo_roi.yaml` 只保留可以解释、可复现实验或生产测试所需参数。
- 对照配置必须在文件头说明用途，尤其是模型、DLA/GPU、tracker、stereo 策略差异。
- 废弃配置先在本页标注“历史/专项”，确认无人使用后再删除。
- 所有配置字段说明集中到 [配置参考](配置参考.md)，不要在多个页面重复维护完整字段表。
