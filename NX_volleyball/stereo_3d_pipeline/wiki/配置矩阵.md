# 配置矩阵

最后核对: 2026-07-19

`config/` 只保留当前运行、数据采集和基础参考配置。新实验应从主配置复制到 `/tmp` 或测试输出目录，避免把一次性参数写回正式配置目录。

## 推荐使用方式

联合项目正式运行:

```bash
./stereo_pipeline --config ../config/pipeline_rdk_joint.yaml
```

轨迹模型数据采集:

```bash
./stereo_pipeline \
  --config ../config/pipeline_record_p0p1.yaml \
  --recording-out ~/trajectory_dataset/traj_p0p1_001.csv
```

录制基准片段:

```bash
./stereo_pipeline \
  --config ../config/pipeline_dual_yolo_roi.yaml \
  --record-baseline-clip \
  --baseline-out baseline_clips \
  --baseline-duration 3.0 \
  --baseline-image-mode bgr \
  --baseline-format png
```

只采连续左右目图像、不等待检测框时追加 `--baseline-start-immediately`。灰度大批量采集可改为 `--baseline-image-mode gray --baseline-format pgm`。

算法矩阵测试:

```bash
python3 scripts/nx_algorithm_matrix_test.py \
  --project . \
  --out /tmp/codex_algo_tests \
  --duration-sec 10
```

矩阵脚本默认单算法隔离: 生成临时配置时先关闭全部 depth mode、`subpixel_enabled`、`fallback_epipolar_search` 和 neural，再只打开目标 case。`corner/texture/binary` sparse-lite 需要显式追加 `--include-approx`，正式算法矩阵不要使用混跑结果。

注意:

- 当前脚本内部固定运行 `<project>/build/stereo_pipeline`。如果实机构建目录是 `build_standalone/`，需要先调整脚本或建立对应路径。
- `neural_xfeat` case 当前期望 `models/neural/xfeat_extractor_128.engine`，并写入 `roi_size=128`, `top_k=64`。`scripts/nx_build_xfeat_extractor_engine.sh` 当前默认 `ROI_SIZE=128`，会生成这个 engine。
- `neural_aliked` 和 `neural_superpoint_lightglue` case 分别期望 `aliked_extractor_224_top128.engine`、`superpoint_extractor_224_top128.engine`，可由 `scripts/nx_build_lightglue_extractor_engines.sh` 生成。

## 维护规则

- `pipeline_dual_yolo_roi.yaml` 只保留可以解释、可复现实验或生产测试所需参数。
- 对照配置必须在文件头说明用途，尤其是模型、DLA/GPU、tracker、stereo 策略差异。
- 废弃配置先在本页标注“历史/专项”，确认无人使用后再删除。
- 所有配置字段说明集中到 [配置参考](配置参考.md)，不要在多个页面重复维护完整字段表。
