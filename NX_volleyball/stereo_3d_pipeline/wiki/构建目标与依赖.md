# 构建目标与依赖

最后核对: 2026-07-03

本页记录 `CMakeLists.txt` 当前实际构建目标和条件依赖。修改构建脚本后必须同步更新本页。

## 核心目标

| 目标 | 条件 | 用途 |
|---|---|---|
| `stereo_pipeline` | 始终构建 | 实时双目采集、检测、深度、轨迹和记录主程序 |
| `stereo_calibrate` | 始终构建 | OpenCV 双目标定求解，不依赖海康 SDK |
| `capture_chessboard` | 找到海康 MVS SDK 时构建 | C++ 标定图采集工具 |
| `hik_frame_metadata_probe` | 找到海康 MVS SDK 时构建 | 实机水印、帧号、触发稳定性探针 |
| `stereo_depth_viewer` | 找到海康 MVS SDK 时构建 | 双目深度查看和诊断工具 |

## 必需依赖

| 依赖 | 作用 |
|---|---|
| CUDA Toolkit | CUDA kernel、TensorRT buffer、GPU 图像处理 |
| VPI | CUDA/PVA 校正、可选 TNR、深度查看器 |
| OpenCV | 标定、图像 IO、几何、特征、可视化、可选 CUDA 模块 |
| TensorRT | YOLO 和实时 TensorRT 模型推理 |
| yaml-cpp | 主程序 YAML 配置解析 |
| pthread/stdc++fs | 后台线程、文件系统兼容 |

## 条件依赖

| 条件 | 编译宏或行为 | 影响 |
|---|---|---|
| 找到 Hikvision MVS SDK | `HIK_CAMERA_ENABLED` | 启用实机相机采集和海康相关工具 |
| 找到 libgpiod | 链接 libgpiod | 使用 GPIO PWM 触发 |
| 未找到 libgpiod | `NO_GPIOD` | 走 sysfs 或禁用 PWM fallback |
| 找到 ROS2 `ament_cmake` | `HAS_ROS2` | 编译 ROS2 话题桥和诊断发布 |
| 找到 ONNX Runtime | `HAS_ONNXRUNTIME` 仅 viewer | viewer 可加载 ONNX 深度模型 |
| 找到 OpenCV ximgproc | `HAS_XIMGPROC` 仅 viewer | viewer 可使用 WLS 等可选滤波 |
| 找到 OpenCV cudafeatures2d | `HAS_OPENCV_CUDAFEATURES2D` 仅 `stereo_pipeline` | 启用 true OpenCV CUDA ORB ROI 特征候选；未找到时 CUDA ORB 候选不可用 |

## 源码分组

`src/` 当前按功能拆分:

| 目录 | 内容 |
|---|---|
| `calibration/` | 棋盘采集、双目标定、PWM 触发封装 |
| `capture/` | 海康工业相机封装 |
| `detect/` | TensorRT 检测器、预处理和 DFL 后处理 CUDA |
| `rectify/` | VPI/CUDA 双目校正 |
| `pipeline/` | 实时调度、帧槽、检测和回调数据结构 |
| `stereo/` | 双目 ROI 深度、GPU/CPU 特征匹配、神经特征、VPI/ONNX stereo |
| `fusion/` | 坐标转换、混合深度、轨迹预测、深度提取 CUDA |
| `track/` | NanoTrack、MixFormer、裁剪缩放、SOT 接口 |
| `utils/` | CSV 记录、基准片段记录、日志、profile、zero-copy 辅助 |
| `ros/` | ROS2 目标点桥和诊断发布 |

## 安装结果

默认安装:

```bash
cmake --build build_standalone -j
cmake --install build_standalone
```

安装目标:

- `stereo_pipeline`
- `stereo_calibrate`
- `capture_chessboard`，仅海康 SDK 可用时

## 维护要求

- 新增实时算法源文件时，优先放入对应功能目录，再加入 `PIPELINE_SOURCES` 或 `CUDA_SOURCES`。
- 新增可执行工具时，写明是否依赖海康 SDK、ROS2、ONNX Runtime 或 ximgproc。
- 任何新增编译宏都必须在本页说明触发条件和运行影响。
