# 轨迹模型数据采集流程

最后核对: 2026-07-04

轨迹模型训练使用固定配置 `config/pipeline_record_p0p1.yaml`。该配置冻结 P0/P1 深度候选，开启 `recording.detail_level: extended`、`raw_mode: true` 和 frame sidecar。训练读取原始候选字段，例如 `z_bbox_center`、`z_circle_center`、`z_roi_edge_centroid`、`z_roi_radial_center`、`z_roi_edge_pair_center`、`z_roi_multi_point`、`z_roi_center_patch`；不要把 `z_stereo` 或 `z` 当训练候选或标签。

## NX 启动

配置或代码更新后先构建一次:

```bash
cd ~/NX_volleyball/stereo_3d_pipeline
cmake --build build_standalone -j$(nproc)
```

每段数据使用递增文件名前缀，例如 `traj_p0p1_001`、`traj_p0p1_002`。录制前先写同名前缀的 metadata:

```bash
cd ~/NX_volleyball/stereo_3d_pipeline
mkdir -p ~/trajectory_dataset
nano ~/trajectory_dataset/traj_p0p1_001.metadata.yaml
```

metadata 示例:

```yaml
dataset_type: static
scene: center_3m
known_z: 3.0
static: true
notes: static ball, center
```

启动录制:

```bash
cd ~/NX_volleyball/stereo_3d_pipeline
sudo jetson_clocks
cd build_standalone
./stereo_pipeline \
  --config ../config/pipeline_record_p0p1.yaml \
  --recording-out ~/trajectory_dataset/traj_p0p1_001.csv \
  2>&1 | tee ~/trajectory_dataset/traj_p0p1_001.log
```

按 `Ctrl-C` 停止，程序会 flush CSV。下一段把文件名前缀递增为 `traj_p0p1_002`。每段输出:

```text
traj_p0p1_001.csv
traj_p0p1_001.frames.csv
traj_p0p1_001.metadata.yaml
traj_p0p1_001.log
```

## 采集内容

先采静态已知距离，再采动态。静态覆盖 `2m, 3m, 4m, 5m, 6m`，每段 60-120s，并在 metadata 写 `known_z`、`static: true`、场景位置。动态覆盖慢速前后移动、左右/上下移动、正常抛球/击球、遮挡或单侧漏检；没有真实距离时 `known_z: null`，但必须写清楚 `scene` 和 `notes`。

采集完成后只做最小确认: `log` 末尾应有 `TrajectoryRecorder: saved ... frames`，同名前缀下应有主 CSV 和 `.frames.csv`。

## 下载到本机

先在 NX 上确认最新文件前缀:

```bash
sshpass -p nvidia ssh nvidia@10.42.0.149 'ls -t ~/trajectory_dataset/traj_p0p1_*.csv | head'
```

按同名前缀下载:

```bash
mkdir -p ~/mid360_datasets/trajectory

sshpass -p nvidia rsync -av --progress \
  nvidia@10.42.0.149:/home/nvidia/trajectory_dataset/traj_p0p1_001* \
  ~/mid360_datasets/trajectory/
```
