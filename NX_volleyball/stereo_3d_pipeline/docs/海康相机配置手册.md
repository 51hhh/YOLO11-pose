# 海康双目相机配置手册

## 1. 硬件概述

| 项目 | 规格 |
|------|------|
| 相机型号 | 海康 GigE 工业相机 (双目) |
| 分辨率 | 1440 × 1080 |
| 像素格式 | BayerRG8 (8-bit Bayer, RG 排列) |
| 接口 | GigE Vision |
| 平台 | NVIDIA Jetson Orin NX 16GB |

## 2. 相机参数

### 2.1 当前生产参数

```yaml
# pipeline.yaml / pipeline_dla.yaml
camera:
  exposure_us: 9867.0        # 曝光时间 (μs)
  gain_db: 11.9906           # 增益 (dB)
  use_trigger: true          # 硬件外触发
  trigger_source: "Line0"    # 触发信号输入线路
  trigger_activation: "RisingEdge"  # 上升沿触发
  trigger_chip: "gpiochip2"  # GPIO 芯片
  trigger_line: 7            # GPIO 线路号
  width: 1440
  height: 1080
```

### 2.2 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `exposure_us` | 9867.0 | 固定曝光，禁用自动曝光。排球场景验证的最优值 |
| `gain_db` | 11.9906 | 固定增益，禁用自动增益。与曝光配合获得最佳信噪比 |
| `use_trigger` | true | 启用硬件外触发模式。双目同步必须使用 |
| `trigger_source` | Line0 | 相机 Line0 引脚接收触发信号 |
| `trigger_activation` | RisingEdge | 上升沿触发采集 |

### 2.3 SDK 枚举值映射

```
// hikvision_camera.cpp - configureCamera()
PixelFormat      → BayerRG8
ExposureAuto     → 0 (Off)
ExposureTime     → 9867.0
GainAuto         → 0 (Off)
Gain             → 11.9906
TriggerMode      → 1 (On) / 0 (Off)
TriggerSource    → Line0=0, Line1=1, Software=7
TriggerActivation → RisingEdge=0, FallingEdge=1
```

## 3. 触发模式

### 3.1 硬件触发 (生产模式)

双目相机必须通过外部硬件信号同步触发，确保左右图像严格同步。

**信号链路**:
```
Orin NX GPIO (gpiochip2 line 7 / sysfs 393)
    ↓  方波 PWM 信号
相机 Line0 引脚
    ↓  上升沿触发
相机采集一帧
```

**PWM 频率**:
- 标定采集: 10 Hz (capture_chessboard)
- 运行流水线: 100 Hz (pipeline)

### 3.2 FreeRun 模式 (调试/单机测试)

无需外部触发信号，相机连续采集。适用于：
- 单相机调试
- 无 GPIO 硬件的开发机测试
- 初步验证相机连通性

```bash
# capture_chessboard 使用 --free-run 模式
./capture_chessboard -o calibration_images --free-run
```

> ⚠️ FreeRun 模式下双目不保证帧同步，不可用于生产标定。

## 4. GPIO 触发器实现

### 4.1 硬件接线

| Orin NX 引脚 | 连接 | 相机引脚 |
|---|---|---|
| GPIO gpiochip2 line 7 | → 触发线缆 → | 左右相机 Line0 |

**sysfs GPIO 编号计算**:
```
gpiochip2 base = 386
line = 7
sysfs number = 386 + 7 = 393
```

### 4.2 软件实现 (pwm_trigger.h)

PWMTrigger 支持两种后端：

| 后端 | 条件 | 精度 | 依赖 |
|------|------|------|------|
| libgpiod | 编译时未定义 `NO_GPIOD` | 最佳 (μs 级) | `libgpiod-dev` |
| sysfs GPIO | 默认 fallback | 良好 (100μs 级) | 无额外依赖 |

**优先级**: libgpiod → sysfs GPIO → 禁用

#### sysfs 操作流程

```bash
# 1. 导出 GPIO
echo 393 > /sys/class/gpio/export

# 2. 设置方向
echo out > /sys/class/gpio/gpio393/direction

# 3. 翻转产生 PWM
echo 1 > /sys/class/gpio/gpio393/value
sleep <half_period>
echo 0 > /sys/class/gpio/gpio393/value
sleep <half_period>

# 4. 释放
echo 393 > /sys/class/gpio/unexport
```

#### 代码中的使用

**标定脚本** (`capture_chessboard`):
```cpp
PWMTrigger trigger("gpiochip2", 7, 10);  // 10Hz
trigger.start();
// ... 采集棋盘格图像 ...
trigger.stop();
```

**主流水线** (`pipeline.cpp`):
```cpp
// init() 中自动创建 (config.use_trigger=true 时)
pwm_trigger_ = std::make_unique<PWMTrigger>(
    config_.trigger_chip, config_.trigger_line, config_.trigger_freq_hz);
// start() 中启动 → stop() 中停止
```

### 4.3 GPIO 权限

NX 上需要 root 权限或 gpio 组权限：

```bash
# 方法1: 添加用户到 gpio 组
sudo usermod -aG gpio nvidia

# 方法2: udev 规则 (推荐)
echo 'SUBSYSTEM=="gpio", KERNEL=="gpiochip*", MODE="0660", GROUP="gpio"' | \
    sudo tee /etc/udev/rules.d/99-gpio.rules
sudo udevadm control --reload-rules
```

## 5. BayerRG8 像素格式

### 5.1 为什么使用 BayerRG8

| 格式 | 每帧数据量 | 带宽 @100fps | 优势 |
|------|------|------|------|
| RGB8 (3ch) | 4.67 MB | 467 MB/s | 无需转换 |
| **BayerRG8 (1ch)** | **1.56 MB** | **156 MB/s** | **带宽节省 67%** |

BayerRG8 大幅降低 GigE 传输带宽需求，实现更高帧率。

### 5.2 Bayer 去马赛克 (Demosaic)

```cpp
// 海康 BayerRG8 → OpenCV BayerBG2BGR 转换
cv::cvtColor(bayer_image, bgr_image, cv::COLOR_BayerBG2BGR);
```

> **注意**: 海康 SDK 的 BayerRG8 对应 OpenCV 的 `COLOR_BayerBG2BGR`，不是 `BayerRG2BGR`。
> 这是因为海康和 OpenCV 对 Bayer 模式的命名约定不同。

### 5.3 GPU 加速去马赛克

生产流水线中可使用 CUDA kernel 或 VPI 进行 GPU 端 Bayer → RGB 转换，
避免 CPU 瓶颈。具体方案见 `detect_preprocess.cu`。

## 6. 统一使用规范

### 6.1 标定采集与主流水线一致性

| 配置项 | capture_chessboard | pipeline |
|--------|-------------------|----------|
| 曝光 (μs) | 9867.0 (默认) | 9867.0 (YAML) |
| 增益 (dB) | 11.9906 (默认) | 11.9906 (YAML) |
| 像素格式 | BayerRG8 | BayerRG8 |
| 触发模式 | 硬件触发 (默认) | 硬件触发 (YAML) |
| 触发源 | Line0 / RisingEdge | Line0 / RisingEdge |
| PWM 频率 | 10 Hz | 100 Hz |
| GPIO | gpiochip2 line 7 | gpiochip2 line 7 (YAML) |

> 确保标定时的相机参数与运行时完全一致，避免因曝光/增益差异导致标定偏差。

### 6.2 命令行覆盖

```bash

  -o calibration_images \
  -e 9867          # 曝光时间 (μs)

```

## 7. 故障排除

### GrabPair partial fail: L=0 R=0

**原因**: 相机处于硬件触发模式但未收到触发信号。

**解决**:
1. 确认 GPIO 权限: `ls -la /sys/class/gpio/`
2. 手动测试 GPIO:
   ```bash
   echo 393 > /sys/class/gpio/export
   echo out > /sys/class/gpio/gpio393/direction
   echo 1 > /sys/class/gpio/gpio393/value
   cat /sys/class/gpio/gpio393/value  # 应输出 1
   echo 393 > /sys/class/gpio/unexport
   ```
3. 使用 `--free-run` 临时绕过 (仅调试)
4. 检查触发线缆连接

### 相机未找到

```bash
# 列出 GigE 相机
opt/MVS/bin/EnumDevices

# 检查网络
ping <相机IP>
ethtool <网卡> | grep Speed  # 应为 1000Mb/s
```

### sysfs GPIO 已被占用

```bash
# 检查已导出的 GPIO
ls /sys/class/gpio/gpio393/
# 如果存在，先解除
echo 393 > /sys/class/gpio/unexport
```

### libgpiod vs sysfs

程序会自动选择可用后端：
```
[INFO] Using libgpiod backend      ← 精度最佳
[WARN] libgpiod failed, using sysfs fallback  ← 正常可用
[ERROR] PWM trigger disabled       ← 需要检查 GPIO 权限
```
