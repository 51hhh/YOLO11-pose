# 问题速查：海康 GigE 双目相机

> 相机: 海康 GigE · 1440×1080 · BayerRG8 · PWM 100Hz 触发

## 采集模式

### BayerRG8（推荐）
- 单通道 1.56MB/帧，带宽省 2/3
- GPU 去马赛克 `detect_preprocess.cu` → 1.8ms
- 帧率 >100fps

### RGB8（备选）
- 三通道 4.67MB/帧，带宽瓶颈 ~76fps
- 无需去马赛克

## Bayer 颜色映射（易错点）

海康 SDK 报告 `BayerRG8`，但实际数据排列是 OpenCV 的 `BayerBG`：
```cpp
// 正确
cv::cvtColor(bayer, rgb, cv::COLOR_BayerBG2RGB);
// 错误（偏红）
cv::cvtColor(bayer, rgb, cv::COLOR_BayerRG2RGB);
```

## PWM 触发配置

### 相机端
```
TriggerMode = On
TriggerSource = Line0 (或 Line2)
TriggerActivation = RisingEdge
```

### NX 端 GPIO
```bash
# libgpiod 控制 (用户态，无需 root)
gpioset gpiochip0 <pin>=1; sleep 0.0001; gpioset gpiochip0 <pin>=0
```

### PWM 精度
- `usleep()` 精度差 → 使用 `clock_nanosleep(CLOCK_MONOTONIC, ...)`
- 配合 `SCHED_FIFO` 实时调度优先级

## GigE 网络优化
```bash
# 巨帧
sudo ip link set eth0 mtu 9000
# 接收缓冲区
sudo sysctl -w net.core.rmem_max=33554432
sudo sysctl -w net.core.rmem_default=33554432
```

## 常见问题

| 问题 | 原因 | 解决 |
|---|---|---|
| 枚举不到相机 | 网段不匹配 | `sudo MVS/bin/SetIP` 配置同网段 |
| 帧率 <60 | MTU 未开巨帧 | `ip link set mtu 9000` |
| 图像偏红/偏绿 | Bayer RG↔BG 反转 | 改用 `COLOR_BayerBG2RGB` |
| GPIO 无权限 | `/dev/gpiochip*` 权限 | 添加 udev rule |
