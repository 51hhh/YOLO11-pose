# 03 模型与推理

## 模型信息

### YOLO26 (当前生产模型)

| 属性 | 值 |
|------|-----|
| 文件 | yolo26.onnx |
| 输入 | 1×3×640×640 (BGR, CHW, FP32) |
| 输出 | Pre-NMS 格式 (bbox + conf + keypoints) |
| 节点数 | 402 |
| 架构 | 自定义 YOLO + Attention (Softmax) |
| 关键点 | 5个 (球心Center + 4个轮廓点) |

### 废弃模型

| 模型 | 问题 |
|------|------|
| yolo26n.onnx (320×320) | Post-NMS + TopK 算子, DLA不支持 |
| yolov10n | TensorRT 构建全部失败 |
| yolov11n_dla | DLA FP16 太慢 27ms |

## TensorRT 引擎矩阵

### 推荐引擎

| 引擎 | 精度 | 设备 | 延迟 | 吞吐 | 检测率 | 状态 |
|------|------|------|------|------|--------|------|
| yolo26_gpu_int8_640 | INT8 | GPU | 2.89ms | 345qps | 65% | 备选(精度低) |
| yolo26_gpu_fp16_640 | FP16 | GPU | 4.10ms | 265qps | 98% | ★ 推荐 |
| yolo26_dla0_fp16_640 | FP16 | DLA0 | 27ms | 37qps | 95% | DLA可用时备选 |

> **注意**: Orin NX 当前 DLA 设备节点缺失, 实际使用 GPU FP16。

### 精度问题

- **GPU INT8**: Attention 块中的 Softmax 量化损失严重 → 检测率仅 65%
- **GPU FP16**: 曾因 ForeignNode fusion bug 构建失败, TRT 10.3 已修复
- **DLA FP16**: 正常工作但延迟高 (27ms), 适合作为 GPU 负载分流

## 引擎构建

### GPU FP16 (推荐)

```bash
trtexec \
  --onnx=yolo26.onnx \
  --saveEngine=yolo26_fp16.engine \
  --fp16 \
  --workspace=1024 \
  --inputIOFormats=fp16:chw \
  --outputIOFormats=fp16:chw
```

### GPU INT8 (需校准)

```bash
trtexec \
  --onnx=yolo26.onnx \
  --saveEngine=yolo26_int8.engine \
  --int8 \
  --calib=calibration_cache.bin \
  --workspace=1024
```

### DLA 构建

```bash
trtexec \
  --onnx=yolo26.onnx \
  --saveEngine=yolo26_dla0_fp16.engine \
  --fp16 \
  --useDLACore=0 \
  --allowGPUFallback
```

## DLA-GPU 混合推理

### Orin DLA 2.0 能力

**支持算子**: Conv2D, Deconv, Pooling, ReLU/Sigmoid/Tanh, BatchNorm, Concat, Resize  
**不支持**: Softmax (Attention核心算子), TopK, Gather, Split

### Split A 方案 (DLA+GPU混合)

```
Model Layers 0-9 → DLA0 (卷积主干)
Model Layers 10+ → GPU  (Attention块)
```

- 延迟: 4.38ms (DLA部分 2.1ms + GPU部分 2.3ms)
- GPU 占用率从 100% 降至 67%

### Attention Softmax 分解

DLA不支持 Softmax, 需分解为 DLA 可执行的算子:
```
Softmax(x) = exp(x - max(x)) / sum(exp(x - max(x)))
```
分解后: Max(reduce) → Sub → Exp → Sum(reduce) → Div, 全部可跑 DLA。

## INT8 量化指南

### 校准流程

1. 准备 500-1000 张训练集图像 (代表性分布)
2. 构建时指定 `--int8 --calib=calibration_cache.bin`
3. TRT 自动进行 KL 散度最小化找最优量化阈值

### 校准注意事项

- 随机噪声校准 vs 真实数据校准: 真实数据精度高 10-15%
- Attention 块量化损失大, 建议混合精度: Attention FP16 + 其余 INT8
- 校准缓存文件可跨构建复用

## 预处理 Pipeline

```

    │   └── BGR8/Gray (校正后, 1280×720)
    │
    └── letterboxAndNormalize_kernel
        └── FP16 CHW (1×3×640×640) → TRT输入
```
