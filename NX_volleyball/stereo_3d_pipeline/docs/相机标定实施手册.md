# 双目相机标定实施手册（给实施同学）

> 目标：产出可用于 `stereo_pipeline` 的高质量 `stereo_calib.yaml`，为下一步真实深度图输出打基础。  
> 平台：Jetson Orin NX + 海康双目 GigE 相机。

---

## 1. 交付物定义

你需要最终提交以下文件：

1. `calibration/stereo_calib.yaml`（主产物）
2. `calibration_images/` 原始采集图像（至少 20-30 对）
3. 标定质量记录（重投影误差、有效图像对数、基线）

建议新增一个记录文档：`docs/标定记录_YYYYMMDD.md`，包含设备参数与本次结果。

---

## 2. 标定前检查清单

- [ ] 两个相机固定牢靠，标定期间不移动
- [ ] 镜头焦距/光圈/对焦固定
- [ ] 曝光、增益固定（禁止自动曝光）
- [ ] 分辨率固定为运行分辨率（推荐 1440×1080）
- [ ] 触发模式确定：
  - 联调阶段可 FreeRun
  - 生产建议硬触发同步
- [ ] 棋盘格平整，无翘曲、反光可控

> 注意：标定参数与分辨率强相关，分辨率变了要重标。

---

## 3. 图像采集规范

### 3.1 推荐采集数量

- 最少：20 对
- 推荐：30-50 对

### 3.2 视角覆盖要求

每种至少 3-5 对：

- [ ] 棋盘格居中
- [ ] 左上/右上/左下/右下
- [ ] 近距离（占画面 60%-80%）
- [ ] 中距离（占画面 30%-50%）
- [ ] 远距离（占画面 15%-25%）
- [ ] 大角度倾斜（绕 X/Y 轴）

### 3.3 采集命令（NX）

在 `stereo_3d_pipeline/build` 下执行：

- **FreeRun 采集（推荐，无需 PWM 触发线）：**
  ```bash
  ./capture_chessboard --free-run -o calibration_images -e 3000
  ```
- 硬触发采集（需 libgpiod + PWM 线已接 Line0）：
  ```bash
  ./capture_chessboard -o calibration_images -e 3000
  ```

> ⚠️ 如果看到 `GrabPair partial fail: L=0 R=0`，说明相机处于触发模式但未收到脉冲。
> 请加 `--free-run` 参数或检查 PWM 接线。

交互按键：

- 空格：保存当前左右图
- `c`：清空已采集
- `q` / `ESC`：退出

---

## 4. 执行标定

在 `stereo_3d_pipeline/build` 下执行：

- `./stereo_calibrate -s 30.0 -d calibration_images -o calibration/stereo_calib.yaml`

参数说明：

- `-s`：棋盘格方格边长（毫米）
- `-d`：采集图目录
- `-o`：输出标定文件

如棋盘格规格不是 9x6，请补充：

- `--board-w <列数>`
- `--board-h <行数>`

---

## 5. 质量验收标准（必须满足）

建议阈值：

1. 单目标定重投影误差：< 0.5 px
2. 双目标定重投影误差：< 1.0 px
3. 有效图像对：>= 20
4. 基线长度稳定：与物理测量误差 < 5%
5. 立体校正后：同一物理点在左右图极线方向偏差接近 0（水平线对齐）

若不满足：

- 优先排查：
  - 图像模糊/过曝
  - 角点覆盖不足
  - 相机参数自动模式未关闭
  - 镜头被误触重新对焦

---

## 6. 在 Pipeline 中启用

更新配置文件：

- `config/pipeline.yaml`
- `config/pipeline_dla.yaml`

字段：

- `calibration.file: "calibration/stereo_calib.yaml"`

然后做 10-20 秒试跑，确认无加载错误。

---

## 7. 面向“真实深度图输出”的附加检查

为下一阶段深度图输出，建议额外确认：

- [ ] 左右相机时间戳差（同步触发后）可控
- [ ] 近距离（0.5-2m）和中距离（2-8m）都有实测样本
- [ ] 视差图中平面区域无大面积撕裂/空洞
- [ ] 已记录深度真值点（卷尺测量）用于误差对比

推荐记录 3 个标定后验证点：

1. 1m 目标中心点深度误差
2. 3m 目标中心点深度误差
3. 5m 目标中心点深度误差
