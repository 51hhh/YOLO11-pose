# 双目相机标定实施手册

> 目标：产出可用于 `stereo_pipeline` 的高质量 `stereo_calib.yaml`，为下一步真实深度图输出打基础。  
> 平台：Jetson Orin NX + 海康 USB3 双目相机。

---

## 1. 结论

正式长基线标定使用 `stereo_3d_pipeline` 下的 C++ 工具：

- `capture_chessboard`：采集左右同步棋盘图
- `stereo_calibrate`：检测角点并生成 `stereo_calib.yaml`

`NX_volleyball/calibration/*.py` 暂时不删除，只保留为 Legacy Python 参考和离线交叉验证。等 C++ 流程在新基线相机上完成一次标定、深度验证和稳定性测试后，再统一归档或删除。

需要同时保留 GUI 和非 GUI 两种采集方式：

- GUI 模式：人工摆棋盘时使用，可实时看左右画面，并用 `--live-check` 确认两目都识别到棋盘。
- headless 模式：SSH/无显示环境使用，适合棋盘位置已确认后的固定节奏自动采集。

## 2. 交付物定义

你需要最终提交以下文件：

1. `calibration/stereo_calib.yaml`（主产物）
2. `calibration_images/` 原始采集图像（推荐 30-50 对）
3. 标定质量记录（重投影误差、有效图像对数、基线）
4. `calibration_images/capture_metadata.csv`（同步水印记录）

建议新增一个记录文档：`docs/标定记录_YYYYMMDD.md`，包含设备参数与本次结果。

---

## 3. 标定前检查清单

- [ ] 两个相机固定牢靠，标定期间不移动
- [ ] 镜头焦距/光圈/对焦固定
- [ ] 曝光、增益固定（禁止自动曝光）
- [ ] 分辨率固定为运行分辨率（推荐 1440×1080）
- [ ] 触发模式确定：生产标定必须使用硬触发同步
- [ ] 左右相机序列号已记录，运行和标定保持同一个 left/right 绑定
- [ ] 棋盘格平整，无翘曲、反光可控

> 注意：标定参数与分辨率强相关，分辨率变了要重标。

---

## 4. 构建标定工具

在 NX 的 `NX_volleyball/stereo_3d_pipeline` 目录执行：

```bash
cmake -S . -B build_standalone -DCMAKE_BUILD_TYPE=Release -DCUDA_ARCH=87 \
  -DCMAKE_DISABLE_FIND_PACKAGE_ament_cmake=ON \
  -DCMAKE_CUDA_COMPILER=/usr/local/cuda-12.6/bin/nvcc \
  -DCUDAToolkit_ROOT=/usr/local/cuda-12.6
cmake --build build_standalone --target capture_chessboard stereo_calibrate -j$(nproc)
cd build_standalone
```

## 5. 图像采集规范

### 5.1 推荐采集数量

- 最少：20 对
- 推荐：30-50 对

### 5.2 视角覆盖要求

每种至少 3-5 对：

- [ ] 棋盘格居中
- [ ] 左上/右上/左下/右下
- [ ] 近距离（占画面 60%-80%）
- [ ] 中距离（占画面 30%-50%）
- [ ] 远距离（占画面 15%-25%）
- [ ] 大角度倾斜（绕 X/Y 轴）

### 5.3 采集命令（NX）

在 `stereo_3d_pipeline/build_standalone` 下执行：

> 采集工具默认只采集图像，不做棋盘检测。开启 `--live-check` 时才在 GUI 预览中做轻量棋盘检测，用于确认棋盘是否同时进入左右相机视野；正式角点检测仍在 `stereo_calibrate` 阶段统一完成。

- **GUI 硬件触发采集（推荐，100Hz PWM 同步）：**
  ```bash
  ./capture_chessboard \
    -o calibration_images \
    --serial-left 00D39342665 \
    --serial-right 00219471413 \
    --live-check
  ```
- FreeRun 采集（仅用于排查相机画面，不用于正式标定）：
  ```bash
  ./capture_chessboard --free-run -o calibration_images
  ```
- SSH 无头模式自动采集：
  ```bash
  ./capture_chessboard --headless -n 30 -o calibration_images \
    --serial-left 00D39342665 \
    --serial-right 00219471413
  ```

> 采集工具会先让两台相机进入 grabbing 状态，再启动 PWM，避免左右水印 FrameCounter 起点错位。
> 每次保存会同步写入 `capture_metadata.csv`，正式标定时 `frame_counter_delta` 应稳定为 0。`timestamp_delta_ns` 是 SDK 设备时间戳差，当前海康 USB3 实测单位按 ns 诊断使用，不能替代 FrameCounter 作为主同步键。
> ⚠️ 如果看到 `GrabPair partial fail: L=0 R=0`，说明相机未收到触发信号。
> 请检查 GPIO 是否已释放（见 `海康相机配置手册.md` §4.4）或使用 `--free-run`。

交互按键：

- 空格：保存当前左右图
- `c`：清空已采集
- `l`：开关实时棋盘检测叠加
- `q` / `ESC`：退出

GUI 版本用于人工摆棋盘，`--live-check` 会在左右画面叠加棋盘识别状态，确认棋盘同时进入两个相机视野。非 GUI/headless 版本用于固定节奏自动采集，不能实时看画面，建议只在棋盘位置已经确认后使用。

当前 GUI 行为：

- 相机采集和触发仍按 `100Hz` 运行。
- 预览显示限到约 `30fps`，降低 `imshow` 和全分辨率棋盘检测带来的卡顿。
- `--live-check` 仅每 10 个预览帧检测一次，只在左上角显示 `L: OK/MISS`、`R: OK/MISS`。
- 默认按 `6x9` 内角点检测，同时自动尝试交换方向 `9x6`。

## 6. 执行标定

在 `stereo_3d_pipeline/build_standalone` 下执行：

```bash
./stereo_calibrate -s 24.2 -d calibration_images -o calibration/stereo_calib.yaml
```

> SSH 环境加 `--no-vis` 跳过可视化预览。可加 `--gpu-preprocess` 用 CUDA 做 Bayer/BGR 转灰预处理；棋盘角点检测和亚像素优化仍由 OpenCV CPU 完成。

参数说明：

- `-s`：棋盘格方格边长（毫米）
- `-d`：采集图目录
- `-o`：输出标定文件

如棋盘格规格不是 6x9，请补充：

- `--board-w <列数>`
- `--board-h <行数>`

默认流程会先做左右单目标定，再用 `CALIB_FIX_INTRINSIC` 固定单目标定内参求双目外参。若确实需要让 `stereoCalibrate` 联合优化内参，显式增加 `--optimize-intrinsics`。

## 7. 质量验收标准（必须满足）

建议阈值：

1. 单目标定重投影误差：< 0.5 px
2. 双目标定重投影误差：< 1.0 px
3. 有效图像对：>= 20
4. 基线长度稳定：与物理测量误差 < 5%
5. 立体校正后：同一物理点在左右图极线方向偏差接近 0（水平线对齐）
6. `capture_metadata.csv` 中正式保存帧的 `frame_counter_delta=0`

若不满足：

- 优先排查：
  - 图像模糊/过曝
  - 角点覆盖不足
  - 相机参数自动模式未关闭
  - 镜头被误触重新对焦

---

## 8. 在 Pipeline 中启用

更新配置文件：

- `config/pipeline.yaml`
- `config/pipeline_dla.yaml`

字段：

- `calibration.file: "calibration/stereo_calib.yaml"`

然后做 10-20 秒试跑，确认无加载错误。

---

## 9. 面向“真实深度图输出”的附加检查

为下一阶段深度图输出，建议额外确认：

- [ ] 左右相机时间戳差（同步触发后）可控
- [ ] 近距离（0.5-2m）和中距离（2-8m）都有实测样本
- [ ] 视差图中平面区域无大面积撕裂/空洞
- [ ] 已记录深度真值点（卷尺测量）用于误差对比

推荐记录 3 个标定后验证点：

1. 1m 目标中心点深度误差
2. 3m 目标中心点深度误差
3. 5m 目标中心点深度误差
