# 双目相机标定实施手册

目标：产出可被 `stereo_pipeline` 直接加载的 `stereo_calib.yaml`。平台为 Jetson Orin NX
+ 海康 USB3 双目相机，当前棋盘参数为 `5x8` 内角点、`26.0mm` 方格。

## 1. 统一入口

正式采集只使用 C++ 工具：

- `capture_chessboard`：在 NX 上采集左右同步棋盘图。
- `stereo_calibrate`：在 NX 上用 C++ 直接求解。
- `stereo_calibration.py`：把采集图下载到本机后，用 Python 离线求解。

Python 采集脚本已删除。不要再使用 Python 进行相机采集。

## 2. 标定前检查

- 两个相机固定牢靠，标定期间不移动。
- 镜头焦距、光圈、对焦固定。
- 曝光和增益固定，禁止自动曝光和自动增益。
- 分辨率固定为运行分辨率，当前为 `1440x1080`。
- 左右相机序列号固定：左 `00D39342665`，右 `00219471413`。
- 棋盘格平整、刚性足够，无明显反光、过曝和运动模糊。
- 正式标定必须使用硬触发同步，`capture_metadata.csv` 中 `frame_counter_delta` 应稳定为 `0`。

## 3. 构建工具

在 NX 的 `~/NX_volleyball/stereo_3d_pipeline` 下执行：

```bash
cmake -S . -B build_standalone -DCMAKE_BUILD_TYPE=Release -DCUDA_ARCH=87 \
  -DCMAKE_DISABLE_FIND_PACKAGE_ament_cmake=ON \
  -DCMAKE_CUDA_COMPILER=/usr/local/cuda-12.6/bin/nvcc \
  -DCUDAToolkit_ROOT=/usr/local/cuda-12.6
cmake --build build_standalone --target capture_chessboard stereo_calibrate -j$(nproc)
cd build_standalone
```

## 4. 采集图像

在 NX 的 `build_standalone` 下执行：

```bash
./capture_chessboard \
  -o calibration_images \
  -g 17.0 \
  --serial-left 00D39342665 \
  --serial-right 00219471413
```

交互按键：

- 空格：保存当前左右图。
- `c`：清空已采集图像。
- `q` / `ESC`：退出。

采集要求：

- 触发保持 `100Hz`，GUI 预览约 `60fps`。
- 采集端只负责保存同步图像，不做棋盘检测或质量筛选。
- 建议采集 `30-50` 对，至少覆盖画面中心、四角、近/中/远距离和绕 X/Y 轴倾斜。
- 当前 0.93m 长基线下，优先覆盖约 `1.5m - 8m` 的近/中距离样本。
- 棋盘在左右图中都要清晰可见，不能只在单目画面里覆盖充分。
- 如果某些姿态角点检测失败，回到采集端重拍，不在求解端靠自动删图掩盖问题。

`--free-run` 只用于排查画面，不用于正式标定。

## 5. 求解方式 A：NX 上 C++ 标定

在 NX 的 `build_standalone` 下执行：

```bash
./stereo_calibrate -s 26.0 --board-w 5 --board-h 8
```

默认读取 `calibration_images/left` 和 `calibration_images/right`，默认输出
`stereo_calib.yaml`。SSH 环境可追加 `--no-vis` 跳过可视化。

## 6. 求解方式 B：本机 Python 标定

在本机仓库根目录下载采集图：

```bash
scp -r nvidia@10.42.0.148:~/NX_volleyball/stereo_3d_pipeline/build_standalone/calibration_images ./NX_volleyball/calibration
```

然后执行：

```bash
cd ./NX_volleyball/calibration
python3 stereo_calibration.py -s 26.0
```

Python 标定脚本按左右同名文件配对，默认使用与 C++ 工具一致的 `CALIB_FIX_INTRINSIC`，
报告最差重投影误差，不自动剔除图像对。如果立体 RMS 超过 `1.0px`、校正 ROI 为空、
焦距或基线无效，脚本会拒绝保存结果。需要联合优化内参时显式追加
`--optimize-intrinsics`。

## 7. 质量验收

| 指标 | 要求 |
|------|------|
| 单目 RMS | 目标 `< 0.5px`，良好 `< 0.3px` |
| 立体 RMS | 必须 `< 1.0px`，良好 `< 0.5px` |
| 有效图像对 | 正式标定建议 `>= 20` |
| 基线 | 与物理测量误差应在合理范围内 |
| 校正 ROI | 左右均不能为空 |
| 极线验证 | 校正后同一物理点 y 方向偏差接近 0 |
| 同步水印 | `frame_counter_delta=0` |

当前一次可用结果示例：

```text
左 RMS:   0.2506 px
右 RMS:   0.1433 px
立体 RMS: 0.2744 px
基线:     925.73 mm
ROI:      左 1440x1079，右 1440x1080
```

## 8. 在 Pipeline 中使用

把标定文件放到运行配置引用的位置，例如：

```bash
cp stereo_calib.yaml ../stereo_3d_pipeline/build_standalone/calibration/stereo_calib.yaml
```

配置中保持：

```yaml
calibration:
  file: "calibration/stereo_calib.yaml"
```

当前基线接近 `0.93m`，近距离视差很大。运行配置不能继续用 `max_disparity: 256`
覆盖所有距离；优先使用 `config/pipeline_dual_yolo_roi.yaml` 这类长基线配置，或把当前
配置的 `max_disparity` 提高到能覆盖目标最近距离。
