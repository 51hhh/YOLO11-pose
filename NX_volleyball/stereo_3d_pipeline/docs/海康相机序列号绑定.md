# 海康双目相机序列号绑定指南

## 问题背景

海康 MVS SDK 通过 USB 总线枚举设备，**枚举顺序不确定**——同一台主机重启后，
物理左/右相机可能交换索引。若以索引 (0/1) 分配相机，会导致：

- 目标检测运行在错误的相机上
- 双目标定的左右关系不一致
- 深度结果出错

**解决方案：用序列号(SN)唯一绑定相机。**

## 获取相机序列号

### 方法一：枚举脚本

```bash
# NX 上运行 (需先设置环境变量)
export MVCAM_COMMON_RUNENV=/opt/MVS/lib
python3 scripts/enum_cameras.py
```

输出示例：
```
Found 2 cameras
  [0] Model: MV-CA016-10UC, SN: 00D39342665, UserID: 1
  [1] Model: MV-CA016-10UC, SN: 00219471413, UserID: 0
```

### 方法二：MVS 客户端

在 Windows 上打开 MVS 客户端，连接相机后在设备属性中查看 SN。

## 确认左右相机

1. 运行管线，不设序列号（按索引）
2. 遮住其中一个相机镜头
3. 观察检测画面是否消失
4. 通过 `enum_cameras.py` 找到被遮住相机的 SN

**当前设备**：

| 物理位置 | 序列号 | 枚举 UserID |
|---------|--------|-----------|
| 左相机（检测用） | `00219471413` | 0 |
| 右相机（双目辅助） | `00D39342665` | 1 |

## YAML 配置

在 `config/pipeline_yolo26_mixed.yaml` 的 `camera` 节：

```yaml
camera:
  serial_left: "00219471413"    # 物理左相机 SN → 用于 YOLO 检测
  serial_right: "00D39342665"   # 物理右相机 SN → 双目辅助
  left_index: 0                 # 仅在 serial 为空时生效 (回退)
  right_index: 1
```

**规则**：
- `serial_left` / `serial_right` 非空时，按序列号匹配（推荐）
- 序列号为空 `""` 时，按 `left_index` / `right_index` 回退（不推荐）

## 代码逻辑

```
hikvision_camera.cpp::openCamera()
  ├── serial 非空 → 遍历设备列表，匹配 chSerialNumber
  └── serial 为空 → 按 index 选择 devList.pDeviceInfo[index]
```

## 注意事项

1. **新设备必须重新绑定**：更换相机后需重新运行枚举脚本
2. **标定一致性**：标定时的左右分配必须与运行时一致，否则深度计算错误
3. **USB 带宽**：两个 USB3 相机应连接到不同的 USB 控制器根
4. **环境变量**：NX 上需设置 `MVCAM_COMMON_RUNENV=/opt/MVS/lib`
