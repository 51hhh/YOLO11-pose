# 追踪模型导出与 TensorRT 引擎构建指南

**平台**: Jetson Orin NX Super 16GB, JetPack 6.2, TensorRT 10.3.0

本文记录 YOLO 检测 + SOT 补帧链路里使用的追踪模型导出方法，重点覆盖：
- MixFormerV2-small
- LightTrackM
- 现有 NanoTrack / SiamFC 兼容方式

## 1. 运行模式

当前 pipeline 采用固定节拍调度：
- YOLO 检测按 `detect_interval=3` 运行，固定约 30fps
- 追踪模型在非检测帧补帧，输出维持 100fps

因此，追踪模型的目标不是“替代检测”，而是尽可能稳定地填补检测间隙。

## 2. MixFormerV2-small

### 2.1 导出特点

MixFormerV2 使用单一 ONNX / 单一 TRT 引擎：
- 输入：template + search
- 输出：`[score, cx, cy, w, h]`

### 2.2 导出步骤

1. 获取仓库：`researchmm/MixFormerV2`
2. 下载官方 `mixformerv2_small.pth.tar`
3. 使用 `scripts/export_mixformerv2_onnx.py` 导出 ONNX
4. 在 NX 上用 `trtexec` 构建 FP16 引擎

### 2.3 兼容性修正

为适配当前导出环境，做了以下修正：
- `lib/config/mixformer2_vit/config.py`
  - 补充缺失的 `PRETRAINED_STATIC`
- `lib/models/mixformer2_vit/head.py`
  - 移除 `.cuda()` 依赖，避免 CPU 导出失败
- `scripts/export_mixformerv2_onnx.py`
  - 使用配置里的 `DATA.TEMPLATE.SIZE` / `DATA.SEARCH.SIZE`
  - `torch.load(..., weights_only=False)` 兼容 PyTorch 2.11+ 的 checkpoint

### 2.4 NX 构建命令

```bash
/usr/src/tensorrt/bin/trtexec \
  --onnx=/home/nvidia/sot_export/exported/mixformerv2_s.onnx \
  --saveEngine=/home/nvidia/sot_export/exported/mixformerv2_s.engine \
  --fp16 \
  --memPoolSize=workspace:512MiB
```

### 2.5 结果

- 导出成功
- NX 上 FP16 引擎构建成功
- 平均 GPU latency 约 2.1 ms

## 3. LightTrackM

### 3.1 导出特点

LightTrack 最终采用 NanoTrackTRT 兼容的两段式结构：
- Backbone engine：提取 template / search 特征
- Head engine：执行相关性、预测与回归

### 3.2 兼容性修正

为了让导出链路在当前环境可用，做了以下修正：
- `lib/models/backbone/models/utils.py`
  - 用 `collections.abc.Iterable` 替换 `torch._six`
  - 修正 `_ntuple()` 返回值
- `lib/utils/utils.py`
  - `load_pretrain()` 改成自动判断 CUDA / CPU
- `scripts/export_lighttrack.py`
  - 使用 LightTrackM 官方 `path_name`
  - head 导出时绕过 BN 预处理
  - 将相关性模块切换到卷积版以提升 TRT 兼容性
  - 使用 legacy ONNX exporter (`dynamo=False`)

### 3.3 NX 构建命令

```bash
/usr/src/tensorrt/bin/trtexec \
  --onnx=/home/nvidia/sot_export/exported/lighttrack_backbone.onnx \
  --saveEngine=/home/nvidia/sot_export/exported/lighttrack_backbone.engine \
  --fp16 \
  --memPoolSize=workspace:256MiB

/usr/src/tensorrt/bin/trtexec \
  --onnx=/home/nvidia/sot_export/exported/lighttrack_head.onnx \
  --saveEngine=/home/nvidia/sot_export/exported/lighttrack_head.engine \
  --fp16 \
  --memPoolSize=workspace:256MiB
```

### 3.4 结果

- Backbone engine 构建成功
- Head engine 构建成功
- 使用 opset 13 的 legacy 导出图更稳

## 4. 配置对接建议

### 4.1 MixFormerV2

```yaml
tracker:
  enabled: true
  type: "mixformer"
  engine_path: "/home/nvidia/sot_export/exported/mixformerv2_s.engine"
  head_engine_path: ""
  detect_interval: 3
```

### 4.2 LightTrackM

```yaml
tracker:
  enabled: true
  type: "nanotrack"
  engine_path: "/home/nvidia/sot_export/exported/lighttrack_backbone.engine"
  search_engine_path: ""
  head_engine_path: "/home/nvidia/sot_export/exported/lighttrack_head.engine"
  detect_interval: 3
```

### 4.3 统一节拍

建议所有 SOT 配置统一使用：
- `performance.pwm_frequency: 100`
- `performance.target_fps: 100`
- `tracker.detect_interval: 3`

这样可以保证：
- YOLO 只跑 30fps
- tracker 补到 100fps
- 输出节拍稳定

## 5. 注意事项

1. **YOLO 检测频率已固定**：不再因 tracker 状态切换而抢跑。
2. **LightTrack 导出是兼容性调整版**：与原论文/原仓库的训练图不完全相同，但可用于当前 pipeline 的推理链路。
3. **NX 上 engine 路径固定**：请确保 `/home/nvidia/sot_export/exported/` 下对应 `.engine` 文件存在。
4. **如果更换 tracker**：只需要切换 `tracker.type` 和 engine 路径即可，pipeline 调度逻辑不需要再改。
