# YOLO 模型导出与 TensorRT 引擎构建指南

**平台**: Jetson Orin NX Super 16GB, JetPack 6.2, TensorRT 10.3.0

## 1. 问题背景

### 1.1 GPU FP16 构建失败的根因

TRT 10.3.0 将 YOLO 模型中的 attention 块（C2PSA/PSA）融合为 `ForeignNode`：
```
Split → Reshape → MatMul → Softmax → MatMul → Add  →  ForeignNode (fused)
```

**在 GPU FP16 模式下，TRT 没有这个 ForeignNode 的 kernel 实现**，导致构建失败：
```
Error[4]: IBuilder::buildSerializedNetwork: Error Code 4: Internal Error 
(Could not find any implementation for node ...)
```

这个问题与 workspace 大小无关（4GB 和 8GB 均失败）。

### 1.2 DLA 的"意外"精度保护效应

DLA 无法执行 attention 中的 MatMul → 自动回退到 GPU 以 FP16 运行。这反而**保护了精度**：

| 推理路径 | Attention 精度 | 检测率 |
|----------|---------------|--------|
| GPU INT8 | INT8 (量化) | 65-86% |
| DLA INT8 | FP16 (GPU 回退) | 87-88% |
| DLA FP16 | FP16 (GPU 回退) | 98-99% |

**结论**: Attention 块对 INT8 量化极度敏感，必须保持 FP16 精度。

## 2. 解决方案：Attention Softmax 分解

### 2.1 原理

将 `F.softmax(attn)` 分解为 4 个基础算子：
```python
# 原始: attn = F.softmax(attn, dim=-1)
# 分解为:
max_attn = attn.max(dim=1, keepdim=True).values
exp_attn = torch.exp(attn - max_attn)
sum_attn = exp_attn.sum(dim=1, keepdim=True)
attn = exp_attn / sum_attn
```

**效果**: TRT 不再将整个 attention 块融合为 ForeignNode → 每个算子独立选择精度 → GPU FP16 构建成功。

## 3. 引擎构建方法

### 3.1 GPU FP16（从分解 ONNX）

```bash
trtexec --onnx=model/yolov11n_decomposed.onnx \
    --fp16 \
    --saveEngine=model/yolov11n_gpu_fp16.engine \
    --memPoolSize=workspace:4096MiB
```

### 3.2 混合精度（Attention FP16 + 其余 INT8）

```bash
# 从原始 ONNX + 校准缓存
trtexec --onnx=model/yolo26.onnx \
    --int8 --fp16 \
    --calib=model/yolo26_calib_entropy2.cache \
    --precisionConstraints=prefer \
    --layerPrecisions="*attn*":fp16 \
    --layerOutputTypes="*attn*":fp16 \
    --saveEngine=model/yolo26_mixed.engine \
    --memPoolSize=workspace:4096MiB
```

**关键参数**:
- `--precisionConstraints=prefer`: 尽量遵守精度约束（`obey` 为强制）
- `--layerPrecisions="*attn*":fp16`: 所有含 "attn" 的层使用 FP16
- `--layerOutputTypes="*attn*":fp16`: 输出也使用 FP16

### 3.3 GPU INT8（从分解 ONNX）

```bash
trtexec --onnx=model/yolov11n_decomposed.onnx \
    --int8 --fp16 \
    --calib=model/yolov11n_dla_calib_entropy2.cache \
    --saveEngine=model/yolov11n_decomposed_gpu_int8.engine \
    --memPoolSize=workspace:4096MiB
```

### 3.4 DLA（从分解 ONNX）

```bash
trtexec --onnx=model/yolov11n_decomposed.onnx \
    --fp16 \
    --useDLACore=0 --allowGPUFallback \
    --saveEngine=model/yolov11n_decomposed_dla0_fp16.engine \
    --memPoolSize=workspace:4096MiB
```

### 3.5 自动化构建


## 4. 引擎命名约定

```
{model}_{backend}_{precision}[_{calibrator}].engine

model:      yolov11n / yolo26 / yolo26n
backend:    gpu / dla0 / dla1
precision:  fp16 / int8 / mixed (attn_fp16)
calibrator: entropy2 / legacy / minmax (INT8 only)

特殊:
{model}_decomposed_{backend}_{precision}.engine  # 从分解 ONNX 构建
```

## 5. 导出速度优化

| 方法 | 导出时间 | 说明 |
|------|---------|------|
| 默认 (simplify=True, opset=13) | ~10s | 推荐，TRT 兼容性最好 |
| --skip-simplify | ~5s | 跳过 onnxsim，略微降低 TRT 优化 |
| half=True (FP16 ONNX) | ~10s | 减小文件但不影响 TRT 构建 |

**注意**: `half=True` 不能解决 GPU FP16 构建失败问题。该错误源于 TRT kernel 缺失，与权重精度无关。

## 6. 校准缓存兼容性

| 校准方法 | trtexec 兼容 | Python API 兼容 | 推荐 |
|----------|-------------|----------------|------|
| Entropy2 | ✅ | ✅ | ⭐推荐 |
| Legacy | ✅ | ✅ | 可用（与 Entropy2 精度相同） |
| MinMax | ❌ | ✅ | 仅 Python API |

**MinMax 校准缓存**使用不同的头格式，trtexec 默认使用 Entropy2 校准器无法读取。

## 7. 注意事项

1. **opset 版本**: 推荐 opset 13+（TRT 10.x 最佳兼容）
2. **torchvision mock**: Jetson 上导出需要 mock torchvision（避免 C extension 编译错误）
3. **dynamic shapes**: 使用 `dynamic=False`（固定 640x640），避免不必要的复杂度
4. **校准图片**: 建议使用 500+ 张场景相关图片（calib_500 目录）
5. **DLA 构建远快于 GPU INT8**: DLA ~60-210s vs GPU INT8 ~600-730s
