# 08 故障排除

## VPI 3.2.4 已知Bug

### Bug 1: Bayer→BGR 静默黑屏

**现象**: `vpiSubmitConvertImageFormat(Bayer→BGR)` 返回成功, 但输出图像全黑  
**原因**: VPI 3.2.4 不支持 Bayer→BGR 转换, 但不报错  
**解决**: 使用自定义 CUDA kernel `bayerToBGR8_kernel` (detect_preprocess.cu)

### Bug 2: HOST_PITCH_LINEAR 零数据

**现象**: `vpiImageLockData(HOST_PITCH_LINEAR)` 锁 BGR8 图像, 读到全零  
**原因**: VPI HOST 端传输 BGR8 格式有 Bug, GPU 端数据正常  
**解决**: 使用 `CUDA_PITCH_LINEAR` + `cudaMemcpy2D` 拷贝到 Host

```cpp
VPIImageData data;
vpiImageLockData(img, VPI_LOCK_READ, VPI_IMAGE_BUFFER_CUDA_PITCH_LINEAR, &data);
cudaMemcpy2D(host_ptr, width*3, data.buffer.pitch.planes[0].data,
             data.buffer.pitch.planes[0].pitchBytes, width*3, height,
             cudaMemcpyDeviceToHost);
vpiImageUnlock(img);
```

### Bug 3: 无 CUDA Stream 互操作

**现象**: 没有 `vpiStreamGetCUDAStream()` API  
**影响**: 无法在 VPI stream 上直接提交 CUDA kernel  
**解决**: 使用独立 CUDA stream + cudaEvent 同步

## PWM 触发问题

### 频率抖动

**现象**: PWM 100Hz 频率抖动 ±5% (95-105 Hz)  
**原因**: `std::this_thread::sleep_for` 不是实时精度  
**解决**:
1. 改用 `clock_nanosleep(CLOCK_MONOTONIC, TIMER_ABSTIME, ...)` — 绝对时间定时
2. 设置线程调度策略 `SCHED_FIFO`, 优先级 80
3. `mlockall(MCL_CURRENT | MCL_FUTURE)` 锁定内存, 防止页面换出
4. CPU 亲和性绑定到指定核心

**结果**: 抖动从 ±500μs 降至 < 10μs

## Bayer 颜色映射

### 红蓝互换

**现象**: BGR 图像红蓝通道交换  
**原因**: 海康 BayerRG8 ≠ OpenCV BayerRG  
**解决**:
```
海康 BayerRG8 → OpenCV COLOR_BayerBG2BGR (注意是 BG 不是 RG)
```

### 自定义 kernel 颜色异常

**现象**: `bayerToBGR8_kernel` 输出颜色偏移  
**原因**: RGGB 模式行列偏移计算错误  
**解决**: 根据像素坐标 (row%2, col%2) 判断当前像素颜色:
```
(0,0)=R  (0,1)=G
(1,0)=G  (1,1)=B
```

## TensorRT 问题

### GPU FP16 构建失败

**现象**: `trtexec --fp16` 构建 yolo26.onnx 失败  
**原因**: ForeignNode fusion 导致的内部错误 (TRT 10.3 bug)  
**解决**: 使用 `--optimizeForTarget` 或等待 TRT 补丁; 当前使用 DLA FP16 替代

### GPU INT8 检测率低

**现象**: INT8 引擎检测率仅 65% (FP16 为 98%)  
**原因**: Attention 块中 Softmax 量化损失严重  
**解决**:
- 使用混合精度: Attention 层 FP16 + 其余 INT8
- 或使用真实数据校准 (非随机噪声)

### DLA 不可用

**现象**: `useDLACore=0` 报错 "DLA not available"  
**原因**: Orin NX 的 DLA 设备节点缺失 (驱动问题)  
**验证**:
```bash
ls /dev/nvdla*    # 应存在 nvdla0, nvdla1
cat /proc/device-tree/dla*/status  # 应为 "okay"
```
**解决**: GPU fallback (当前方案), 或等待固件更新

## 环境修复

### PyTorch 破坏

**现象**: `pip install torchvision` 升级 torch 到 cu13 版本, CUDA 不可用  
**解决**:
```bash
pip uninstall torch torchvision
pip install torch-2.4.0+nv24.08-...whl  # 使用 NVIDIA JP6 专用 wheel
```
> 不要在 JetPack 环境使用 `pip install torch` — 必须使用 NVIDIA 发布的 wheel

### jtop JetPack 版本显示 MISSING

**现象**: jtop 显示 JetPack 版本为 "MISSING"  
**原因**: jtop 4.3.2 缺少 L4T R36.4.7 → JetPack 6.2 映射  
**解决**:
```python
# /usr/lib/python3/dist-packages/jtop/core/jetson_variables.py
NVIDIA_JETPACK["36.4.7"] = "6.2"
```

### OpenCV 缺少 contrib 模块

**现象**: `cv::ximgproc::createDisparityWLSFilter` 未定义  
**原因**: JetPack 预装 OpenCV 不含 contrib  
**影响**: WLS 视差后处理不可用 (CMake 自动降级)  
**解决**: 编译安装 OpenCV with contrib (可选, ROI 模式不需要)

```bash
pip config set global.proxy http://192.168.31.30:10000
```

## 性能问题速查

| 症状 | 可能原因 | 排查 |
|------|---------|------|
| FPS < 50 | 非 MAXN_SUPER 模式 | `nvpmodel -q`, `jetson_clocks` |
| 黑屏无输出 | VPI Bayer→BGR bug | 检查是否用了 vpiSubmitConvertImageFormat |
| 颜色异常 | Bayer 模式映射错误 | BayerRG8 → OpenCV BayerBG |
| 检测率低 | INT8 量化损失 | 切换 FP16 引擎 |
| 帧率不稳定 | PWM 抖动 | 检查 SCHED_FIFO + clock_nanosleep |
| 内存不足 | 三缓冲 + TRT 占用 | 降低分辨率或减少缓冲数 |
| 相机打不开 | USB 权限/带宽 | `sudo chmod 666 /dev/bus/usb/*` |
