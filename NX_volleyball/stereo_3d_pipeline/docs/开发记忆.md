# 项目开发记忆 — 问题分类速查

> 记录开发过程中遇到的关键问题及解决方案，按类型分类。

---

## 一、VPI 3.2.4 问题

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| Bayer→BGR 静默失败 | `vpiSubmitConvertImageFormat` 返回成功但输出全黑 | 自定义 `bayerToBGR8_kernel` (detect_preprocess.cu) |
| HOST 读取 BGR8 全零 | `vpiImageLockData(HOST_PITCH_LINEAR)` 对 BGR8 返回零 | 用 `CUDA_PITCH_LINEAR` + `cudaMemcpy2D` |
| 无 CUDA Stream 互操作 | 缺少 `vpiStreamGetCUDAStream()` API | 独立 `cudaStreamBGR` + cudaEvent 同步 |
| VPI lock/unlock 高开销 | 每帧 8 次 lock/unlock 耗 2.4ms | Tegra 统一内存指针固定，init 时缓存 |

## 二、相机与触发

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| Bayer 颜色互换 | BGR 图像红蓝反转 | 海康 BayerRG8 → OpenCV `COLOR_BayerBG2BGR` |
| PWM 频率抖动 | ±5% (95-105Hz) | `clock_nanosleep(TIMER_ABSTIME)` + SCHED_FIFO 80 + mlockall |
| USB 带宽瓶颈 | BGR8 双目 ~107FPS 极限 | 改用 BayerRG8 (1/3 带宽) + CUDA demosaic |

## 三、TensorRT / DLA

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| GPU FP16 构建失败 | ForeignNode fusion 内部错误 | TRT 10.3 补丁修复；或用 DLA FP16 |
| INT8 检测率低 | Softmax 量化损失 → 65% 检测率 | 混合精度 (Attention FP16 + 其余 INT8) |
| DLA 不可用 | 设备节点 `/dev/nvdla*` 缺失 | GPU fallback；检查驱动/固件 |
| DLA 不支持 TopK | Post-NMS 模型无法跑 DLA | 导出 Pre-NMS 格式 ONNX |
| DLA Softmax 慢 | 56% 时间在 2 个 Attention 块 | Split A 方案: 层0-9→DLA, 层10+→GPU |

## 四、性能优化

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| Stage0 占 70% 时间 | 7.21ms (含同步 Grab) | 异步 Grab 线程 + 零拷贝按需模式 |
| Staging buffer 无效 | 额外 memcpy 抵消异步收益 | 直写 VPI Image (Tegra 统一内存) |
| 流水线无法重叠 | Stage0 阻塞在 Grab | 4阶段重叠: requestGrab→detect→fuse→waitGrab |
| VPI lock/unlock 累积 | 8 次 × 0.3ms = 2.4ms | init 时缓存 CUDA 指针，运行时零 VPI 调用 |
| remap 等待高 | WaitRect 1.28ms | 移动 vpiStreamSync 到 Stage0 正确位置 → 0.01ms |
| OpenCV demosaicing 慢 | ~56 FPS | 自研 CUDA kernel，双线性4邻域 → ~0.6ms |

## 五、环境配置

| 问题 | 现象 | 解决方案 |
|------|------|---------|
| pip 破坏 PyTorch | `pip install torchvision` 升级到 cu13 | 只用 NVIDIA JP6 wheel: `torch-2.4.0+nv24.08` |
| jtop 显示 MISSING | 缺少 L4T R36.4.7 映射 | 手动添加 `"36.4.7": "6.2"` 到 jetson_variables.py |
| 代理配置 | NX 无法联网 | 设置 `http_proxy=http://192.168.31.30:10000` |
| plink 非 ASCII 乱码 | Windows→Linux 传输 UTF-8 损坏 | 上传文件而非内联命令；源码用 ASCII 注释 |

## 六、设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 传输格式 | BayerRG8 (非 BGR8) | USB3 带宽仅够 BGR8@107FPS，BayerRG8@321FPS 余量大 |
| 深度算法 | ROI SAD (非全帧 SGM) | 全帧 70ms 不可接受；ROI <0.1ms |
| 缓冲策略 | 三缓冲环形队列 | 双缓冲有竞争；三缓冲完全隔离 Stage0/1/2 |
| 深度融合 | 单目+双目 Kalman | 近距离(0-4m)单目准；远距离(4-15m)双目准 |
| BGR 入口 | 先 Bayer→BGR 再送检测 | 彩色特征比灰度检测精度高 |
