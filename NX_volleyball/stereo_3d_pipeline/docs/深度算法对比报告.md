# Jetson Orin NX 16GB — 深度算法对比报告

> 硬件: Jetson Orin NX 16GB Super · JetPack 6.2 · CUDA 12.6  
> 相机: 海康 GigE 双目 1440×1080 · 基线 328.7mm · 焦距 1671.2px  
> 测试图像: 室内排球场景 · max_disparity=128

---

## 1. 算法总览

本预研共评估 **13 种** 双目深度算法，覆盖三大类：

| 类别 | 算法 | 后端 |
|---|---|---|
| **VPI 硬件加速** | VPI CUDA SGM Full / Half / Bilateral | VPI 3.2 + CUDA |
| **OpenCV 传统** | CUDA SGM / BM / BP / CSBP / SGBM CPU / SGBM+WLS / SGBM+Census | OpenCV 4.10 |
| **DL 深度学习** | CREStereo / HITNet / RAFT-Stereo / PSMNet | PyTorch / ONNX Runtime |

---

## 2. C++ Viewer 实测 (VPI + OpenCV CUDA/CPU)

在 NX 上用 `stereo_depth_viewer --headless` 对 1440×1080 图像 30 次迭代：

| 算法 | 均值(ms) | FPS | 有效率 | 深度中位(mm) | 备注 |
|---|---|---|---|---|---|
| **VPI CUDA SGM** | 64.2 | 15.6 | 99.0% | 6784 | 质量最佳传统算法 |
| **VPI OFA ds4** | 39.4 | 25.4 | 99.9% | 1734 | 硬件加速低延迟 |
| **OpenCV CUDA SGM** | 33.3 | 30.0 | 30.0% | 425 | 有效率过低 |
| **OpenCV CUDA BM** | 20.1 | 49.9 | 66.1% | 6037 | 最快GPU算法 |
| **OpenCV SGBM CPU** | 186.5 | 5.4 | 41.5% | 6618 | CPU瓶颈明显 |

### 要点
- **VPI CUDA SGM** 综合最优：99% 有效率、中位深度合理、15.6 FPS 可用
- **OpenCV CUDA BM** 速度最快但有效率仅 66%，噪点多
- **OpenCV CUDA SGM** 有效率 30% 不可用（参数待调优或驱动问题）
- CPU SGBM 186ms 无法满足实时需求

## 3. Python DL 模型评估 (480×640 输入)

在 NX 上用 `depth_algo_compare.py` 对同一标定对运行：

| 算法 | 耗时(s) | FPS | 有效率 | 深度均值(mm) | 深度范围(mm) |
|---|---|---|---|---|---|
| **HITNet** | 1.49 | 0.67 | 100% | 6244 | 1594 - 16216 |
| **PSMNet** | 2.09 | 0.48 | 100% | 5745 | 1440 - 22547 |
| **RAFT-Stereo** | 8.36 | 0.12 | 100% | 4837 | 1024 - 10737 |
| **CREStereo** | 12.06 | 0.08 | 100% | 5593 | 2022 - 10603 |
| **SGBM (对照)** | 0.35 | 2.86 | 30.4% | 5737 | 2154 - 29998 |

### 要点
- 所有 DL 模型有效率 **100%**，远超传统算法
- **HITNet** 速度最快（1.49s），深度范围合理
- **RAFT-Stereo** 深度范围最窄（精度最高），但 8.36s 过慢
- **CREStereo** 质量好但 12s 完全不可用于实时
- DL 模型当前为 PyTorch 推理，未经 TensorRT 加速

## 4. 综合对比矩阵

按 **实时性 → 质量 → 可用性** 排序：

| 排名 | 算法 | 类型 | 延迟 | 有效率 | 实时可行 | 推荐 |
|---|---|---|---|---|---|---|
| 1 | VPI CUDA SGM | 传统 | 64ms | 99% | ✅ 15 FPS | **主力算法** |
| 2 | OpenCV CUDA BM | 传统 | 20ms | 66% | ✅ 50 FPS | 低延迟备选 |
| 3 | HITNet (ONNX) | DL | ~1.5s* | 100% | ⚠️ 需TRT | **DL首选** |
| 4 | PSMNet | DL | ~2.1s* | 100% | ⚠️ 需TRT | DL备选 |
| 5 | VPI OFA ds4 | 传统 | 39ms | 100% | ✅ 25 FPS | 质量存疑** |
| 6 | SGBM+WLS | 传统 | ~200ms | ~60% | ❌ CPU | 后处理提升 |
| 7 | SGBM+Census | 传统 | ~220ms | ~45% | ❌ CPU | 光照鲁棒 |
| 8 | RAFT-Stereo | DL | ~8.4s* | 100% | ❌ | 离线高精度 |
| 9 | CREStereo (ONNX) | DL | ~12s* | 100% | ❌ | 离线高精度 |
| 10 | OpenCV SGBM CPU | 传统 | 187ms | 42% | ❌ | 仅参考 |
| 11 | OpenCV CUDA SGM | 传统 | 33ms | 30% | ❌ 有效率差 | 不推荐 |

> \* DL 延迟为 PyTorch 推理，TensorRT FP16 预计提速 5-20×  
> \*\* VPI OFA 中位深度 1734mm 异常偏低，需验证

## 5. C++ 集成方案

当前 `stereo_depth_viewer` 已整合 13 种算法：

```
模式 0-8:  原有 VPI/OpenCV 算法（已部署）
模式 9:    SGBM + WLS 后处理（需 ximgproc）
模式 10:   SGBM + Census 变换（自研）
模式 11:   CREStereo ONNX Runtime
模式 12:   HITNet ONNX Runtime
```

### 构建依赖
| 依赖 | 用途 | 条件编译 |
|---|---|---|
| ONNX Runtime | DL模型推理 | `HAS_ONNXRUNTIME` |
| OpenCV ximgproc | WLS滤波器 | `HAS_XIMGPROC` |

### Headless 基准测试
```bash
./stereo_depth_viewer --headless --crestereo dl_models/crestereo_init_iter10_480x640.onnx \
    --hitnet dl_models/hitnet_eth3d_480x640.onnx
```
输出：`diagnose_output/benchmark_report.json` + `comparison_grid.png`

## 6. 推荐方案

### 排球追踪实时方案
```
主力: VPI CUDA SGM (15 FPS, 99% 有效率)
  ↓ 当 GPU 余量不足时
备选: OpenCV CUDA BM (50 FPS, 66% 有效率 + 后处理)
```

### DL 升级路径
```
阶段1: HITNet ONNX → TensorRT FP16 (预期 10-30 FPS)
阶段2: CREStereo TensorRT (预期 2-5 FPS, 高精度场景)
```

### 后处理增强
- SGBM+WLS 可作为 VPI SGM 的后处理补充（WLS 滤波器可独立应用）
- Census 变换对光照变化鲁棒，适合排球场强灯光环境
