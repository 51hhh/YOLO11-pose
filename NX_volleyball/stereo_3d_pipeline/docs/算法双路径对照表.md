## 当前门槛

- 默认配置必须保持 default_geometry 级别: ROI FPS 约 100，`Stage2_DualYoloGpuCandidates` 均值约 0.26 ms，且无持续 stale drop。
- 新算法默认关闭；打开单项后必须记录 `Stage2_DualYoloGpuCandidates`、`Stage2_DualYoloMatch`、ROI FPS、stale drop。
- final3 中 `all_sparse_gpu` 为 89.8 FPS/stale 7/GPU avg 1.07 ms，不能把多种稀疏候选同时作为 100Hz 默认配置。
- offline 与 realtime 语义必须通过 `tools/compare_offline_online_depth.py` 对拍后才能进入优先级链或融合链。

## 对拍入口

当前可用的实时单帧入口:

```bash
./build/stereo_pipeline \
  --config config/pipeline_dual_yolo_roi.yaml \
  --debug-feature-matches \
  --debug-feature-matches-dir test_logs/feature_match_debug
```

当前可用的 CSV 对拍工具:

```bash
python3 tools/compare_offline_online_depth.py \
  --offline-summary test_logs/offline_keypoint_probe_recommended/summary.csv \
  --online-csv dual_yolo_observation_data.csv
```

注意: `--debug-feature-matches` 目前主要输出图像/ROI 调试材料,不是完整 extended CSV。要实现同一 stereo pair 的全自动闭环,后续应新增 `--probe-once` 或扩展 debug 输出为 `TrajectoryRecorder` 宽表。
