# 02 架构设计

## 流水线架构

### ROI模式 — 4阶段重叠流水线 (生产配置)

```
时间轴 ──────────────────────────────────────→

帧 N+2:  [Stage0: WaitGrab + Bayer→BGR + Remap]
帧 N+1:           [Stage1: YOLO检测]
帧 N:                      [Stage2: ROI匹配+3D测距]
帧 N+2:                               [Stage0: WaitGrab...]
```

**Phase A** → requestGrab(N+1): 信号Grab线程 (~0ms)  
**Phase B** → Stage1 detect(N): TensorRT异步提交 (~3.2ms, 与Grab并行)  
**Phase C** → Stage2 fuse(N-1): ROI匹配+融合 (~0.1ms, 与Grab并行)  
**Phase D** → waitGrab(N+1) + stage0_process(N+1): 等待+处理 (~6.8ms)

### 三缓冲环形队列

```
Slot 0: Stage2 处理中 (帧N)
Slot 1: Stage1 处理中 (帧N+1)  
Slot 2: Stage0 采集中 (帧N+2)
```

每个 FrameSlot 包含:
- VPIImage: rawL, rawR (BayerRG8 原始), rectL, rectR (校正后), tempBGR_L/R (BGR中间)
- GpuMat: 多点匹配缓冲
- CachedGPU: CUDA指针缓存 (rawL_gpu, rawR_gpu, tempBGR_L_gpu, tempBGR_R_gpu)
- cudaEvent: 帧间同步

## 关键架构决策

### 1. BayerRG8 原始传输 (vs BGR8)

| 方案 | 带宽/帧 | 说明 |
|------|---------|------|
| BGR8 | 4.67 MB | 3通道, USB带宽瓶颈 |
| **BayerRG8** | **1.56 MB** | **1通道, 节省2/3带宽** |

选用 BayerRG8: USB3 带宽 5Gbps 下, BGR8 理论极限 ~107 FPS, BayerRG8 可达 ~321 FPS, 余量充足。

### 2. 自定义CUDA Bayer→BGR Kernel (vs VPI/OpenCV)

VPI 3.2.4 Bug: `vpiSubmitConvertImageFormat(Bayer→BGR)` 静默输出全黑图像。  
OpenCV `cv::cuda::demosaicing` 太慢 (~56 FPS)。

自研 `bayerToBGR8_kernel`: 双线性4邻域 RGGB→BGR8, ~0.6ms 处理 2×1440×1080。

### 3. 异步Grab线程 (零拷贝按需模式)

```
主线程                    Grab线程
  │                         │
  ├── requestGrab(slot) ──→ │ 唤醒
  ├── Stage1 detect(N)      │ 锁定VPI Image
  ├── Stage2 fuse(N-1)      │ 调用grabFramePair
  ├── waitGrab() ←───────── │ 写入完毕, 解锁, 通知
  └── stage0_process()      │ 休眠等待下次请求
```

**零拷贝**: Tegra统一内存, Grab线程直接写入VPI Image缓冲区, 无staging buffer, 无memcpy。

### 4. 缓存CUDA指针 (消除VPI lock/unlock)

Tegra统一内存的CUDA设备地址在VPI Image生命周期内固定不变。  
Init时一次性缓存所有Slot的CUDA指针:

```cpp
auto cachePtr = [](VPIImage img, CachedGPU& out) {
    VPIImageData d;
    vpiImageLockData(img, VPI_LOCK_READ, VPI_IMAGE_BUFFER_CUDA_PITCH_LINEAR, &d);
    out.data = d.buffer.pitch.planes[0].data;
    out.pitchBytes = d.buffer.pitch.planes[0].pitchBytes;
    vpiImageUnlock(img);
};
```

运行时直接使用缓存指针, 省去每帧8次VPI lock/unlock (~2.4ms → 0ms)。

### 5. 深度融合策略

| 距离 | 方法 | 原理 |
|------|------|------|
| 0-4m | 单目几何 | 关键点拟合圆半径→距离, 近距离精度高 |
| 4-15m | 双目视差 | ROI SAD匹配→三角测距, 远距离可靠 |
| 3-5m | 加权过渡 | 两种方法加权融合, 平滑切换 |

6自由度 Kalman 滤波: 状态 [x, y, z, vx, vy, vz], 匀速模型。

### 6. ROI多点匹配 (vs 全帧视差)

| 方案 | 耗时 | 说明 |
|------|------|------|
| 全帧 VPI SGM | ~70ms | 全分辨率视差图 |
| 半分辨率 VPI SGM | ~12ms | 有效率 99% |
| **ROI 5×5 SAD** | **<0.1ms** | **仅计算检测区域** |

ROI模式: 在每个检测框内取5×5网格25个点, 每点做 blockSize×blockSize SAD块匹配, IQR中值滤波去野值。

## CUDA Stream拓扑

```
cudaStreamGrab    — Grab线程 (Camera SDK回调)
cudaStreamBGR     — Bayer→BGR kernel (非阻塞)
vpiStreamRectify  — VPI Remap (CUDA后端)
cudaStreamDetect  — TensorRT推理 (enqueueV3)
cudaStreamROI     — ROI SAD匹配 + 深度提取
```
