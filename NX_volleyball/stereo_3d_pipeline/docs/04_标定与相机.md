### BayerRG8 颜色映射注意事项

海康 SDK 输出的 BayerRG8 对应 OpenCV 的 **BayerBG** (行列偏移不同):
```
海康 BayerRG8 → OpenCV COLOR_BayerBG2BGR
```
**常见错误**: 使用 `COLOR_BayerRG2BGR` 导致红蓝互换。

### GigE 网络优化 (如用GigE版本)

- 设置 MTU 9000 (巨帧)
- 增大接收缓冲区: `sysctl -w net.core.rmem_max=26214400`

## 双目标定

### 工具链

| 工具 | 路径 | 用途 |
|------|------|------|
| capture_chessboard | calibration/capture_chessboard.py | 采集棋盘格图像 |
| stereo_calibration | calibration/stereo_calibration.py | 计算标定参数 |
| stereo_depth_test | calibration/stereo_depth_test.py | 深度验证 |

### 标定流程

```
1. 打印棋盘格 (推荐 9×6, 格子 25mm)
2. 采集图像对 (20-50对, 覆盖各角度)
   python capture_chessboard.py --pairs 30
3. 计算标定参数
   python stereo_calibration.py --input images/ --output stereo_calib.yaml
4. 验证深度精度
   python stereo_depth_test.py --calib stereo_calib.yaml
```

### 质量标准

| 指标 | 合格标准 | 良好标准 |
|------|---------|---------|
| 单目 RMS | < 0.5 px | < 0.3 px |
| 立体 RMS | < 1.0 px | < 0.5 px |
| 重投影误差 | < 1.0 px | < 0.5 px |

### 图像采集要求

- 覆盖视野的四角和中心
- 包含不同距离 (0.5m - 3m)
- 棋盘格倾斜角度 ≤ 45°
- 避免运动模糊和反光
- 左右图像必须同时采集 (硬件触发)

### 当前标定结果

| 参数 | 值 |
|------|-----|
| Baseline | 328.71 mm |
| 立体 RMS | 1.11 px |
| 有效深度范围 | 0.5 - 15 m |

## PWM 硬件触发

### 原理

GPIO → PWM 方波 → 海康相机 Line0 (上升沿触发) → 双目同步采集

### libgpiod 实现

```
频率: 100Hz (周期 10ms)
占空比: 50% (高电平 5ms)
调度: SCHED_FIFO 优先级 80
内存: mlockall(MCL_CURRENT | MCL_FUTURE)
定时: clock_nanosleep(TIMER_ABSTIME) → 抖动 < 10μs
```

### PWM 修复历史

- **问题**: `sleep_for` 导致频率抖动 ±5%
- **修复**: 改用 `clock_nanosleep(TIMER_ABSTIME)` + SCHED_FIFO + mlockall
- **结果**: 抖动从 ±500μs 降至 < 10μs
