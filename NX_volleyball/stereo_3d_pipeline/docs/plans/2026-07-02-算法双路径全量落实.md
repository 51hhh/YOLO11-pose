**Goal:** 让"离线图片测试(Python 探针)"与"NX 实时管线(C++)"两条路径的取点/特征测距算法按优先级对齐。实时侧缺失算法必须先通过语义对拍和 NX profiler 门槛,再进入候选链路;默认配置必须守住 100 FPS。

- `roi_subpixel` 与 `all_sparse_gpu` 实测不能守 100Hz,必须保持默认关闭。

## 设计决策

**D1. 新增 GPU 候选优先复用 `dual_yolo_depth_gpu.cu` 的批处理入口,但必须按实测成本门控。** 同 kernel 分派能减少固定 launch/sync 开销,但 NX profiler 已证明 sparse 特征循环不是近零成本。final3 中 default geometry 为 0.26 ms/约 100 FPS,单项多数可用,但 `roi_subpixel` 69.7 FPS、`all_sparse_gpu` 89.8 FPS。所有新方法必须默认关闭,逐项 profiler,不得把"同 kernel"等同于"免费"。

**D2. 彩色方案(iou_region_color_patch / patch_iou_color_edge)**:已把 `rectBGR` 指针传入 kernel,在 ROI 内现算 BGR label、color likelihood、edge/patch score,不预留全帧 buffer。当前是实时快速近似,后续要和离线 LAB 版本对拍。

**D3. ORB/BRISK/AKAZE/SIFT 以 GPU-lite 形态落地。** 为满足实时链路和默认 100Hz,不实现完整 OpenCV 尺度空间/描述子语义;ORB/BRISK 使用二值 patch/census 风格候选,AKAZE/SIFT 使用梯度/纹理风格候选。CPU OpenCV 仅作为无 GPU 候选时的 fallback/A-B。

**D4. 神经匹配三件套用统一 extractor→matcher 契约:**
- XFeat / ALIKED:extractor engine 输出 keypoints/descriptors/scores → GPU 描述子互近邻 NN + 极线/视差/簇 gate。
- SuperPoint+LightGlue:extractor + matcher(或 fused)engine。当前实时实现支持 fused engine 输出直接给出匹配点;split extractor/matcher 仍待实现。
- 固定 ROI 尺寸(224,config 已有),左右 ROI 尽量 batch。
- 所有神经匹配输出**必须过与传统方案同一套 gate**(y残差/视差delta/球体半径),不裸信网络。

**D5. 每个新算法两条路径共用同一份「参考实现语义」**:先在离线探针补齐(若缺),再在实时 kernel 复刻并对拍(disparity/depth 数值一致性验证),保证两路径不漂移。

**D6. 100fps 守护:所有新方法默认 `false`**;GPU 候选纳入现有 `Stage2_DualYoloGpuCandidates` 计时。`neural_time_budget_ms` 仍是待补项,真实神经 engine 上线前必须补独立预算闸并跑单项 profiler。

## 风险与决策点

| 风险 | 说明 | 处置 |
|---|---|---|
| **AKAZE GPU 化成本** | 非线性扩散尺度空间难 GPU 化 | Phase 2.3 先出评估;若不划算,标注保留 CPU,不阻塞其他阶段 |
| **SIFT 对光滑球无效** | 离线实测仅 1 点 | 最小实现 + 文档标注,不投入完整尺度空间 |
| **神经匹配 100fps** | 3 engine 推理大概率超预算 | 定位为「按需/低频/A/B」,不要求常开 100fps;必须有 time budget 闸 |
| **彩色 kernel 带宽** | BGR 比灰度多 3× 读 | ROI 内现算,限 `circle_max_roi_pixels`;实测带宽影响 |
| **两路径漂移** | 复刻语义易偏 | D5 对拍工具强制数值一致性 |
| **100fps 回归** | 新方法拖帧;NX 已证明 all_sparse_gpu 会降到 16-34 FPS | 全部默认关闭 + 每阶段 FPS 回归门;禁止在默认配置同时开启全部 sparse GPU |

## 落地顺序理由

Phase 1(彩色)先做:离线冠军方案,收益最确定,且打通「BGR 指针入 kernel」这条新通路,后续复用。
Phase 2(传统 GPU 化)次之:复用现有稀疏点鲁棒聚合,风险中等。
Phase 3(神经)最后:工作量与风险最大,依赖 3.1 模型导出,且不要求常开,可独立推进不阻塞前两阶段交付。
