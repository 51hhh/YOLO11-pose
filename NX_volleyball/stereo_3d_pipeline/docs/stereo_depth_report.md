# 双目深度多点测量方案报告

> 平台：Jetson Orin NX Super 16GB | JetPack 6.2 | 双目海康 MV-CA016  
> 基线：328.7mm | 焦距：1671.2px | 分辨率：1440×1080 → 校正后 1280×720  
> 目标范围：0-15m | 排球直径：22cm | 帧率：100Hz

## 0. 距离-视差-单目分析

### 0.1 视差 vs 距离计算表

```
公式: disparity = focal × baseline / Z = 1671.2 × 328.7mm / Z

距离(m)  |  视差(px)  |  排球像素宽  |  推荐方法
---------|-----------|-------------|----------
  1.0    |   549.6   |   367.6     |  单目 (视差超限)
  2.0    |   274.8   |   183.8     |  单目 (视差超限)
  3.0    |   183.2   |   122.5     |  单目+双目交叉验证
  4.0    |   137.4   |    91.9     |  双目(search=160可覆盖)
  5.0    |   109.9   |    73.5     |  双目 (search=128覆盖)
  7.0    |    78.5   |    52.5     |  双目
 10.0    |    54.9   |    36.8     |  双目
 15.0    |    36.6   |    24.5     |  双目 (接近检测极限)

排球像素宽 = focal × 0.22m / Z
```

### 0.2 关键发现

- **search_range=128** 只能覆盖 Z ≥ 4.3m
- **0-4m 近距离**: 视差超出搜索范围，**必须用单目估计**
- **4-15m**: 双目 ROI SAD 有效
- **3-5m 过渡带**: 两种方法均可用，交叉验证

## 1. 现有方案概览

系统已实现三种深度计算策略，通过 `pipeline.yaml` 的 `stereo.strategy` 切换：

| 策略 | 实现模块 | 耗时 | 精度 | 适用场景 |
|------|---------|------|------|---------|
| `full_frame` | VPI SGM 全帧视差 → 直方图峰值融合 | ~64ms | 高（密集视差） | 环境重建、离线分析 |
| `half_resolution` | VPI SGM 半分辨率 → 视差×2补偿 | ~16ms | 中 | 折中方案 |
| **`roi_only`** | **5×5 网格 SAD 匹配 → 中值三角测距** | **<0.5ms** | 中-高（依赖纹理） | **100Hz 实时追踪** |

### 1.1 当前生产配置（pipeline_roi.yaml）

```yaml
stereo:
  strategy: roi_only           # 仅在检测到的 BBox 内算深度
  max_targets: 32              # 最多同时测 32 个目标
  patch_radius: 3              # SAD 匹配块 7×7
  search_range: 128            # 搜索范围 128px (对应近端 ~1.2m)
  uniqueness_ratio: 0.25       # 唯一性检验 (secondBest - best) / best > 0.25
  min_confidence: 0.3          # 最低置信度阈值
```

## 2. ROI 多点匹配算法详解

### 2.1 算法流程

```
输入: 校正后左右图(1280×720) + N个检测BBox (max 32)
      ↓
┌─────── 每个 BBox = 1 个 CUDA Block (max 25 threads) ───────┐
│                                                              │
│  1. 自适应网格: 根据BBox大小选择采样密度                        │
│     - <30px:  3×3 = 9 点                                     │
│     - <60px:  4×4 = 16 点                                    │
│     - ≥60px:  5×5 = 25 点                                    │
│                                                              │
│  2. 每线程处理1个采样点:                                       │
│     a. 在左图取 (2r+1)² 块                                    │
│     b. 沿右图极线水平搜索 SAD 最小值                            │
│     c. 唯一性检验: (2nd_best - best) > best/4                 │
│     d. 亚像素拟合: d_sub = d + (S₋ - S₊)/(2*(S₋ - 2S₀ + S₊)) │
│                                                              │
│  3. Thread 0 汇总:                                            │
│     a. 冒泡排序所有有效视差                                     │
│     b. IQR 过滤: 只取 Q1~Q3 范围内数据                          │
│     c. 中值视差 d_median = mean(filtered)                      │
│     d. Z = focal × baseline / d_median                        │
│     e. X = (cx - cx₀) × Z / focal                            │
│     f. Y = (cy - cy₀) × Z / focal                            │
│     g. confidence = validRatio × min(1, 1/IQR)                │
└──────────────────────────────────────────────────────────────┘
输出: Object3D { x, y, z, confidence } × N
```

### 2.2 性能实测

| 指标 | 数值 | 说明 |
|------|------|------|
| 1目标延迟 | ~0.01ms | 单个BBox 25点匹配 |
| 10目标延迟 | ~0.1ms | 10个BBox并行 |
| 32目标延迟 | ~0.3ms | 满载，瓶颈在全局内存带宽 |
| GPU 占用率 | <1% | 仅启用1-32个Block |

### 2.3 精度分析

**优点**:
- 仅在 BBox 内采样 → 零背景干扰
- IQR 过滤 → 抗遮挡/纹理缺失
- 亚像素拟合 → 理论精度 ±0.5px → 在3m处 Z 误差约 ±3mm

**局限**:
- 依赖纹理：排球纯白区域可能匹配失败 (置信度低)
- 最大搜索范围 128px → 最近测距 ~4.3m (可调)
- 不产生密集视差图 → 无法做碰撞检测/环境建模

## 3. 推荐方案: 单目+双目混合测距 + Kalman 滤波

### 3.1 架构设计

```
输入: YOLO 检测结果 (BBox + keypoints) + 校正后左右图
          ↓
    ┌──── 距离预估(上一帧 Kalman 预测 或 BBox 面积) ────┐
    │                                                    │
    ▼                                                    ▼
 Z_pred < 4m?                                     Z_pred ≥ 4m?
    │                                                    │
    ▼                                                    ▼
 [单目测距]                                          [双目 ROI SAD]
 Z = f × D_real / bbox_px                          5×5 网格匹配
 D_real = 0.22m (排球)                              Z = f×B/d
    │                                                    │
    └──────────────┬─────────────────────────────────────┘
                   ▼
             3-5m 过渡带?
              YES → 加权平均: Z = α×Z_mono + (1-α)×Z_stereo
                    α = (5 - Z_pred) / 2  (线性过渡)
                   ▼
            [Kalman Filter 时域滤波]
            状态: [Z, Vz]  (距离 + 径向速度)
            观测: Z_raw
            过程噪声 Q: 根据排球最大加速度 (~30m/s²)
            观测噪声 R: 根据测量方法动态调整
              - 单目: R = 0.5m  (BBox 精度有限)
              - 双目: R = 0.02m (亚像素精度)
              - 过渡带: R = 线性插值
                   ▼
            输出: Z_filtered, Vz_estimate, confidence
```

### 3.2 单目测距原理

已知排球直径 D_real = 0.22m，通过 BBox 宽度 w_px 估计距离：

```
Z_mono = focal × D_real / w_px
       = 1671.2 × 0.22 / w_px
       = 367.6 / w_px  (meters)

精度分析:
  Z=1m, w=368px → ΔZ/Δw ≈ Z²/(f×D) = 1/(f×D/Z) ≈ 0.003m/px → ±1cm
  Z=3m, w=123px → ≈ 0.024m/px → ±5cm   ← 精度足够
  Z=5m, w= 74px → ≈ 0.068m/px → ±14cm  ← 开始粗糙
```

优点：
- **不受视差搜索范围限制** → 0m 到 ∞ 均可工作
- **零额外计算** → 使用已有 BBox 数据
- **排球标准尺寸** → 先验知识强

局限：
- 依赖 BBox 精度（YOLO BBox 比实际球体偏大）
- 非排球形目标需要不同的参考尺寸
- 视角/透视导致球体投影非正圆

### 3.3 双目 ROI SAD 匹配（4m+ 距离）

保持现有 CUDA kernel 不变，调整参数：

```yaml
stereo:
  strategy: roi_only
  search_range: 160       # 从128增到160, 覆盖 ~3.4m
  patch_radius: 4         # 从3增到4, 9×9块更鲁棒
  uniqueness_ratio: 0.25
  min_confidence: 0.3
  max_targets: 32
```

### 3.4 Kalman 滤波器设计

```cpp
// 状态向量: [z, vz] (距离, 径向速度)
// 预测模型: z(k+1) = z(k) + vz(k) × dt
//           vz(k+1) = vz(k)
//
// dt = 0.01s (100Hz)

struct KalmanState {
    float z;           // 距离 (m)
    float vz;          // 径向速度 (m/s)
    float P[2][2];     // 协方差矩阵
    int age;           // 连续跟踪帧数
};

// 过程噪声 (排球加速度 ~30m/s² → σ_a = 30)
// Q = [dt⁴/4  dt³/2] × σ_a² = [2.5e-9  5e-7] × 900
//     [dt³/2  dt²  ]           [5e-7    1e-4]

// 观测噪声 (距离依赖)
float R_mono = 0.25f;   // 单目: σ ≈ 0.5m → R = 0.25
float R_stereo = 0.01f;  // 双目: σ ≈ 0.1m → R = 0.01
```

### 3.5 连续检测的丢帧处理

```
检测到目标 → Kalman update (测量更新)
未检测到   → Kalman predict only (纯预测)
  ├─ 连续丢失 < 5帧: 保持跟踪, 置信度递减
  ├─ 连续丢失 5-20帧: 降级为"预测"状态
  └─ 连续丢失 > 20帧: 删除跟踪
```
