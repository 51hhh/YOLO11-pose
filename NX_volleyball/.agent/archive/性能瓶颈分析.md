# 性能分析报告 - 深度瓶颈分析

## 🚨 当前核心问题诊断 (2026-01-25)

### 问题1: 串行阻塞架构

当前程序是**严格串行**执行的，CPU/GPU在处理图像时，相机回调被阻塞。

```
当前执行流程 (串行 - 慢):
┌──────────────────────────────────────────────────────────────────────┐
│ PWM触发 → [等待回调 9ms] → [检测 16ms] → [后处理] → [等待下一帧...]  │
│                                                                      │
│  相机空闲 ────────────┬──────────────────────────────────────►      │
│                        └── 这16ms相机本可以预取下一帧！              │
└──────────────────────────────────────────────────────────────────────┘
```

**问题**: 检测期间相机闲置，浪费了16ms的预取时间。

### 问题2: 推理速度本身不达标

```
当前推理速度: 16.7ms → 理论上限 1000/16.7 ≈ 60 FPS
目标帧率:     150 FPS → 需要 6.6ms/帧
差距:         60 FPS vs 150 FPS (相差 2.5倍)
```

### 问题3: CPU预处理瓶颈

日志显示 `GPU并行(预处理+推理+D2H): 16.64ms`，其中包含:
- CPU上的 `cv::resize` 和数据拷贝: ~3-5ms
- cudaMemcpy H2D: ~1-2ms
- 实际GPU推理: ~10ms

## 🛠️ 解决方案一: 流水线架构 (推荐)

将"采集"和"推理"解耦到不同线程，通过队列通信:

```
目标架构 (流水线并行 - 快):
═══════════════════════════════════════════════════════════════════════

线程1 (采集): [PWM触发] → [回调取帧] → [放入队列] → [立即返回] → [下一帧]
                              │
                              ▼ 队列 (size=2, 无锁环形缓冲)
                              │
线程2 (推理): [从队列取] → [预处理+推理+后处理] → [发布结果]

时间重叠:
T0        T10       T20       T30       T40
├─────────┼─────────┼─────────┼─────────┤
│ 采集N   │ 采集N+1 │ 采集N+2 │ 采集N+3 │  ← 线程1
│         │ 推理N   │ 推理N+1 │ 推理N+2 │  ← 线程2
└─────────┴─────────┴─────────┴─────────┘
                              
延迟 = max(采集, 推理) ≈ 16ms，而非 采集+推理 = 25ms
```

### 代码实现:

```cpp
// 新增: 帧队列 (无锁环形缓冲)
#include <atomic>
template<size_t N>
class FrameQueue {
public:
    struct Frame {
        cv::Mat left, right;
        rclcpp::Time stamp;
    };
    
    bool push(Frame&& f) {
        size_t w = write_.load();
        size_t next = (w + 1) % N;
        if (next == read_.load()) return false;  // 队列满
        buffer_[w] = std::move(f);
        write_.store(next);
        return true;
    }
    
    bool pop(Frame& f) {
        size_t r = read_.load();
        if (r == write_.load()) return false;  // 队列空
        f = std::move(buffer_[r]);
        read_.store((r + 1) % N);
        return true;
    }
    
private:
    std::array<Frame, N> buffer_;
    std::atomic<size_t> read_{0}, write_{0};
};

// 采集线程
void captureThread() {
    while (running_) {
        cam_left_->waitForNewFrame(20);
        cam_right_->waitForNewFrame(5);
        
        FrameQueue<3>::Frame frame;
        frame.left = cam_left_->getLatestImage().clone();  // 必须clone
        frame.right = cam_right_->getLatestImage().clone();
        frame.stamp = this->get_clock()->now();
        
        frame_queue_.push(std::move(frame));
    }
}

// 推理线程
void inferenceThread() {
    FrameQueue<3>::Frame frame;
    while (running_) {
        if (frame_queue_.pop(frame)) {
            auto [det_l, det_r] = detector_->detectDual(frame.left, frame.right, 640);
            // 后处理 + 发布...
        } else {
            std::this_thread::sleep_for(std::chrono::microseconds(100));
        }
    }
}
```

**预期收益**: 
- 帧率从 38 FPS → **55-60 FPS** (+50%)
- 延迟从 26ms → **16ms** (-40%)

## 🛠️ 解决方案二: GPU预处理 (CUDA核函数)

当前: CPU上 `cv::resize` + `cv::cvtColor` → cudaMemcpy → GPU推理
优化: 直接上传原图 → GPU kernel resize/normalize/CHW转换

```cpp
// preprocess_kernel.cu
__global__ void preprocessKernel(
    const uchar3* src, float* dst,
    int src_w, int src_h, int dst_w, int dst_h) {
    
    int x = blockIdx.x * blockDim.x + threadIdx.x;
    int y = blockIdx.y * blockDim.y + threadIdx.y;
    if (x >= dst_w || y >= dst_h) return;
    
    // 双线性插值采样
    float scale_x = (float)src_w / dst_w;
    float scale_y = (float)src_h / dst_h;
    float src_x = x * scale_x;
    float src_y = y * scale_y;
    
    // ... 双线性插值 ...
    
    // BGR → RGB, 归一化, CHW转换
    dst[0 * dst_w * dst_h + y * dst_w + x] = r / 255.0f;  // R通道
    dst[1 * dst_w * dst_h + y * dst_w + x] = g / 255.0f;  // G通道
    dst[2 * dst_w * dst_h + y * dst_w + x] = b / 255.0f;  // B通道
}

// 调用
void launchPreprocessKernel(const uchar3* src, float* dst, 
                            int src_w, int src_h, int dst_w, int dst_h,
                            cudaStream_t stream) {
    dim3 block(32, 32);
    dim3 grid((dst_w + 31) / 32, (dst_h + 31) / 32);
    preprocessKernel<<<grid, block, 0, stream>>>(src, dst, src_w, src_h, dst_w, dst_h);
}
```

**预期收益**: 预处理从 3-5ms → **<1ms** (-80%)
