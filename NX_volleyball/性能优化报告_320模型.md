# 🚀 320模型帧率优化总结

## 📊 问题分析

### 当前性能
- **理论FPS**: 116-119 Hz（基于推理时间8-9ms）
- **实际FPS**: 70-73 Hz
- **性能差距**: 约40% 的性能损失

### 瓶颈定位

```
推理循环时间分解:
├─ 推理时间: 8.5ms ✅ (已优化)
├─ 帧等待: ~5ms ⚠️ (瓶颈1)
├─ 同步sleep: 0.1-0.2ms ⚠️ (瓶颈2)  
├─ 图像clone: 0.5-1ms ⚠️ (瓶颈3)
└─ ROS发布: 0.5-1ms (可接受)

总耗时: ~14ms → 实际FPS ~70Hz
```

## ✅ 已实施优化

### 1. 减少帧等待超时
**位置**: `volleyball_tracker_node.hpp:116`
```cpp
// 优化前
static constexpr int FRAME_WAIT_TIMEOUT_MS = 5;

// 优化后  
static constexpr int FRAME_WAIT_TIMEOUT_MS = 1;
```
**预期提升**: 每次循环节省4ms → +5-8 FPS

### 2. 减少同步重试休眠
**位置**: `volleyball_tracker_node.hpp:117`
```cpp
// 优化前
static constexpr int SYNC_RETRY_SLEEP_US = 100;

// 优化后
static constexpr int SYNC_RETRY_SLEEP_US = 10;
```
**预期提升**: 减少90us等待 → +2-3 FPS

### 3. 消除重复图像赋值
**位置**: `volleyball_tracker_node.cpp:541-543`
```cpp
// 优化前：重复赋值
img_left_ = left;  // 第1次在473行
img_right_ = right;
// detectVolleyball() 中又赋值一次 (第2次)

// 优化后：只在473行赋值一次
```
**预期提升**: 节省内存操作 → +1-2 FPS
