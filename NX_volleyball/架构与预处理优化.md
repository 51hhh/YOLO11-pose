# 架构与预处理优化方案

> 硬件: Jetson Orin NX 16GB · 海康 GigE 双目 1440×1080 · PWM 100Hz 触发

---

## 1. Bayer RG8 CUDA 去马赛克

### 方案
RGB8 模式传输 4.67MB/帧（3通道），带宽瓶颈限制 ~76fps。
改用 **BayerRG8 单通道**（1.56MB/帧），GPU 端 CUDA 去马赛克：

```
相机 → BayerRG8(1ch) → GigE传输 → GPU CUDA demosaic → RGB → TRT推理
```

### 性能
| 模式 | 帧数据量 | 传输帧率 | 去马赛克 |
|---|---|---|---|
| RGB8 | 4.67 MB | ~76 fps | 无需 |
| **BayerRG8** | **1.56 MB** | **>100 fps** | 1.8ms GPU |

### 关键代码
- `detect_preprocess.cu`: `bayerToRGB_kernel` — 双线性插值去马赛克
- 注意: 海康 SDK 报告 `BayerRG8` 实际为 OpenCV `BayerBG`（RG/BG命名反转）

## 2. 串行→流水线瓶颈

### 问题
推理 8.2ms + 采集等待 ~12ms = 串行周期 20ms，实测仅 75-80 FPS（理论 100Hz）。

```
串行: 采集[9ms] → 等待[2ms] → 推理[8.2ms] → 等下一帧...  → ~50fps
```

### 双缓冲流水线方案

```
曝光N:   [──9ms──]
推理N-1:  [──8.2ms──]
曝光N+1:           [──9ms──]
推理N:              [──8.2ms──]
→ 受限于PWM周期 10ms，实际 100 FPS
```

核心改动：
1. **双缓冲指针交换** — 相机写 A，推理读 B，回调时原子交换
2. **条件变量** — 替代轮询 `waitForNewFrame(1ms)`
3. **零拷贝** — `cudaHostAllocMapped` 避免 H2D memcpy

### 已实现
- `stereo_3d_pipeline/src/pipeline/` — 四级流水线（Grab+Rectify → Detect → Stereo → Fuse）
- `stereo_3d_pipeline/src/utils/zero_copy_alloc.h` — 零拷贝内存池
- `stereo_3d_pipeline/src/pipeline/frame_slot.h` — 三缓冲帧槽
- 实测 19.7 FPS（单线程顺序），优化后目标 60-100 FPS

## 3. PWM 与 Bayer 修复记录

| 问题 | 原因 | 修复 |
|---|---|---|
| PWM 精度不足 | `usleep` 精度差 | 改用 `clock_nanosleep` + `SCHED_FIFO` |
| Bayer 颜色偏红 | RG/BG 命名映射反转 | SDK `BayerRG8` 对应 OpenCV `COLOR_BayerBG2RGB` |
| GPIO 权限 | `/dev/gpiochip*` 无权限 | udev rules + libgpiod 用户态访问 |
