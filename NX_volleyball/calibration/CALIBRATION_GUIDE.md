### 2. 采集标定图像

#### 编译采集工具
```bash

```

#### 运行采集程序

```bash
./capture_stereo

# 或指定相机索引
./capture_stereo 0 1
```

#### 采集要求

**图像数量**: 15-20 对图像（最少10对）

**棋盘格摆放要求**:
- ✅ **不同位置**: 画面中心、四个角落、边缘
- ✅ **不同角度**: 平行、倾斜15°、倾斜30°、倾斜45°
- ✅ **不同深度**: 近距离(1m)、中距离(3m)、远距离(5m)
- ✅ **充满视野**: 棋盘格占据画面的30-70%
- ⚠️ **避免**: 运动模糊、过曝、欠曝、遮挡

**操作提示**:
```
空格键  - 保存当前帧对
q/ESC  - 退出程序
```

**采集技巧**:
1. 确保棋盘格**完整出现**在两个相机视野中
2. 保持棋盘格**静止**1-2秒再按空格
3. 每次改变位置后等待相机**自动曝光稳定**
4. 尽量覆盖工作区域的**所有深度和角度**

### 3. 执行标定

#### 运行标定脚本
```bash
cd ~/NX_volleyball/calibration

# ⚠️ 先用调试模式查看第一张图像
python3 stereo_calibration.py \
    --square-size 25.60 \
    --pattern-width 8 \
    --pattern-height 5 \
    --debug

# 调试模式会：
# 1. 显示原始图像
# 2. 显示灰度图
# 3. 显示检测结果（绿色=成功，红色=失败）
# 4. 提示是否找到正确的角点数
# 按任意键继续，ESC退出

# 如果调试模式检测成功，去掉--debug继续：
python3 stereo_calibration.py \
    --square-size 25.60 \
    --pattern-width 9 \
    --pattern-height 6

# 如果检测失败，尝试其他角点数：
# 常见配置：
# - 10x7格子 → 9x6角点
# - 11x8格子 → 10x7角点
# - 8x6格子 → 7x5角点
python3 stereo_calibration.py \
    --square-size 25.60 \
    --pattern-width 10 \  # 尝试其他值
    --pattern-height 7 \   # 尝试其他值
    --debug

# 自定义参数
python3 stereo_calibration.py \
    --square-size 实际测量值 \
    --pattern-width 横向角点数 \
    --pattern-height 纵向角点数 \
    --output my_calib.yaml
```

#### 参数说明
| 参数 | 说明 | 示例 |
|------|------|------|
| `--square-size` | 方格边长(mm)，**必须精确测量** | 25.3 |
| `--pattern-width` | 横向内部角点数 = 列数-1 | 9 |
| `--pattern-height` | 纵向内部角点数 = 行数-1 | 6 |
| `--images-dir` | 图像目录 | calibration_images |
| `--output` | 输出文件名 | stereo_calib.yaml |

**注意**: 
- `pattern_width/height` 是**内部角点数**，不是格子数
- 例如 10x7 格子的棋盘 → 9x6 角点
- `square_size` 必须使用**实际测量值**，不能估算

### 4. 验证标定结果

#### 可视化校正效果
标定完成后会自动显示3组校正后的图像对。

**检查要点**:
- ✅ 绿色水平线应该在左右图像中**完美对齐**
- ✅ 同一物体在左右图像中应该在**同一水平线上**
- ❌ 如果有明显错位，说明标定失败

按任意键切换下一对图像，ESC退出。

#### 评估标定质量

**优秀标定**:
- 重投影误差 < 0.5 像素
- RMS误差 < 0.5
- 基线长度与实际测量值相差 < 5mm

**可接受标定**:
- 重投影误差 < 1.0 像素
- RMS误差 < 1.0
- 基线长度与实际测量值相差 < 10mm

**需要重新标定**:
- 重投影误差 > 1.5 像素
- RMS误差 > 1.5
- 立体校正后图像严重错位

## 🔧 常见问题

### 问题1: 无法检测到棋盘格角点

**原因**:
- 棋盘格参数设置错误
- 图像质量差（模糊、曝光不当）
- 棋盘格被遮挡或不完整

**解决**:
1. 确认 `pattern_width` 和 `pattern_height` 正确
2. 检查图像清晰度和曝光
3. 重新采集有问题的图像

### 问题2: 重投影误差过大

**原因**:
- square_size 测量不准确
- 棋盘格本身不平整
- 采集图像时棋盘格移动了
- 图像数量太少或分布不均匀

**解决**:
1. 重新精确测量方格尺寸
2. 更换平整的棋盘格
3. 采集更多图像（20+对）
4. 确保图像覆盖各种角度和深度

### 问题3: 立体校正后图像错位

**原因**:
- 相机在标定后移动了位置
- 标定质量不佳
- 左右相机索引搞反了

**解决**:
1. 确保相机固定不动
2. 重新标定
3. 检查相机索引是否正确

### 问题4: 深度测量不准确

**原因**:
- 基线长度标定错误
- 相机内参不准
- 图像校正质量差

**解决**:
1. 用实际基线长度验证标定结果
2. 重新标定
3. 增加标定图像数量

## 📚 参考资料

### 棋盘格尺寸建议

| 工作距离 | 方格尺寸 | 棋盘格大小 |
|---------|---------|-----------|
| 1-3m | 20-25mm | 9x6 角点 |
| 3-6m | 30-40mm | 9x6 角点 |
| 6-10m | 50-60mm | 7x5 角点 |

### 标定精度与基线的关系

基线越长，深度精度越高，但重叠视野越小：
- 基线 15cm: 适合 1-5m 测量
- 基线 25cm: 适合 2-10m 测量（当前配置）
- 基线 50cm: 适合 5-20m 测量

### OpenCV标定原理

双目标定包含三个步骤：
1. **单目标定**: 获取每个相机的内参和畸变
2. **立体标定**: 计算两相机间的旋转R和平移T
3. **立体校正**: 使极线水平对齐，简化立体匹配
