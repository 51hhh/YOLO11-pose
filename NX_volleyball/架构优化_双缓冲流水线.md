# 🚀 架构优化方案：双缓冲流水线并行

## 📊 当前性能瓶颈分析

### 实测数据（2026-01-26）
```
推理能力: 121 Hz (理论) ✅
实际FPS:  75-80 Hz     ⚠️ 仅达理论值的62-66%
```

### 🔴 根本原因：串行处理架构

```cpp
// 当前代码逻辑 (volleyball_tracker_node.cpp)
while (running_) {
    // 1. 等待新帧（轮询，阻塞）
    updateLeftFrame();   // waitForNewFrame(1ms) ← 1ms阻塞
    updateRightFrame();  // waitForNewFrame(1ms) ← 1ms阻塞
    
    // 2. 同步检查（可能失败重试）
    if (!waitForSyncedPair(...)) {
        sleep(10us);     // ← 重试累积延迟
        continue;
    }
    
    // 3. 推理处理（8.2ms）
    detectVolleyball();  // ← GPU工作
    
    // 循环回到步骤1，等待下一帧
}
```

**时间线图示**：
```
PWM触发: ──10ms──┬──10ms──┬──10ms──┬──10ms──┬──10ms──
                │       │       │       │
帧1曝光:  [──9ms──]
帧1传输:         [1ms]
帧1等待:          [2ms] ← updateFrame + waitForSyncedPair
帧1推理:              [──8.2ms──] ← GPU工作，相机空闲
                               │
帧2曝光:                       [──9ms──] ← 错过了触发时机
帧2处理:                               [10.2ms]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
实际周期: 9 + 1 + 2 + 8.2 = 20.2ms/帧

理论FPS: 1000 / 10 = 100 Hz ✅
实际FPS: 1000 / 20.2 ≈ 49 Hz，实测75-80 Hz (有部分overlap)
```

**三大损耗**：
1. ❌ **轮询等待**：2ms/帧（updateFrame × 2）
2. ❌ **同步重试**：sleep累积延迟
3. ❌ **串行执行**：曝光期间GPU空闲，推理期间相机等待

## ✅ 优化方案：双缓冲 + 条件变量流水线

### 核心思想

**让曝光和推理并行执行**，充分利用硬件资源：

```
PWM触发: ──10ms──┬──10ms──┬──10ms──┬──10ms──┬──10ms──
                │       │       │       │
帧1曝光:  [──9ms──]
帧1推理:         [8.2ms] ← GPU工作
                │
帧2曝光:         [──9ms──] ← 相机工作，GPU推理上一帧
帧2推理:                [8.2ms]
                        │
帧3曝光:                [──9ms──]
帧3推理:                       [8.2ms]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
实际周期: 10ms/帧 (受PWM限制)
实际FPS: 100 Hz ✅
```

### 架构设计

#### 1. 双缓冲机制

```cpp
class FrameBuffer {
public:
    cv::Mat image;
    FrameMetadata metadata;
    std::atomic<bool> ready{false};
};

class DoubleBuffer {
private:
    FrameBuffer buffer_[2];     // 双缓冲
    std::atomic<int> write_idx_{0};  // 写缓冲索引
    std::atomic<int> read_idx_{1};   // 读缓冲索引
    
public:
    // 相机线程调用：写入新帧（零拷贝指针交换）
    void push(cv::Mat&& img, const FrameMetadata& meta) {
        int idx = write_idx_.load();
        buffer_[idx].image = std::move(img);  // 移动语义
        buffer_[idx].metadata = meta;
        buffer_[idx].ready.store(true);
        
        // 交换缓冲区（原子操作）
        write_idx_.store(read_idx_.load());
        read_idx_.store(idx);
    }
    
    // 推理线程调用：读取帧
    std::pair<cv::Mat, FrameMetadata> pop() {
        int idx = read_idx_.load();
        buffer_[idx].ready.store(false);
        return {buffer_[idx].image, buffer_[idx].metadata};
    }
};
```

#### 2. 条件变量同步（取代轮询）

```cpp
class SyncedFramePair {
private:
    DoubleBuffer left_buffer_;
    DoubleBuffer right_buffer_;
    
    std::mutex sync_mutex_;
    std::condition_variable frame_ready_cv_;
    std::atomic<bool> pair_ready_{false};
    
public:
    // ========== 相机回调（在相机线程执行）==========
    void onLeftFrameReady(cv::Mat&& img, const FrameMetadata& meta) {
        left_buffer_.push(std::move(img), meta);
        checkAndNotify();
    }
    
    void onRightFrameReady(cv::Mat&& img, const FrameMetadata& meta) {
        right_buffer_.push(std::move(img), meta);
        checkAndNotify();
    }
    
    // ========== 检查同步并通知 ==========
    void checkAndNotify() {
        auto [left_img, left_meta] = left_buffer_.pop();
        auto [right_img, right_meta] = right_buffer_.pop();
        
        // PWM同步检查
        int frame_diff = std::abs(
            static_cast<int>(left_meta.frame_number) - 
            static_cast<int>(right_meta.frame_number)
        );
        
        if (frame_diff <= 3) {  // 同步成功
            std::lock_guard<std::mutex> lock(sync_mutex_);
            synced_left_ = left_img;
            synced_right_ = right_img;
            pair_ready_.store(true);
            frame_ready_cv_.notify_one();  // ⚡ 唤醒推理线程
        }
    }
    
    // ========== 推理线程调用：等待同步帧对 ==========
    std::pair<cv::Mat, cv::Mat> waitForSyncedPair() {
        std::unique_lock<std::mutex> lock(sync_mutex_);
        
        // ⚡ 条件变量等待（无CPU开销，精确唤醒）
        frame_ready_cv_.wait(lock, [this]{ 
            return pair_ready_.load(); 
        });
        
        pair_ready_.store(false);
        return {synced_left_.clone(), synced_right_.clone()};
    }
};
```

#### 3. 推理线程改造

```cpp
void VolleyballTrackerNode::inferenceLoop() {
    RCLCPP_INFO(this->get_logger(), "🧠 推理线程已启动 (流水线并行模式)");
    
    while (running_ && rclcpp::ok()) {
        // ========== 1. 等待同步帧对（条件变量，零CPU开销）==========
        auto [left, right] = synced_pair_.waitForSyncedPair();
        // ⚡ 此处被唤醒时，帧已准备好，无轮询延迟
        
        // ========== 2. 立即开始推理（8.2ms）==========
        auto [det_left, det_right] = detector_->detectDual(left, right);
        
        // ========== 3. 立体匹配 + 追踪 ==========
        if (det_left.valid && det_right.valid) {
            computeStereoMatch();
            updateTracker();
            publishResults();
        }
        
        frame_count_++;
        
        // 循环立即回到等待状态，下一帧曝光时GPU已在推理当前帧
    }
}
```

## ⚠️ 注意事项

### 1. 内存管理
- 使用 `std::move` 避免拷贝
- 双缓冲避免锁竞争
- 及时释放旧帧内存

### 2. 线程安全
- 条件变量必须配合 `unique_lock`
- 原子变量用于标志位
- 避免死锁（锁顺序一致）

### 3. 同步策略
- PWM帧号差异 ≤ 3
- 时间戳差异 < 25ms（2.5个周期）
- 失配帧立即丢弃，不累积

### 4. 测试验证
- 逐步启用，先验证同步率
- 监控丢帧率和延迟
- 对比优化前后性能
